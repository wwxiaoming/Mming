## 4. 你的"代表作 alpha"拆解

### 4.1 完整表达式

```python
group_neutralize(
    ts_decay_linear(
        ts_backfill(anl4_adjusted_netincome_ft, 120),
        240
    ),
    densify(subindustry)
)
```

### 4.2 逐行白话翻译

| 代码层 | 白话解释 |
|---|---|
| `anl4_adjusted_netincome_ft` | 分析师对"调整后净利润"的预测（未来期） |
| `ts_backfill(x, 120)` | 遇到空值用最近一次有效值填充，最多回看 120 天 |
| `ts_decay_linear(x, 240)` | 距离今天越近权重越大，240 天线性衰减到 0 |
| `densify(subindustry)` | 把子行业分类补全成完整的"股票×日期"矩阵 |
| `group_neutralize(x, ...)` | 在每个子行业内做 z-score 中性化 |

### 4.3 整体策略逻辑（一句话版）

> **买入**：所在子行业内，分析师最近 240 天持续看好（且预测未过期超过 120 天）的盈利前景的股票
>
> **卖出**：相反

类型：**分析师盈利预测动量 / 盈利预期因子**

### 4.4 为什么这条能跑出高 Sharpe？

| 设计 | 价值 |
|---|---|
| 用 **调整后净利润** 而非毛利润/营收 | 直接对应 EPS，最贴近"赚钱能力" |
| **120 天回填** | 解决分析师更新频率低导致的信号缺失 |
| **240 天线性衰减** | 抓的是"分析师观点的变化方向" |
| **子行业中性化** | 剔除"科技股整体高估"这类行业 β |

### 4.5 潜在风险与坑

1. **`_ft` 字段的发布时点** ⚠️ 最大风险点
   - 如果存在"前视偏差"（发布日早于实际公告日），样本外会崩
2. **子行业分类会变** → 历史信号可能"漂移"
3. **参数（120、240）可能过拟合** → 未来不一定稳
4. **公开信息因子** → 用的人越多超额收益越薄

## 5. Alpha 质量提升：从"会用工具"到"会改 alpha"

### 5.1 好 alpha 的 5+1 个健康指标

5 个硬指标 + 1 个隐藏指标：

| 指标 | 含义 | 健康区间 | 避坑 |
|---|---|---|---|
| **Sharpe** | 风险调整后收益 | 1.5 ~ 2.5 | > 1.25 即可过线，但越高越难维护 |
| **Fitness** | 性价比 × 持久性 | 1.5 ~ 3.0 | Sharpe 高但 Fitness 低 = 换手太频繁 |
| **Turnover** | 换手率 | 10% ~ 30% | < 1% 信号弱；> 70% 手续费吞噬收益 |
| **Returns** | 累计收益 | 越高越好 | 注意看 OS 表现 |
| **Drawdown** | 最大回撤 | 越小越好 | 真实交易最怕这个 |
| **OS Sharpe** ⭐ | 样本外夏普 | > IS 50% | 区分过拟合与真信号的关键 |

### 5.2 3 类算子的金融含义

#### 时序算子（对单只股票的时间序列操作）
- `ts_decay_linear(x, d)`：线性衰减，强调最近
- `ts_backfill(x, d)`：回填空值，保持连续
- `ts_delta(x, d)`：d 天变化量（抓"修订"）
- `ts_rank(x, d)`：在 d 天窗口内的排名（去极值）
- `ts_mean(x, d)` / `ts_std_dev(x, d)`：滚动均值/标准差

#### 截面算子（对同一日所有股票）
- `rank(x)`：在所有股票中的排名（0-1）
- `quantile(x, n)`：分成 n 等分
- `normalize(x)`：z-score 标准化

#### 中性化/向量算子
- `group_neutralize(x, group)`：按组做 z-score
- `densify(group)`：把分类字段稠密化
- `trade_when(条件, 开仓, 平仓)`：条件开仓

### 5.3 6 类经典 alpha 模式构造模板

```python
# 1. 动量（Momentum）
ts_delta(close, 20)  # 20日价格变化

# 2. 反转（Reversal）
-ts_delta(close, 5)  # 短期反转

# 3. 质量（Quality）
rank(roe)  # 盈利质量

# 4. 价值（Value）
-rank(pe)  # 低估值

# 5. 情绪（Sentiment）
ts_decay_linear(ts_delta(anl4_eps_mean, 60), 240)  # 分析师预期修订

# 6. 波动率（Volatility）
-ts_std_dev(returns, 60)  # 低波动率
```

### 5.4 5 个可立即尝试的改造案例（基于你的 anl4 alpha）

#### 案例 A：把"水平值"改成"60 天变化量"
```python
# 原版：分析师预测水平值
group_neutralize(
    ts_decay_linear(ts_backfill(anl4_adjusted_netincome_ft, 120), 240),
    densify(subindustry)
)
# 改造：分析师预测的修订方向
group_neutralize(
    ts_decay_linear(ts_delta(anl4_adjusted_netincome_ft, 60), 120),
    densify(subindustry)
)
```
> **改造原理**：不看"分析师觉得能赚多少"，而看"**分析师最近把预测调高还是调低了**"——更强动量信号

#### 案例 B：叠加价格动量过滤
```python
group_neutralize(
    ts_decay_linear(ts_backfill(anl4_adjusted_netincome_ft, 120), 240)
    * ts_decay_linear(ts_delta(close, 20), 60),  # 叠加20日动量
    densify(subindustry)
)
```
> 基本面预期 + 短期价格动量 = 双重确认

#### 案例 C：换数据集交叉验证
```python
group_neutralize(
    ts_decay_linear(ts_backfill(anl4_adjusted_netincome_ft, 120), 240)
    + 0.5 * rank(pv13_close_divide_ma),  # 叠加量价信号
    densify(subindustry)
)
```

#### 案例 D：换中性化层级
把 `densify(subindustry)` 依次试 `industry` / `sector` / `market`，对比哪个 Sharpe 更高。

#### 案例 E：换衰减函数
把 `ts_decay_linear` 换成 `ts_decay_exp` —— 指数衰减更"重视最近"，看哪种更稳定。

> 🎯 **改造优先级建议**：A → D → E → B → C（先单点改，再叠加改）

### 5.5 中性化层级的合理选择

| 层级 | 含义 | 适用 | 风险 |
|---|---|---|---|
| **Market** | 剔除整个市场 β | 高换手策略 | 信号消失 |
| **Sector** | 剔除行业倾向 | 跨行业横截面策略 | 行业 β 残留 |
| **Industry** | 剔除细分行业 | **多数情况下最优** | 样本量要够 |
| **Subindustry** | 剔除子行业 | 你现在用的 | **样本量小，可能过细** |

> 💡 你现在用 Subindustry，建议先回测 Industry 对比，看哪个更稳

### 5.6 常见"伪 alpha"的识别

- **数据前瞻偏差（look-ahead bias）**：用了"未来"才能拿到的字段
- **幸存者偏差**：只考虑当前还存在的股票
- **过度拟合**：IS 漂亮，OS 拉胯
- **流动性陷阱**：选了"理论上能买但实际买不到"的小票

## 6. 降低重复率：让 alpha 与众不同的 10 个技巧

### 6.1 降重的 3 大核心思路

| 思路 | 操作 | 难度 | 效果 |
|---|---|---|---|
| **A. 换数据集** | pv1 → analyst4 / option8 / news18 | 低 | 中 |
| **B. 换中性化** | Subindustry → Industry / Sector | 低 | 中 |
| **C. 换算子组合** | 用不常见的算子、组合方式 | 中 | 高 |

> 🎯 **核心原则**：**3 维度中改 2 个**，重复率立即下降 50%+

### 6.2 10 条可立即执行的降重技巧

1. **把"水平值"换成"变化量"** —— `ts_backfill` 改 `ts_delta(..., 60)`
2. **把单一数据集换成多数据集交叉** —— 加 pv1 量价信号
3. **把简单时序窗口换成多窗口组合** —— `ts_decay_linear` + `ts_rank`
4. **用 `trade_when` 加上开仓/平仓条件**
5. **把截面 rank 换成 quantile 分位数**
6. **用 `densify` 处理的字段做更细的中性化**
7. **尝试 option8 / option9 这类冷门期权数据** ⭐ 奖金池大
8. **尝试 news18 / socialmedia8 这类情绪数据的非常规用法**
9. **用 `ts_decay_exp` 替代 `ts_decay_linear`**
10. **参考论文改造** —— 读经典论文里的 alpha 公式

### 6.3 必读论文 List

1. **101 Formulaic Alphas**（WorldQuant 公开 101 个 alpha 公式）
2. **"Finding Alphas"**（WorldQuant 出版的 alpha 构造书）
3. **"Earnings Momentum"**（分析师预测修订经典论文）
4. **"Value Investing"**（Graham/Buffett 价值投资量化版）
5. **"Low Volatility Anomaly"**（低波动率异象）

### 6.4 撞库检测与应对

**提交前自检**：
- 用 2-3 年缩样本快速测试 Self-Correlation
- 看平台的"相似 alpha 推荐"列表
- 观察同 Region 同 Dataset 的常见组合

**被判重复后**：
- 先换数据集（最有效）
- 再换算子组合
- 最后改中性化层级

### 6.5 避坑提醒

- ⚠️ 单纯换 delay = 假差异化（不算）
- ⚠️ 加噪声降重 = 违反 IQC 协议第 23 条，会被封号
- ⚠️ 不同 Region 的同样表达式 = 真撞车
- ⚠️ 第三方工具内置的"降重" = 大多无效

# C.1 动量
ts_delta(close, 20)

# C.2 反转
-ts_delta(close, 5)

# C.3 质量
rank(roe)

-rank(pe)

# C.6 波动率
-ts_std_dev(returns, 60)

# C.7 三源共振模板（推荐组合）
signal_fundamental = rank(roe)
signal_analyst    = ts_decay_linear(ts_delta(anl4_eps_mean, 60), 240)
signal_technic    = ts_rank(close / volume, 20)
composite = signal_fundamental * signal_analyst * signal_technic
group_neutralize(
    trade_when(composite > 0, composite),
    densify(industry)
)
