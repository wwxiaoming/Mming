## 1. 总览：答题的两条铁律

> 这是两个教程反复强调的、所有方法论**最底层的出发点**。

### 铁律 1：理解 > 模板

- AI 是**翻译机**和**参考书**，**不是答题器**
- 拿到 AI 答案后必须"理解 → 改写"，不能"复制 → 提交"
- 一旦你照抄 1 句原话，你的整篇答案就被 AI 染色了

### 铁律 2：差异化 > 正确性

- 平台反作弊 = **相似度匹配**，可观测到的 4 类机制：
  - **文字相似度**（字符串匹配 / 编辑距离）
  - **语义相似度**（NLP 嵌入向量）
  - **答题时长**（5 分钟答完 9 题 = 可疑）
  - **IP / 设备指纹**（同 IP 多账号 = 高风险）
- 100% 正确的"标准答案"**会被雷同检测直接打死**
- 80% 正确但完全是你自己写的答案**才会被通过**

> 💡 **反直觉结论**：写错一点细节 ≠ 失败；和别人撞库 1 句 = 失败。

## 2. 5 步答题 SOP

```
Step 1  复制题目（保存到本地 + 截图原文）
Step 2  查官方资料（Learn 文档 / BRAIN 帮助中心）
Step 3  让 AI 解释（用 DeepSeek / 豆包 / IMA，⚠️ 必须开【新对话】）
Step 4  用知识库或官方文档验证 AI 答案的准确性
Step 5  拿到"骨架"后，自己改写、加个人理解、避开 AI 味
```

**关键心态**：每个 Step 都有**1 个最容易翻车的点**：

| Step | 翻车点 | 怎么避 |
|---|---|---|
| 1 | 复制时漏掉原题关键限定词 | 用截图不要用纯文本 |
| 2 | 引用第三方 CSDN 当主来源 | 只用 platform.worldquantbrain.com/learn |
| 3 | 在已有上下文的对话里继续问 | 每题开新对话 |
| 4 | 信任 AI 没核对 | 关键术语（vec_、densify、group_neutralize）必须对照官方 |
| 5 | 改写时偷懒保留 AI 套话 | 严格过 §4 的"红线词清单" |

## 3. AI 使用的 7 条铁律（乌龙视频提炼）

| # | 铁律 | 原因 |
|---|---|---|
| 1 | **每题开新对话** | 旧对话会污染新回答的语境 |
| 2 | **要求"2-3 句话回答"** | 越长越像 AI 模板话术 |
| 3 | **删掉所有括号注释** | `(其实这里指的是...)` 是 AI 最爱的套路，**100% 必删** |
| 4 | **删掉"举个例子便于理解"** | 看到这 6 个字基本就是 AI |
| 5 | **删掉"注意："作为开头** | "注意：..." 是 AI 教学风格的标志 |
| 6 | **不要用"如果你想..."** | 这 4 个字出现在答卷里非常违和 |
| 7 | **混合用英文专有名词** | 例如 BRAIN、neutralization、long count 直接用英文，**降低和其他人答案的撞库率** |

## 4. AI 味红线词清单（看到必删/必改）

### 4.1 必删的开头词
- "注意："
- "举个例子便于理解"
- "简单来说"
- "首先 / 其次 / 最后"（连续使用时）
- "如果你想..."
- "值得注意的是"

### 4.2 必删的括号注释
- `（这里的 X 指的是...）`
- `(其实 Y 是 Z 的另一种说法)`

### 4.3 必删的过渡词
- "此外"
- "不仅如此"
- "综上所述"
- "总而言之"

### 4.4 必改的句式

| AI 风格 | 改成你自己的说法（举例）|
|---|---|
| "这一过程的核心在于..." | "关键点是..." 或 "我理解的核心是..." |
| "从本质上讲..." | "说白了就是..." 或 "本质上" |
| "值得注意的是..." | 直接删掉这句 |
| "举例来说..." | "比如..." 或 "拿 anl4_adjusted_netincome_ft 来说" |
| "在金融领域..." | 删掉，直接说具体场景 |

## 5. Q4：Matrix Data vs Vector Data

> **题面**：What is the difference between Matrix Data and Vector Data?
>
> **⚠️ 必避陷阱**：题里"Vector"在 BRAIN 中**特指"事件型非规整数据"**，**不是**线性代数里的"一列数"。如果题问到"中性化是矩阵还是向量操作"——那里的"向量"是数学向量，**和本题无关**。详见 [01-wq-learn-concepts.md §A.4](01-wq-learn-concepts.md)。
>
> **答题类型**：**名词解释类**——按 §12 长度策略，3-5 句即可。

### 5.1 思路模板

**必讲点**（200 字以内）：
1. Matrix Data = 固定股票 × 日期的二维表，每天每只股票 1 个值
2. Vector Data = 每天每只股票对应 N 个值的事件型数据（N 可为 0、1、5、20…）
3. BRAIN 的 alpha 输出必须是 Matrix，所以 Vector 必须先经 `vec_*` 算子聚合
4. 举例：财务报表字段是 Matrix；逐条新闻、分析师评级是 Vector

**必避坑**：
- ❌ 把"Vector"答成数学向量（一列数）
- ❌ 漏掉"`vec_*` 算子"这一关键处理步骤
- ❌ 只说"一个是表格、一个是数组"——没落到 BRAIN 语境

**推荐结构**：**定义对比 → 举 1 个具体字段 → 落到 vec_ 算子 → 一句金融含义**

## 6. Q5：Long/Short Count

> **题面**：什么是 long count?什么是 short count?如何使用 long count 和 short count 的信息探查数据字段的统计特征？请给出具体案例并阐述其数学原理。
>
> **⚠️ 必避陷阱**：题里**4 个子问题**都要答到——"定义 × 2 + 探查用法 + 数学原理"，不能只解释 long 不解释 short。
>
> **答题类型**：**数学原理类 + 案例题**——按 §12 长度策略，3 段式（是什么 / 为什么 / 怎么算），每段 2-3 句 + 1 个自己跑过的案例。

### 6.1 思路模板

**必讲点**（200 字以内）：
1. **long count** = 截面日 alpha > 0 的股票数（被做多的池子）
2. **short count** = 截面日 alpha < 0 的股票数（被做空的池子）
3. **探查用法**：用 `(long + short) / universe_size` 反推覆盖率，判断数据字段是否健康
4. **数学原理**：指示函数求和 `L = Σ I(αᵢ > 0)`，与 `quantile` / `sign` 算子直接相关
5. **必须配合**：Long + Short 一起看，不能只算一边

**必避坑**：
- ❌ 把 long count 答成"做多赚的钱"——它只是**数量**不是金额
- ❌ 只算 long 不算 short
- ❌ 用"覆盖率"代替"统计特征"——覆盖率只是统计特征的一种
- ❌ 编造"Sharpe 提升 0.3 / 0.5"等具体数字——属于编造，平台可能核对

**推荐结构**：**4 个子问题分别作答**—— long 是什么 / short 是什么 / 怎么用（覆盖率反推 + 失衡检测）/ 数学原理（指示函数）

## 7. Q6：Neutralization（中性化）

> **题面**：请查阅 Learn 页面的有关文档，结合文档谈谈您对中性化（neutralization）的理解及其计算过程。
>
> **答题类型**：**数学原理类**——按 §12 长度策略，3 段式（是什么 / 为什么 / 怎么算），每段 2-3 句。

### 7.1 思路模板

**必讲点**（200 字以内）：
1. **是什么**：BRAIN 平台的一种预处理算子，作用是"剥离指定因子（市场/行业/子行业）对 alpha 信号的污染"
2. **为什么**：不中性化 → alpha 被行业 β 主导 → 选股能力被掩盖；中性化后 → 只剩"组内相对表现"
3. **怎么算**：分两步——按分组求组内均值 `μ_g`、每个值减去组均值 `a_i - μ_g`
4. **4 个层级**：Market / Sector / Industry / Subindustry，多数场景 Industry 最优
5. **z-score 是可选进阶**：`(a_i - μ_g) / σ_g` 属于 scale 范畴，**不是**中性化默认行为

**必避坑**：
- ❌ 把"中性化"答成"多空对冲"——中性化是**统计操作**（改数据），对冲是**组合操作**（改头寸）
- ❌ 写成"z-score 标准化"——BRAIN 默认是**减均值**，z-score 是可选
- ❌ 漏掉"4 个层级"中的某几个
- ❌ 漏掉 `densify(group)` 这一步的前置说明

**推荐结构**：**是什么 → 为什么 → 怎么算（落到公式）→ 4 层级对比表（可选）**

## 8. Q7：Alpha Example 复现 + 改进

> **题面**（典型问法）：请复现平台提供的某个 Alpha Example，并尝试改进。
>
> **⚠️ 必避陷阱**：Q7 是 9 道题里**最危险**的一题——它要"复现 + 改进"，极易抄原 example。**必须有"我做了什么改进"+"为什么这样改"**。
>
> **答题类型**：**案例题**——按 §12 长度策略，150-300 字 + 1 个对比指标（Sharpe / Fitness / Self-Corr 改进前后）。

### 8.1 思路模板

**必讲点**（200 字以内）：
1. **选一个简单的 example**（不要挑字段多、嵌套深的）—— 比如基于 `close` + `ts_mean` 的入门 example
2. **先完整跑通**（不改任何东西）——记录 Sharpe / Fitness / Self-Corr / Turnover
3. **改造 1-2 处**（三选一或组合）——换数据集（pv1 → anl4）/ 换中性化层级（None → Industry）/ 改参数（ts_mean(20,60) → ts_mean(5,20)）
4. **对比改进前后的指标**——至少 1 个对比指标
5. **解释"为什么这样改"**——必须带"我"字，体现自己的判断

**必避坑**：
- ❌ 直接背 example 的模板——平台会判定雷同
- ❌ 改进 3 处以上——超出"小幅改进"的语义，反而像过度包装
- ❌ 不写"为什么这样改"——失去"我"的印记
- ❌ 编造具体的 Sharpe 改进数字——用你自己平台截图里的真实值

**推荐结构**：**选哪个 example → 原始指标 → 改了哪 1-2 处 → 改进后指标 → 为什么这样改（带"我"）**

## 9. Q8：Alpha ID 获取

## 10. Q9：Genius 计划

> **题面**（典型问法）：请介绍 Genius Program 各等级及对应的季度奖金范围。
>

### 10.1 思路模板

**必讲点**（200 字以内）：
1. **6 个等级**：Bronze / Silver / Gold / Expert / Master / Grandmaster
2. **3 个积分制 + 3 个评估制**：Bronze / Silver / Gold 靠累计分；Expert / Master / Grandmaster 靠**季度评估**
3. **季度奖金范围**：Bronze-Silver-Gold = 0（积分制无奖）；Expert 起开始有；Master 上限较高；Grandmaster 上限最高
4. **Base Payment ≠ Quarterly Payment**：Base 是日结（按提交质量），Quarterly 是季度评估奖（按 OS 表现），是**两套钱**
5. **季度奖 = 0 是常态**——不是保底，要"上季度有提交 + OS 表现达标"才有

**必避坑**：
- ❌ 漏掉 Bronze / Silver / Gold 这 3 个等级
- ❌ 把 Master / Grandmaster 奖金范围记反
- ❌ 把 Base 和 Quarterly 混为一谈
- ❌ 漏掉"季度评估"这一关键机制
- ❌ 编造具体奖金数字——用 [03-ima-qa.md §2.5](03-ima-qa.md) 社区汇总范围

**推荐结构**：**6 个等级 + 升级机制 → 季度奖金范围（表格）→ Base vs Quarterly 区分 → 一句"季度奖 = 0 是常态"**

## 11. 改写 3 板斧（快速去 AI 味）

### 11.1 板斧 1 · "加一句我自己"

### 11.2 板斧 2 · "换比喻"

- AI 喜欢"先概念后举例"或"先定义后展开"。

## 12. 交卷前自检清单

> **关于"长度策略"**：本问卷**不卡字数**，但**写太长反而触发 AI 嫌疑**。建议按题型控制：

| 题型 | 建议长度 | 示例 |
|---|---|---|
| **名词解释类**（Q4 / Q8 / Q9 部分） | 3-5 句 | "Matrix 是 1 个数 / Vector 是 N 个数 / 用 vec_ 聚合" |
| **数学原理类**（Q5 / Q6） | 3 段式（是什么 / 为什么 / 怎么算），每段 2-3 句 | 见 §6 / §7 |
| **案例题**（Q7） | 150-300 字 + 1 个对比指标 | 见 §8 |

### 12.1 硬性检查

- [ ] **ID 和邮箱填对了**？（错一个直接作废）
- [ ] **没有括号注释**？`(其实...)` 全删了？
- [ ] **没有"注意 / 举个例子 / 如果你想"**？删干净了？
- [ ] **每个 Q 都有"我自己"的一句**？和别人拉开差异的核心。

### 12.2 风格检查

- [ ] **英文专有名词**混用了？BRAIN、neutralization、long count 都用了？
- [ ] **公式落到位了**？Q5 / Q6 至少各有一个数学符号？
- [ ] **整篇没有"首先 / 其次 / 再次"**？这是 AI 八股标志。

### 12.3 行为检查

- [ ] **用 30-60 分钟答完**？5 分钟交卷 = 可疑。
- [ ] **没换浏览器 / 没换 IP**？同 IP 多账号 = 高风险。
- [ ] **每题是单独新对话**？旧对话的语境不会污染新题。
