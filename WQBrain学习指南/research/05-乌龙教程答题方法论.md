## 3. Q4：Matrix Data vs Vector Data（3 个版本的"骨架"）

> **题面**：What is the difference between Matrix Data and Vector Data?
> **必须改写提示**：3 个版本只是"思路骨架"，每版你都需要**至少再改 30%**才能用。

### 版本 A · "维度视角"（推荐）
**骨架**：
- Matrix Data 是二维的——固定股票 × 日期 → 每天每只股票 1 个数。
- Vector Data 是一维向量的集合——每天每只股票有 N 个值（N 可为 0、1、5、20...）。
- 举例：`close` 是 Matrix；`nws12_afterhsz_sl` 是 Vector（每天 N 条盘后新闻各自的信号）。
- 金融含义差：Matrix 反映"稳定状态"（财报、估值），Vector 反映"瞬时分布"（舆情、事件）。

**怎么改**：
- 删掉"维度视角"4 个字（这是标题不是正文）
- 把"举例"换成你自己跑过的字段（比如你 anl4 用过哪些字段、撞库了哪些 Vector 字段）
- 加一句"我自己的体验"（如"我跑 anl4 时就发现 Vector 字段的覆盖率很不稳定"）

### 版本 B · "Excel 比喻"（直观）
**骨架**：
- Matrix 像 Excel 里股票 × 日期的成绩单，每个格子 1 个数字。
- Vector 像 Excel 里每个格子是一个小数组——你得决定用 `vec_avg` 还是 `vec_max` 等向量算子把它压成 1 个数。
- 区别是处理方式：Matrix 直接进 `ts_*` 算子；Vector 必须先经过 `vec_*` 算子降维。

**怎么改**：
- "Excel 成绩单"换成你自己熟悉的比喻（如记账本、K 线图、Excel 透视表）
- 加一句"在 BRAIN 上"的具体场景

### 版本 C · "代码视角"（技术）
**骨架**：
- Matrix Data shape = `(stock, date)`，每个 cell 是 scalar。
- Vector Data shape = `(stock, date, N)`，每个 cell 是长度为 N 的数组，N 每天每只股都可能不同。
- 算子选择上：Matrix 适合 `rank` / `ts_mean` / `group_neutralize`；Vector 必须先 `vec_avg` / `vec_std` / `vec_max` 等。

**怎么改**：
- 简化代码描述（你不需要答得像 API 文档）
- 加上"为什么不直接存 scalar"的金融解读

## 4. Q5：long count / short count（拆 3 个子问题）

> **题面**：什么是 long count?什么是 short count?如何使用 long count 和 short count 的信息探查数据字段的统计特征？请给出具体案例并阐述其数学原理?
> **必须改写提示**：这一题**结构化拆解**是关键，每个子问题都要有自己的"小标题"。

### 4.1 拆题 SOP（来自两个教程的共识）

| 子问题 | 必答内容 | 必避的坑 |
|---|---|---|
| **问题 1**：long count 是什么 | 定义 + 公式 | 不要答成"看多的股票数量"这种白话 |
| **问题 2**：short count 是什么 | 定义 + 公式 | 同上 |
| **问题 3**：怎么用它们探查数据 | 具体案例（如 anl4 覆盖率反推）| 必须有"具体数据 + 计算过程" |
| **问题 4**：数学原理 | 落到公式（与 quantile / sign 算子的关系）| 不能光说"统计意义" |

### 4.2 概念骨架（自己改写）

- **long count** = 在某个交易日，**alpha 表达式计算结果 > 0 的股票数量**（被做多的池子）。
- **short count** = 在某个交易日，**alpha 表达式计算结果 < 0 的股票数量**（被做空的池子）。
- **覆盖率反推法**：`coverage = (long_count + short_count) / universe_size`
  - 例如美股 TOP3000 每天跑一次 anl4 alpha，若 long = 1800、short = 1200，则覆盖率 = 100%（全覆盖）
  - 若 long = 900、short = 300，覆盖率 = 40%，说明字段在 60% 的股票上没值（缺失或为 0）
- **数学原理**：long count 是 alpha 在某截面上的**正部计数函数**，数学上对应 `sum(sign(alpha > 0))` 或 `count_where(alpha > 0)`，与 `quantile(alpha, 0.5)` 的"上下半样本"切分直接相关。

### 4.3 具体案例骨架（用你自己的 anl4 经历改写）

- 案例：跑 anl4_adjusted_netincome_ft + group_neutralize + subindustry 时，发现 long count 长期 < 1000（占 TOP3000 的 33%）。
- 推论：anl4 字段在 67% 的美股上**没有研报覆盖**（中小盘 + 新上市股票没分析师跟）。
- 改进：把 universe 从 TOP3000 切到 TOP2000，长短仓覆盖率立刻回到 95% 以上，Sharpe 提升 0.3。

> **怎么改**：把你的工具截图里的实际数字填进去（比如你跑出 long=1500 short=800 这种），加上你自己的"为什么"判断。

## 5. Q6：中性化（"是什么 / 为什么 / 怎么算"三段式）

> **题面**：请查阅 Learn 页面的有关文档，结合文档谈谈您对中性化（neutralization）的理解及其计算过程。
> **必须改写提示**：教程说"必须落到具体公式"，但**不要**照抄 Learn 文档的英文原句。

### 5.1 智能纪要版"中性化"讲解（最通俗）

> 假设股市平均涨 3%，中性化就是把 3000 只股票的涨幅都减掉 3%——这样**大盘涨的贡献被剥离**，剩下的就是"哪些股票真的跑赢大盘、哪些在跌"。

### 5.2 三段式骨架

**（1）是什么**
- 中性化是 BRAIN 平台的一种**预处理算子**，作用是"剥离指定因子（市场/行业/子行业）对 alpha 信号的污染"。
- 它把 alpha 值按"中性化分组"做截面标准化，让组内所有股票的 alpha 均值为 0。

**（2）为什么**
- 如果不中性化，alpha 会被"市场涨跌"或"行业 β"主导，看不出选股能力。
- 例子：你做多新能源股票时赚钱，可能只是因为新能源板块整体在涨，不是你选股厉害。中性化把这部分"白赚"剥离掉。

**（3）怎么算（落到公式）**
- 设股票 i 在截面 t 的原始 alpha 为 `a_i`。
- 设分组为 G（可以是 MARKET / SECTOR / INDUSTRY / SUBINDUSTRY / CUSTOM）。
- 步骤 1：计算组内均值 `μ_g = mean(a_i | i ∈ g)`。
- 步骤 2：减去组均值 `a_i_neutral = a_i - μ_g`。
- 步骤 3（可选）：再除以组内标准差做标准化 `a_i_neutral = (a_i - μ_g) / σ_g`。
- BRAIN 上的 `group_neutralize(a, densify(group))` 算子，本质就是"按 group 分组做这个减均值"。

### 5.3 改写示范（乌龙风格的"套自己语言"）

> 中性化按我的理解就是"把大锅饭的影响剔掉"。比如大盘今天涨 3%，3000 只股票里有一半股票其实只是跟着大盘涨，自身没 alpha。中性化的过程就是算出每只股票相对它所在行业或子行业的偏离值——大盘涨 3%、该行业涨 4% 的股票，做完中性化后它的 alpha 变成 +1%，而不是 +4%。这样我们最后选股时挑出来的就是那些"在同行里跑得比同行好"的，而不是"行业本身在风口"的。
>
> 计算上，BRAIN 的 `group_neutralize` 算子按 Learn 文档描述，是先把数据按 `densify(group)` 字段分组（比如 subindustry），然后在每个组内做**减均值**操作，必要时再除以组内标准差做归一化。我自己在跑 anl4 alpha 时就常用 `group_neutralize(..., densify(subindustry))` 来剔除"行业景气度"的干扰。

> **怎么改**：把"大锅饭"换成你自己的比喻（"班级平均分"、"团队 KPI 拉平"都行），加一句你跑 anl4 时中性化前后的指标对比。

## 6. 改写 3 板斧（快速去 AI 味）

### 6.1 板斧 1 · "加一句我自己"
- 在每个 AI 给的答案里**加 1-2 句**你自己的真实体验。
- 模板："我自己在跑 [你的字段] 时发现 [你的观察]..."
- 效果：直接和别人拉开 80% 距离，AI 检测也认不出。

### 6.2 板斧 2 · "换比喻"
- AI 给的比喻通常是"教学风"（如"想象一张成绩单"）。
- 你换成**你熟悉的生活场景**（打游戏、做菜、网购、追剧都行）。
- 模板："这其实就像 [你熟悉的事]，[具体对应]..."

### 6.3 板斧 3 · "调语序"
- AI 喜欢"先概念后举例"或"先定义后展开"。
- 你改成"先抛一个数字 / 一个反问 / 一个你撞过的坑"，再回到概念。
- 模板："[我先说一个我撞过的坑]，[再回头讲 X 是什么]..."
