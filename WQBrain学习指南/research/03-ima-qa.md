### 2.1 Q4：Matrix Data vs Vector Data

#### 生活类比

| 维度 | Matrix Data（矩阵型） | Vector Data（向量型） |
|---|---|---|
| **直觉** | 每天一张成绩单，每人有 1 个分数 | 每天的购物小票，每人有 N 张 |
| **形状** | 表格 = 行（股票）× 列（日期），每格 1 个值 | 表格 + 每个格子里是一个"小数组" |
| **典型数据** | 财务数据（assets/liabilities）、价格（close/open）、基本面（PE/PB） | 新闻、分析师评级、社交媒体、逐笔成交 |
| **为什么需要区分** | 平台大部分算子都只认"每只股票每天 1 个值" | 平台算子不能直接吃"小数组" |

#### 平台铁律

- **Alpha 表达式最终输出必须是 Matrix**（每只股票每天 1 个值 = 持仓权重）
- Vector 数据要变成 Matrix → 用 `vec_` 开头算子做"聚合"

#### `vec_` 算子的作用（举例）

| 算子 | 做什么 | 适用场景 |
|---|---|---|
| `vec_mean(x)` | 把 N 个值取平均 | 每日平均情绪分 |
| `vec_sum(x)` | 求和 | 每日总新闻条数 |
| `vec_max(x)` | 取最大值 | 每日最强情绪 |
| `vec_count(x)` | 计数 | 每日事件数量 |
| `vec_std(x)` | 标准差 | 每日波动 |

#### 答题原理（不要直接抄）

1. **核心区分** = "数据规不规整"（每只股票每天是 1 个值还是 N 个值）
2. **本质原因** = Alpha 输出必须是 Matrix，所以 Vector 要先"压扁"
3. **正确写法流程** = 先用 `vec_*` 聚合 → 再用普通算子（rank/中性化/时序）
4. **直接用 Vector 数据报错** = 平台算子只吃 Matrix

#### 常见误区

- ❌ 误以为"Vector = 数学向量"（一个有序数字序列）—— 在 BRAIN 里它特指**事件型非规整数据**
- ❌ 把"每日多新闻取平均"写成 `mean(news)` —— 必须用 `vec_mean(news)`
- ❌ 漏掉 `vec_*` 步骤直接接 `rank()` / `group_neutralize()` —— 平台会报算子不匹配

### 2.2 Q5：Long/Short Count 是什么，怎么用

#### 生活类比

> 把全班同学的成绩排名：
> - **做多清单（Long Count）** = 前 10 名（你认为他们会跑赢大盘）
> - **做空清单（Short Count）** = 后 10 名（你认为他们会跑输大盘）
> - **覆盖率 = 20 人 / 50 人 = 40%**

#### 平台定义

| 术语 | 含义 | 公式直觉 |
|---|---|---|
| **Long Count** | Alpha 信号建议"做多"的股票数 | 同一日期、alpha 值为正且排在前面的股票数量 |
| **Short Count** | Alpha 信号建议"做空"的股票数 | 同一日期、alpha 值为负且排在后面的股票数量 |
| **Universe Size** | 当日股票池大小（Top3000 / Top2000） | 该 region 当天入选的股票总数 |

#### 数学原理（指示函数）

**Long Count** 的本质 = 累加"在某一方向上达到条件的次数"：

$$
L = \sum_{i} I(\alpha_i > 0) \quad \text{或} \quad L = \sum_{i} I(\text{rank}(\alpha_i) > 0.5)
$$

- $I(\cdot)$ = 指示函数，条件真返回 1，假返回 0
- 通俗：数一数"多少只股票被你的 alpha 建议做多"

**Short Count** 同理，只是条件反过来。

#### 实战用法 1：估算"市场覆盖率"

$$
\text{覆盖率} = \frac{L + S}{\text{Universe Size}}
$$

举例：Top3000 股票池，Long = 1428，Short = 1397，覆盖率 ≈ 0.94（很健康）。

#### 实战用法 2：探查数据字段的统计特征

- **覆盖率低**（如 30%）→ 数据字段覆盖不全 → 用 `group_mean` / `group_backfill` 填补
- **Long/Short 严重失衡**（如 99%/1%）→ 可能是数据有偏、单边策略
- **随时间变化的覆盖率**→ 反映数据更新频率（季度/月度/周度）

> 💡 **不同数据更新频率的影响**：
> - 季度数据（66 交易日）→ Long + Short 接近真实覆盖率
> - 月度数据（22 交易日）→ 覆盖率约为 1/3
> - 周度数据（5 交易日）→ 覆盖率更低

#### 常见误区

- ❌ 把 "Long Count" 误以为"做多赚的钱"——它只是"数量"，不是"金额"
- ❌ 只算 Long 不算 Short —— 真实策略是 long-short，**两个都要看**
- ❌ 用 Long Count 单独评估 alpha 质量 —— 还要看 Sharpe/Fitness/自相关
- ❌ 覆盖率 ≠ 信号质量（覆盖 100% 并不代表 alpha 强）

#### 答题原理（不抄答案）

1. **数学本质** = 指示函数求和（统计"满足条件的次数"）
2. **平台作用** = 估算覆盖率（市场覆盖比例）
3. **统计特征** = 反映数据频率 + 字段健康度 + 策略偏向
4. **必须结合** = Long + Short 一起看，与 Universe Size 配合

### 2.3 Q6：Neutralization（中性化）

#### 生活类比

> 班级数学考试：3 班平均 95，5 班平均 60
> - 你想知道"小张在班里数学排第几"——这就要**把"班级"这个干扰因素剔除掉**
> - 在 3 班排第 10 不算学霸，但在 5 班排第 10 就是学霸
> - **中性化** = 把所有同学放到同一标准下比
> - 在金融中，"班"可以是行业（科技股 vs 银行股） / 子行业 / 整个市场

#### 平台上的两种形式

**A. 在 Settings 面板里设置（点选）**

- Neutralization = None / Market / Sector / Industry / Subindustry
- 配 Decay（衰减）= 0
- 配 Truncation（截断）= 0
- 平台会自动执行中性化

**B. 在表达式里手写**

```python
group_neutralize(your_alpha, densify(subindustry))
```

- 第一个参数：要中性化的信号
- 第二个参数：用哪个"分组维度"（先 `densify` 把它变成完整的股票×日期矩阵）

#### 计算过程（数学本质）

1. 把所有股票按"行业"分组
2. 在**每组内**做这件事：

$$
\alpha_i^{\text{neutralized}} = \alpha_i - \overline{\alpha_{\text{group}}}
$$

3. 也就是**每组的均值归零**（不是全市场归零）

#### 4 种中性化层级的对比

| 层级 | 粒度 | 剔除什么 | 适用 |
|---|---|---|---|
| **Market** | 全市场归一 | 整个大盘 β | 高换手 + 不想赌方向 |
| **Sector** | 大行业 | 行业轮动 | 跨行业横截面 |
| **Industry** | 细分行业 | 细分行业 β | **多数情况下最优** |
| **Subindustry** | 子行业 | 子行业 β | 样本量大时用 |

#### 中性化的 3 大目的

1. **降风险**：避免"我买的全是科技股"导致行业 β 风险集中
2. **多空平衡**：让 long 端和 short 端的"组内头寸"加起来为 0
3. **提纯信号**：让 alpha 只反映"个股选择能力"，不反映"行业偏好"

#### 与"正交化"的区别（容易混淆）

- **中性化（Neutralization）** = 在分组内减均值 → 行业 β 归零
- **正交化（Orthogonalization）** = 数学上把两个向量的相关性变 0 → 完全独立
- 中性化是一种"近似正交化"的方式

#### 常见误区

- ❌ **做完中性化就完全没风险了** —— 还有市场 β 风险（如果是 industry 中性化）
- ❌ **中性化层级越细越好** —— Subindustry 可能样本量过小，反而过拟合
- ❌ **中性化 = 多空对冲** —— 中性化是统计操作，多空对冲是组合操作，两者不同
- ❌ **中性化后没做 group_count 检查** —— 可能因为某行业样本太少而中性化失败

#### 答题原理（不抄答案）

1. **核心目的** = 剔除行业 / 市场 β 风险，让 alpha 更"纯"
2. **数学本质** = 每组内减去组均值（中心化）
3. **平台两种用法** = Settings 选 + 表达式写 `group_neutralize`
4. **必须配合** = `densify` 把分组字段稠密化

### 2.4 Q8：如何通过 BRAIN 平台网页获取 Alpha ID

#### Alpha ID 是什么

- 平台给**每一条 alpha 唯一编号**（字符串 / 数字）
- 提交后的 alpha 才有"正式 ID"
- 未提交的草稿 = 临时 ID（也可能没有 ID）

#### 获取路径（按用户操作流程）

```
路径 A：模拟完成后立即获取
   1. 在 Alpha 编辑器写完表达式 → 点 Simulate
   2. 模拟结果页面 → 顶部 / URL / 详情面板 都能看到 ID
   3. 部分版本在 "Save" 后才生成 ID

路径 B：查询历史 alpha
   1. 登录 platform.worldquantbrain.com
   2. 顶部菜单 → "Alphas" 或 "My Alphas"
   3. 在列表里能看到每个 alpha 的 ID、状态、Sharpe
   4. 点击进入详情 → 顶部 URL 里通常就有 alpha_id 参数

路径 C：已提交 alpha
   1. 菜单 → "Activity" 或 "Submitted Alphas"
   2. 列表里每条都有 ID
```

#### API 方式（如果会用 Python）

```python
# 用 BRAIN 的 API（顾问权限）
from wqb import WQBSession
wqbs = WQBSession(('<邮箱>', '<密码>'))
response = wqbs.locate_alpha('<alpha_id>')  # 用 ID 查详情
```

#### Alpha ID 的用途

1. 复现：拿到 ID 后用 `locate_alpha` 可以拉回原表达式
2. 引用：报告/论文里引用"我用了 WQ BRAIN alpha #xxxxx"
3. 比较：和别人分享 alpha 思路（脱敏后）
4. 答题：你提交问卷 Q8 时如果需要举例，可以用自己刚提交的一条

#### 常见误区

- ❌ **以为模拟失败也有 ID** —— 通常只有 Save 或 Submit 后才有
- ❌ **以为 ID 是表达式 hash** —— 平台 ID 是内部生成，与表达式内容无关
- ❌ **以为 ID 永久不变** —— 平台版本迭代可能改 ID 格式
- ❌ **找不到 ID 时乱编** —— 直接用自己最近提交的 1-2 条即可

#### 答题原理（不抄答案）

1. **核心是讲清楚流程**：从哪里点进去 → 看到什么
2. **关键操作**：模拟 → 保存 / 提交 → 列表查询
3. **可用辅助**：URL 链接、API、复现查找
4. **别瞎编**：可以拿自己真实提交过的一条举例

## 3. 社区常见误区（必看）

### 3.1 概念层面：最容易混淆的 6 组术语

| # | A | B | 区别 | 金融小白记法 |
|---|---|---|---|---|
| 1 | **Vector Data** | **Vector（数学向量）** | 前者是"事件型非规整数据"，后者是"一维数组" | 别被名字骗了，在 BRAIN 里"Vector" = 事件数据 |
| 2 | **Long Count** | **Long Position** | 前者是"做多股票数量"，后者是"做多头寸（金额）" | 一个是"几只股"，一个是"多少钱" |
| 3 | **中性化** | **对冲** | 前者是统计操作（减均值），后者是组合操作（买/卖相反） | 中性化=改数据；对冲=改头寸 |
| 4 | **Sharpe** | **Fitness** | Sharpe=风险调整收益；Fitness=综合性价比 | Sharpe 是"性价比"的"性能"，Fitness 加了"持久性" |
| 5 | **Self-Correlation** | **Production Correlation** | 前者是你提交过的 alpha 之间的相似度，后者是平台"实盘"生产环境的相似度 | 一个查"你自己"，一个查"平台" |
| 6 | **Quarterly Payment** | **Base Payment** | 前者按季度评估（OS 表现），后者按日计算（提交质量） | 一个是"季度考"，一个是"日结工资" |

### 3.2 题目层面：9 道题里最容易答错 / 答偏的

> 来自社区反复总结

| 题目 | 常见错误 | 正确思路 |
|---|---|---|
| **Q4** | 把 Vector 当成"数学向量"，举例跑偏 | 必须落到"事件型非规整数据"+ `vec_*` 算子 |
| **Q5** | 只解释 Long 不解释 Short / 不会给具体案例 | 必须两个都讲 + 给数学公式 + 配合覆盖率 |
| **Q6** | 写成"中性化就是多空对冲" | 中性化是**统计操作**（减均值），不是组合操作 |
| **Q7** | 选一个 example 直接背模板，答不出"自己的改进" | 必须讲"我做了什么改进"+"为什么这样改" |
| **Q8** | 把"Alpha ID"和"User ID"搞混 | Alpha ID 是**单条 alpha** 的 ID，不是你账号的 ID |
| **Q9** | 漏掉 Bronze/Silver/Gold 这 3 个等级 / 漏掉"季度评估"机制 | 必须答全 6 个 + 强调"后 3 级靠季度评估" |

### 3.3 平台机制层面：90% 的人会踩的 5 个坑

1. **【最致命】抄别人答案**
   - 9 道题**仅 2 次答题机会**
   - 平台有**反作弊雷同卷检测**（"主客观题纯文字相似度+语义相似度"算法）
   - 2 次都判错 = **账号与顾问资格脱钩**
   - ⭐ 即使答得"差不多"也容易被判雷同，必须**用自己的话 + 自己的理解**写

2. **【最常见】直接用 Vector 数据**
   - 不加 `vec_*` 算子直接接 `rank()` / `group_neutralize()` → 报算子不匹配
   - 应该先聚合再处理

3. **【最隐蔽】中性化层级选错**
   - 选 Market 反而把信号抹平（=做了全市场减均值）
   - 选 Subindustry 可能样本量过小过拟合
   - **多数场景 Industry 是最优**

4. **【最容易忘记】Base 和 Quarterly 是两套钱**
   - 顾问日结有 1-120 USD（按提交质量）
   - 季度奖 = 0 是常态
   - 别把"季度奖"想成"日薪 × 90 天"

5. **【最易混】"做高分"和"赚钱"是两件事**
   - Bronze → Gold 靠**拼分**（不看 alpha 质量）
   - 顾问后**靠质量**（VF 评分、OS 表现、Weight 累积）
   - 很多人 Gold 后就不知道怎么提升 = "能力陷阱"

### 3.4 "标准答案"为什么不能抄

| 检测维度 | 算法 | 后果 |
|---|---|---|
| 纯文字相似度 | 字符串匹配 / 编辑距离 | 直接标红 |
| 语义相似度 | NLP 嵌入向量 | 换说法也抓得到 |
| 单题相似度阈值 | 通常 0.7+ 即触发 | 触发后整体卷相似度加权重 |
| 答题时间 | 过快（< 3 分钟答完 9 题）= 可疑 | 时间戳也会被记录 |
| IP / 设备指纹 | 同 IP 多个账号 = 同批次 | 高风险标记 |

> ⭐ **正确答题姿态**：
> 1. **先理解** → 用自己的话写（哪怕不漂亮）
> 2. **举自己的例子** → 不要照搬社区案例
> 3. **答 30-60 分钟** → 不要 5 分钟交卷
> 4. **遇到不会的** → 用 IMA 知识库问清楚再写
> 5. **检查 1 遍** → 确认没有 Ctrl+C / Ctrl+V 痕迹

## 4. 答题时的"心法"建议

> 来自社区 + 龙桑知识库反复强调的原则

### 4.1 答题心法 4 句

1. **理解 > 模板**：先搞懂概念，再用自己的话写
2. **原理 > 公式**：公式只是辅助，重点是讲清"为什么"
3. **真实 > 完美**：举自己用过的真实例子，比"凑一个完美答案"更安全
4. **反思 > 复述**：题里如果问"你的理解" / "你的改进"，必须带"我"字

### 4.2 Q7（Alpha Example 复现题）的特殊提醒

Q7 是 9 道题里**最危险**的——因为它要你"复现 + 改进"，非常容易抄原 example。

> **答好 Q7 的姿势**：
> 1. 选一个**简单的 example**（不要挑复杂的）
> 2. 先**完整跑通**它（不改任何东西）
> 3. 记录 Sharpe / Fitness / Self-Corr
> 4. 改造 1-2 处（换数据集 / 换中性化 / 改参数）
> 5. 对比改进前后的指标
> 6. 用**自己的话**写"为什么这样改"（不要背别人的改进思路）

### 4.3 答题时间与机会

- 9 道题 = 2 次机会
- 第 1 次答错 → 平台给反馈 → 第 2 次补救
- 第 2 次再错 → 账号锁定 = 与顾问资格失之交臂
- 建议**不要急着交**——准备好再交（一般 2-7 天都可，看平台规则）
