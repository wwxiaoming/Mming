## 4. Q4：Matrix Data vs Vector Data（核心题 ⭐⭐⭐）

> **本题是 9 道题里最"看起来简单、做起来错"的题**。答错率社区估计 50%+。

### 4.1 考点解析

**题目原文**："What is the difference between Matrix Data and Vector Data?"

**考点拆解**：
1. **概念定义**——什么是 Matrix Data、什么是 Vector Data
2. **金融含义差异**——不只是数据结构差异，更要讲"在金融上意味着什么"
3. **"Vector"在 BRAIN 中的双关陷阱**——必须区分"数据结构 Vector"和"数学 Vector"

### 4.2 核心概念（**只讲原理，不给可抄答案**）

#### 4.2.1 Matrix Data 的精确定义

**定义**：在 BRAIN 平台中，**Matrix Data 是"每天每只股票只有 1 个数"的数据**。

**结构想象**：
- 想象一个 Excel 表格，行 = 股票，列 = 日期
- 每个单元格（行 × 列）里填 1 个数字
- 这就是 Matrix Data

**BRAIN 平台典型 Matrix 字段**：

| 字段名 | 含义 | 数据来源 |
|---|---|---|
| `close` | 收盘价 | 行情数据 |
| `volume` | 成交量 | 行情数据 |
| `returns` | 日收益率 | 计算字段 |
| `anl4_adjusted_netincome_ft` | 分析师调整后净利润预测 | 基本面数据 |
| `assets` / `liabilities` | 总资产 / 总负债 | 基本面数据 |

**金融小白理解**：Matrix Data = 每天一张"成绩单"，每只股票每天 1 个分数。

#### 4.2.2 Vector Data 的精确定义

**定义**：在 BRAIN 平台中，**Vector Data 是"每天每只股票有 N 个数"的数据**（N 可为 0、1、5、20、几十甚至上百）。

**结构想象**：
- 还是 Excel 表格
- 但每个单元格不是 1 个数字，而是一个**小数组**（多个数）

**BRAIN 平台典型 Vector 字段**：

| 字段名 | 含义 | N 的来源 |
|---|---|---|
| `nws12_afterhsz_sl` | 盘后新闻多空信号 | 每天 N 条新闻，每条带 1 个信号 |
| `scl12_buzz` | 社交媒体热度 | 每天 N 条帖子，每条带 1 个情绪分 |
| `analyst_rating_events` | 分析师评级事件 | 每天 N 次评级，每次带评级变动 |
| `optiondata` | 期权数据 | 每天 N 笔成交，每笔含 IV、希腊字母等 |

**金融小白理解**：Vector Data = 每天的"购物小票"，每只股票每天可能有好几张小票，每张记录一个事件。

#### 4.2.3 金融含义差异（核心！）

| 维度 | Matrix Data | Vector Data |
|---|---|---|
| **数据来源** | 标准化、低频、结构化 | 事件型、高频、非结构化 |
| **更新频率** | 日/周/季度 | 日内多次（每条新闻/事件都更新） |
| **金融含义** | 反映"已发生的稳定状态" | 反映"市场情绪/事件冲击的瞬时分布" |
| **典型场景** | 估值、基本面、技术面因子 | 舆情、事件驱动、订单流因子 |
| **N 的大小** | 永远是 1 | 0 ~ 几十（甚至上百） |

**为什么这个差异对做 alpha 关键？**
- Matrix 字段可以直接被 `rank`、`ts_mean`、`sector_neutral` 等算子处理——输入是规整的
- Vector 字段**不能**直接进入这些算子（否则会因为"不知道 N 是几"而报错）
- 平台提供专门的 `vec_` 开头算子（`vec_mean`、`vec_count`、`vec_max`、`vec_std_dev` 等）——它们的作用是**把"每天 N 个值"压成"每天 1 个值"**，再喂给矩阵算子

#### 4.2.4 为什么中性化是"向量操作"（陷阱题点）

**"向量"在 BRAIN 中有双重含义**：

1. **数据结构层面**：Vector Data = 不定长事件数据（4.2.2 讲的）
2. **数学/线性代数层面**：**向量 = 一列数**（一维数组）

**BRAIN 里的中性化算子**（`market_neutral`、`sector_neutral`、`industry_neutral`、`subindustry_neutral`）数学上做的是：

```
neutralized_alpha = (alpha - group_mean) / group_std
```

这本质上是在**对一组数（向量）**做"减均值、除标准差"——即数学意义上的"向量操作"。

**所以回答 Q4 的正确角度**：
- 如果题目问"中性化是矩阵还是向量操作"——这里的"向量"指的是**数学向量**（一列数）
- 中性化对**一组 alpha 权重**（即一个向量）做加减乘除
- **不是**在"Vector Data"这种"每天 N 个事件"的数据结构上操作

### 4.3 自己作答的思路（**段式模板，不给完整答案**）

**推荐 3 段式**：

| 段 | 写什么 | 长度建议 |
|---|---|---|
| **段 1：定义** | 用自己的话定义 Matrix Data 和 Vector Data | 2-3 句 |
| **段 2：举例** | 用 2-3 个 BRAIN 平台的具体字段举例 | 3-4 句 |
| **段 3：差异的金融意义** | 解释为什么"每天 1 个数"和"每天 N 个数"对做 alpha 重要 | 3-4 句 |
| **（可选）段 4：陷阱澄清** | 区分"数据结构 Vector"与"数学 Vector" | 1-2 句 |

### 4.4 避坑提醒

- ❌ **不要混淆"数据结构 Vector"和"数学向量"**——这是 50% 翻车点
- ❌ **不要只写定义不举例**——必须用 BRAIN 平台的具体字段名（`close`、`nws12` 等）
- ❌ **不要把"Vector = 数学向量 = 一列数"**——在 BRAIN 平台里 Vector 特指事件型非规整数据
- ❌ **不要漏掉"必须用 `vec_*` 算子做聚合"**这一关键操作
- ✅ **表达要"像你理解的"而不是"像百科的"**——用自己的话、用真实举例

## 5. Q5：Long Count / Short Count（核心题 ⭐⭐⭐）

> **本题是 9 道题里"看起来最简单、答起来最容易答不到位"的题**。
> 题目要求三件事：(1) 定义 (2) 具体案例 (3) 数学原理——**缺一不可**。

### 5.1 考点解析

**题目原文**："什么是 long count？什么是 short count？如何使用 long count 和 short count 的信息探查数据字段的统计特征？请给出具体案例并阐述其数学原理？"

**考点拆解**：
1. **定义**——Long Count / Short Count 是什么
2. **具体案例**——必须给一个 BRAIN 平台的具体字段例子
3. **数学原理**——必须写公式（不是泛泛而谈）
4. **应用场景**——"探查数据字段的统计特征"具体怎么做

### 5.2 核心概念（**只讲原理**）

#### 5.2.1 Long Count 的精确定义

**Long Count**：在某一时刻（截面）或某段时间（时序），**alpha 值为正（>0）的股票数量**。这些股票会被建**多头**头寸。

#### 5.2.2 Short Count 的精确定义

**Short Count**：在某一时刻（截面）或某段时间（时序），**alpha 值为负（<0）的股票数量**。这些股票会被建**空头**头寸。

**金融小白理解**：
- 你的 alpha 表达式给每只股票输出一个数（正或负）
- 正的归入 long（买入）
- 负的归入 short（卖出）
- long/short count 就是当天买入/卖出的股票各有几只

#### 5.2.3 ⚠️ 关键雷区：区分"数量"与"金额"

| 术语 | 含义 | 金融小白记法 |
|---|---|---|
| **Long Count** | alpha 信号为正的**股票数量** | "几只股" |
| **Long Position** | 做多的**头寸金额** | "多少钱" |
| **Short Count** | alpha 信号为负的**股票数量** | "几只股" |
| **Short Position** | 做空的**头寸金额** | "多少钱" |

**Q5 问的是 Count（数量），不是 Position（金额）**。

#### 5.2.4 两种观察视角

1. **横截面视角**（某一天）：看当天所有股票，alpha>0 的有 N₁ 只，alpha<0 的有 N₂ 只
2. **时序视角**（一段时间）：看每天的 long count，画出趋势图——反映"alpha 信号的偏向性是否稳定"

#### 5.2.5 为什么 long/short count 重要？

- 它是 alpha "**多空对称性**"的最直观指标
- 理想情况下，一个 market neutral 的 alpha 应该 long count ≈ short count
- 极端不平衡（如 10:1）通常意味着 alpha 选股逻辑有偏，或者 universe 选取有问题

### 5.3 具体案例：如何用 long/short count 探查 `anl4_adjusted_netincome_ft` 字段

> ⚠️ **本题要求"具体案例"，必须用 BRAIN 平台真实字段**。
> 这里用 `anl4_adjusted_netincome_ft`（分析师调整后净利润预测）作为示范案例，**但答题时建议你换成你真正在 BRAIN 上跑过的字段**。

**场景**：你想用 `anl4_adjusted_netincome_ft` 构建一个 alpha，但你不确定这个字段的横截面分布是否对称。

**操作步骤（自己写答案时的素材）**：

**第一步**：写一个"诊断 alpha"（不用于正式提交，只用于探查）：

```
signal = anl4_adjusted_netincome_ft
```

**第二步**：提交 simulation，settings 选择：
- Region: USA
- Universe: TOP3000
- Delay: 1
- **Neutralization: NONE**（关键！不然中性化会把分布强行改对称，探查不到原貌）

**第三步**：回测结果 Summary 面板找：
- **Long Count**（通常在 PnL 摘要可查）
- **Short Count**
- 把两者画成时序图，观察 5 年回测期内每天的多空比

**第四步**：解读：
- **如果 long:short ≈ 1:1** → 字段本身在 universe 内分布对称，可以直接用
- **如果 long:short ≈ 7:3** → 字段本身有"长尾正向"——少数公司利润极好拉高均值 → 建议先 `rank()` 或 `winsorize()` 再用
- **如果长期 long >> short** → 字段在 universe 里有结构性偏向（如大部分公司"调整后净利润"为正） → 需要做横截面归一化（`scale`、`zscore`），否则 alpha 会"系统性偏多"

### 5.4 数学原理（**必须写公式**）

**直观逻辑**：
- 横截面 alpha 值是连续数值 → 正负号把样本切两半
- 如果数据是**对称分布**（如 t 分布）→ 正负各占约 50% → long count ≈ short count
- 如果数据是**右偏分布**（如财务数据常见：左边有亏损公司但亏损幅度有限，右边有少数巨无霸利润极高）→ 大部分样本 > 0，少数 < 0 → long count >> short count

**数学公式**：

给定 N 只股票，alpha 值集合为 A = {a₁, a₂, ..., a_N}：

$$
\text{Long Count} = \sum_{i=1}^{N} I(a_i > 0)
$$

$$
\text{Short Count} = \sum_{i=1}^{N} I(a_i < 0)
$$

其中 $I(\cdot)$ 是**指示函数**（条件真返回 1，假返回 0）。

**进阶探查指标**（写进答案会更出彩）：

$$
\text{Pos Ratio} = \frac{\text{Long Count}}{\text{Long Count} + \text{Short Count}}
$$

- 0.5 = 完美对称
- > 0.6 = 偏正
- < 0.4 = 偏负

**多空绝对值之比** = $\frac{\sum_{a_i>0} |a_i|}{\sum_{a_i<0} |a_i|}$ → 这个比 Pos Ratio 更能反映"非对称暴露"

### 5.5 手算示例（**自己写答案时建议带一个**）

**场景**：5 只股票在某一天的 alpha 值（未中性化）：

| 股票 | alpha 值 | 符号 |
|---|---|---|
| AAPL | +0.8 | + |
| MSFT | +0.3 | + |
| GOOG | +0.1 | + |
| TSLA | -0.4 | - |
| NVDA | -0.2 | - |

**计算**：
- Long Count = 3（AAPL, MSFT, GOOG）
- Short Count = 2（TSLA, NVDA）
- Pos Ratio = 3/5 = 0.6
- 多空绝对值之和 = (0.8+0.3+0.1) : (0.4+0.2) = 1.2 : 0.6 = 2:1

**解读**：这个 alpha 当天偏多（3 只多头 vs 2 只空头），且多头暴露的多空绝对值（1.2）是空头（0.6）的 2 倍——说明"看好 AAPL、MSFT"的力量比"看空 TSLA"更强。

## 6. Q6：Neutralization（核心题 ⭐⭐⭐）

> **本题占分最大**。题目要求结合 Learn 文档，意味着"看文档原文"是必要条件。

### 6.1 考点解析

**题目原文**："请查阅 Learn 页面的有关文档，结合文档谈谈您对中性化（neutralization）的理解及其计算过程。"

**考点拆解**：
1. **目的**——为什么要中性化
2. **定义**——中性化是什么
3. **计算过程**——必须配公式
4. **4 个层级**——Market / Sector / Industry / Subindustry 的差异
5. **结合 Learn 文档**——不是闭门造车

### 6.2 核心概念（**只讲原理**）

#### 6.2.1 目的：消除市场/行业 β

**什么是 β？** 在金融里，β（beta）是一只股票相对于"某个参照系"（市场、行业、子行业）的敏感度。
- 市场 β = 1 意味着"跟大盘同涨同跌"
- 行业 β = 1 意味着"跟所在行业同涨同跌"

**为什么要消除 β？**

1. **风险隔离**：你只想赚"选股 alpha"，不想赌大盘方向。如果你买的是一堆科技股，组合可能"看起来 alpha 很高"，其实是搭了科技板块的顺风车
2. **可比性**：A 行业选出的"前 10 名"应该和 B 行业选出的"前 10 名"标准一致。中性化后，跨行业比较才有意义
3. **资金容量**：如果 alpha 隐含了"满仓某行业"的头寸，资金一大就推不动价格。中性化后头寸分散，能容纳更多资金

#### 6.2.2 完整计算过程（**必须写公式**）

**标准化 3 步法**（BRAIN 内部中性化的核心，**注意 BRAIN 用的是 z-score 标准化**）：

```
对每个组（市场/行业/子行业/子子行业）独立执行：

Step 1: 计算组内均值
    group_mean = mean(alpha_values_in_group)

Step 2: 计算组内标准差
    group_std = std(alpha_values_in_group)

Step 3: 逐元素 z-score 标准化
    neutralized_alpha = (alpha - group_mean) / group_std
```

**再对结果做"总和归一"**（保证多空市值相等）：

```
final_alpha = neutralized_alpha / sum(|neutralized_alpha|)
```

**最终 alpha 满足**：
- 组内均值为 0（多空市值相等）
- 总和为 1（总仓位可控）
- 每只股票的权重正比于其"组内相对强弱"

#### 6.2.3 4 个层级的差异（**必考**）

| 层级 | Group Field | 含义 | 大致分组数（USA） | 适用场景 |
|---|---|---|---|---|
| **Market** | `market` | 整个市场 | 1 | 最粗，强制多空市值平衡 |
| **Sector** | `sector` | 大行业（GICS Sector） | ~10 | 板块中性，捕捉行业内选股 alpha |
| **Industry** | `industry` | 中行业（GICS Industry） | ~60 | 更细粒度 |
| **Subindustry** | `subindustry` | 细行业（GICS Sub-Industry） | ~150 | 最细，适合窄行业套利 |

**选择建议**：
- 做大盘/全市场型 alpha（选 TOP3000 全部股票）→ 用 **Market**
- 做板块轮动、跨行业选股 → 用 **Sector**
- 聚焦特定行业内（如半导体 vs 软件）→ 用 **Industry** 或 **Subindustry**
- 越细的层级 = 越窄的押注 = 越像"行业内部套利"——可能夏普高，但容量小、相关性集中

### 6.3 手算示例：5 只股票 + Sector 中性化

> **答题时强烈建议带一个手算例子**——会让答案立刻"真实"起来。

**场景**：3 个行业（Sector），5 只股票，原始 alpha 值：

| 股票 | Sector | 原始 alpha |
|---|---|---|
| AAPL | Tech | +2.0 |
| MSFT | Tech | +0.5 |
| JPM | Finance | -1.0 |
| GS | Finance | -0.3 |
| XOM | Energy | +0.4 |

**计算过程**：

**Tech 组**（AAPL, MSFT）：
- mean = (2.0 + 0.5) / 2 = **1.25**
- std = sqrt(((2.0-1.25)² + (0.5-1.25)²)/2) = sqrt(0.5625) = **0.75**
- AAPL 中性化后 = (2.0 - 1.25) / 0.75 = **+1.0**
- MSFT 中性化后 = (0.5 - 1.25) / 0.75 = **-1.0**

**Finance 组**（JPM, GS）：
- mean = (-1.0 + -0.3) / 2 = **-0.65**
- std = sqrt((0.1225+0.1225)/2) ≈ **0.35**
- JPM 中性化后 = (-1.0 - (-0.65)) / 0.35 = **-1.0**
- GS 中性化后 = (-0.3 - (-0.65)) / 0.35 = **+1.0**

**Energy 组**（XOM 单只）：
- 单只股票 std = 0（除以 0 出错）→ BRAIN 实际会做"分母下限保护"或把这只股票置为 0
- 简化处理：XOM 中性化后 = **0**

**最终结果**：

| 股票 | 原始 | 中性化后 | 多空 |
|---|---|---|---|
| AAPL | +2.0 | +1.0 | 多 |
| MSFT | +0.5 | -1.0 | 空 |
| JPM | -1.0 | -1.0 | 空 |
| GS | -0.3 | +1.0 | 多 |
| XOM | +0.4 | 0 | - |

**观察**：
- 原始 AAPL 看起来"最被看好"，但中性化后它在 Tech 组内只是"略微好于 MSFT"
- 中性化后每个组（Tech、Finance）都是"1 多 1 空"，符合"组内多空平衡"
- XOM 因为组内只有自己，标准差为 0，被"中性化掉"了——这是单只股票行业时的常见现象

### 6.5 避坑提醒

- ❌ **必须写公式**——只写"减去均值"不够，要写出 z-score 的完整三步
- ❌ **必须讲 4 个层级**——只讲 1 个会显得"理解片面"
- ❌ **不要把"中性化"答成"标准化"**——这是 80% 翻车点
  - **标准化（scale/zscore）**：字段内部去量纲，让不同股票可比
  - **中性化（neutralize）**：剥离特定因子影响（如行业 β）

### 6.6 进阶：与"正交化"的区别（可选，写了加分）

- **中性化（Neutralization）** = 在分组内减均值 → 行业 β 归零
- **正交化（Orthogonalization）** = 数学上把两个向量的相关性变 0 → 完全独立
- 中性化是一种"近似正交化"的方式

- 公开社区数据：单日津贴上限 $120/天，季度奖金上限 $25,000

### 9.4 顾问阶段与之前阶段的奖金机制差异

#### Bronze / Silver / Gold 阶段
- **没有现金奖励**——纯积分 + 升级解锁功能
- 解锁的功能包括：更多数据集、更多 region、更长 simulation 周期、可视化、Python API、SuperAlpha 等
- 适合"学习 + 攒作品集 + 等顾问邀请"

#### Consultant 阶段（关键转折点）
- **Base Payment（日津贴）**：按日发放，**$1 ~ $120/天**
  - 来源：每提交一个 alpha 都可以累积"质量分"
  - 上限：alphas 部分 $60/天 + super alphas 部分 $60/天 = 总 $120/天
  - 决定因素：提交数量、提交质量、self-growth、Value Factor（越接近 1 越好）
  - 主题加成：当前活跃的 Dataset/Region/SuperAlpha Theme 有倍数加成

- **Quarterly Payment（季度奖金）**：按季度发放，**$100 ~ $25,000**
  - 前提：上季度超过 20 天有提交
  - 决定因素：Weight（组合权重） × Out-of-Sample 表现
  - Weight 随时间积累（"老顾问比新顾问有更高权重"）
  - OS 表现与 Value Factor 直接相关

- **Competition 奖金**：参加 Super Alpha / ACE / Global Alphathon / Local Alphathon / IQC 等比赛，奖金另计

- **Referral 奖金**：$200 / 成功推荐一位新顾问（被推荐人需完成 10 天提交 + 保持顾问身份 1 个月+）

#### Expert / Master / Grandmaster 阶段
- **奖金机制相同**（Base + Quarterly + Competition + Referral）
- **区别在于**：
  - **季度奖金的"下限"被拉高**（Master ≥ $2,000，Grandmaster ≥ $8,000）
  - **能使用的"超级 alpha 池"更大**（更多高级数据集、专属比赛）
  - **能接触到的资源更专**（如参与 WorldQuant 内部研究讨论）
  - **Grandmaster 偶尔会收到全职 offer**——这是 BRAIN 平台转化为正式 quant 员工的主要通道

### 11.4 ⚠️ 怎么让自己的答案"不像 AI"

**这是 9 道题能不能通过的关键动作**——光有"内容正确"不够，还要"看起来真有人写过"。

- 偶尔漏字
- 用错标点（如中英标点混用）

#### 11.4.2 主动暴露 1 处"可控的逻辑跳跃"

- 如"我跑这个 example 时指标和官方不一样，可能是因为 universe 选取不同"

#### 11.4.3 混合用词风格

- 专业术语 + 偶尔的口语化（"大概 / 其实 / 我觉得 / 老实说"）

- 用"主题-例证-反思"或"问题-分析-方案"等不同结构

#### 11.4.5 避免一整段无换行

- 适当分点

#### 11.4.6 举自己真实跑过的例子

- 加上"我自己的理解是……" / "我的理解可能不全面，但……"

## 附录 B：常用数学公式速查

### B.1 z-score 标准化公式

$$
z = \frac{x - \mu}{\sigma}
$$

- $x$ = 原始值
- $\mu$ = 均值
- $\sigma$ = 标准差
- $z$ = 标准化后的值

### B.2 Long/Short Count 公式

$$
\text{Long Count} = \sum_{i=1}^{N} I(\alpha_i > 0)
$$

$$
\text{Short Count} = \sum_{i=1}^{N} I(\alpha_i < 0)
$$

$$
\text{Pos Ratio} = \frac{\text{Long Count}}{\text{Long Count} + \text{Short Count}}
$$

- $I(\cdot)$ = 指示函数（条件真返回 1，假返回 0）
- $N$ = 当日 universe 大小

### B.3 中性化公式（3 步法）

```
Step 1: group_mean = mean(alpha_values_in_group)
Step 2: group_std = std(alpha_values_in_group)
Step 3: neutralized_alpha = (alpha - group_mean) / group_std
```

最终归一：

$$
\text{final\_alpha} = \frac{\text{neutralized\_alpha}}{\sum |\text{neutralized\_alpha}|}
$$

### B.4 Self-Correlation（自相关）公式

$$
\text{Self-Corr} = \frac{\text{Cov}(\text{PnL}_1, \text{PnL}_2)}{\sigma_1 \cdot \sigma_2}
$$

- 两个 alpha 每日 PnL 序列的相关系数
- BRAIN 默认 4 年滚动窗口、阈值 0.6-0.7

### B.5 Sharpe 公式

$$
\text{Sharpe} = \frac{E[R_p] - R_f}{\sigma_p}
$$

- $E[R_p]$ = 组合平均收益
- $R_f$ = 无风险利率（实际中常忽略）
- $\sigma_p$ = 组合收益标准差

### B.6 Fitness 公式

BRAIN 平台官方公式（简化版）：

$$
\text{Fitness} = \text{Sharpe} \times \sqrt{|\text{Returns}| / \text{Turnover}}
$$

- 综合"风险调整收益"+"收益水平"+"换手率"三因素
- 是平台对 alpha "性价比" 的综合评分
