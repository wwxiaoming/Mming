# BRAIN 平台实操教程：从打开网页到跑出第一条 alpha

> **副标题**：金融小白也能看懂的 7 步上手指南
>
> **目标**：30 分钟内让你在 BRAIN 平台跑出第一条 alpha 并看懂结果
>
> **前提**：你已注册 BRAIN 账户、累计 10000 分金牌
>
> **贯穿示例**：用你熟悉的那条 anl4 alpha
> ```python
> group_neutralize(
>     ts_decay_linear(
>         ts_backfill(anl4_adjusted_netincome_ft, 120),
>         240
>     ),
>     densify(subindustry)
> )
> ```

## 第 1 步：在 Datasets 找数据（5 分钟）

### 1.1 进入 Datasets 页面
- 点击顶部 **`Datasets`** 标签

### 1.2 你会看到 4 大类数据集
| 类别 | 例子 | 含义 |
|---|---|---|
| **价格成交量 (pv)** | pv1, pv13 | 股票的开高低收、成交量、关系 |
| **基本面 (fundamental)** | fundamental2/6 | PE、PB、ROE 等财务数据 |
| **分析师预测 (analyst)** | analyst4 | 机构对 EPS、营收的预测 |
| **新闻舆情 (news)** | news12, news18 | 新闻情绪打分 |
| **期权 (option)** | option8, option9 | 隐含波动率 |
| **社媒情绪 (socialmedia)** | socialmedia8, 12 | 推特/微博情绪 |
| **模型 (model)** | model16, 51 | 风险模型、风格因子 |

### 1.3 怎么"看"一个数据集里有什么字段
- 点击任一数据集 ID（如 `analyst4`）
- 会打开**字段列表**（fields 表格）
- 每行有：`字段名`、`描述`、`类型`、`历史起始时间`

### 1.4 ⭐ 你的 anl4 alpha 用的字段是哪个
- 数据集：`analyst4`（分析师预测）
- 字段：`anl4_adjusted_netincome_ft`
  - 拆解：`anl4`（数据集）`_adjusted_netincome`（调整后净利润）`_ft`（预测期标识，ft=远期 forecast total）

### 1.5 ⚠️ 实战技巧
- **不要只看"前缀"**：必须点击进入字段详情，看**实际历史时间**（有的字段 2010 年才有，别想着用 2005 的数据）
- **关注 `ft`/`f1`/`fy1` 等后缀**：这代表不同的预测期
- **截图保存字段名**：你跑过好的字段，截图记下来，下次直接用

## 第 2 步：进入 Simulate 页面（开始写 alpha）

### 2.1 进入 Simulate 的 3 种方式

- 点击顶部 **`Alphas`** 标签

**方式 B：从任意 example 进入**

- 找到 **`Alpha Examples`** 系列文章
- 点击 **`Simulate`** 按钮，会自动填入 example 表达式

**方式 C：从工具进入（你目前用老强说工具的方式）**
- 老强说工具的"一/二/三阶段"配置里，**实际上还是在 BRAIN 平台上跑**
- 工具只是"帮你自动填参数 + 批量提交"
- 等于"你选好配方，工具帮你按"

### 2.2 Simulate 页面的 5 大块

## 第 3 步：编写你的第一条 alpha（10 分钟）

### 3.1 在表达式输入框里粘贴这条
```python
group_neutralize(
    ts_decay_linear(
        ts_backfill(anl4_adjusted_netincome_ft, 120),
        240
    ),
    densify(subindustry)
)
```

### 3.2 表达式编辑器的功能
- **自动补全**：输入算子前几个字符会提示
- **语法检查**：括号不匹配会变红
- **字段搜索**：输入 `anl4` 会列出所有 anl4 开头的字段
- **运行日志**：模拟时会显示"是否合规"等提示

### 3.3 4 个下拉框怎么选（你的 anl4 alpha 适用配置）

| 配置项 | 推荐值 | 解释 |
|---|---|---|
| **Region** | USA | 美国市场（公开用户只能用 USA，顾问可解锁更多） |
| **Universe** | TOP3000 | 美国市值前 3000 只股票（新手友好，样本够多） |
| **Delay** | 1 | T+1 调仓（最常用，模拟真实交易） |
| **Neutralization** | Subindustry | 按子行业中性化（你这条 alpha 用的） |

### 3.4 ⭐ 一句话配置原则
- **Region/Universe/Delay/Neutralization = 你的 alpha 在什么"市场环境"下跑**
- **不同配置下的 Sharpe/Fitness 不可比**（换 universe 等于换股票池）
- **提交前先确定这套配置** —— 提交后改配置 = 算新 alpha

## 第 4 步：点击 Simulate 跑模拟（2-5 分钟）

### 4.1 点击 `[Simulate]` 按钮
- 等待 1-3 分钟（复杂 alpha 可能 5-10 分钟）
- 期间会显示"Running simulation..."

### 4.2 ⚠️ 常见报错及解决
| 报错 | 原因 | 解决 |
|---|---|---|
| `Field not found` | 字段名拼错 | 查 Datasets 字段列表 |
| `No data for period` | 字段时间不够长 | 换字段或缩短时间窗口 |
| `Memory error` | 表达式太复杂 | 简化表达式 / 缩小 delay 范围 |
| `Region not allowed` | 公开账户用了非 USA 区域 | 改回 USA |

### 4.3 模拟成功后，你会看到 6 大指标
```
┌─────────────────────────────────────┐
│  你的 alpha 结果面板                  │
├─────────────────────────────────────┤
│  Sharpe:        1.85    ✓  > 1.25   │
│  Fitness:       1.42    ✓  > 1.0    │
│  Turnover:      18.5%   ✓  1-70%   │
│  Returns:       12.3%   ✓  > 0     │
│  Drawdown:      -5.2%   ✓  越小越好 │
│  Self-Correlation: 0.45 ✓  < 0.7   │
└─────────────────────────────────────┘
```

### 4.4 5 个指标的"小白版"理解

#### ⭐ Sharpe（夏普比率）
- **含义**：每承担 1 份风险，能赚多少
- **计算公式**：`Sharpe = 平均日收益 / 日收益标准差 × √252`
- **健康区间**：1.5 ~ 2.5 较优
- **平台硬门槛**：> 1.25
- **生活类比**：一份工作的"性价比" —— 同样加班，Sharpe 高的赚得更多

#### ⭐ Fitness（健康度）
- **含义**：Sharpe × 收益绝对值 / 换手率的复合指标
- **计算公式**：`Fitness = Sharpe × √(|Returns| / Turnover)`
- **健康区间**：1.5 ~ 3.0
- **平台硬门槛**：> 1.0
- **生活类比**：性价比 × 持久性 —— 既要赚得多，又要换手不频繁

#### ⭐ Turnover（换手率）
- **含义**：每天交易额占本金的比例
- **健康区间**：10% ~ 30%
- **平台硬门槛**：1% ~ 70%
- **生活类比**：装修翻新频率 —— 太频繁费钱，太懒错过机会

#### ⭐ Returns（累计收益）
- **含义**：alpha 在样本内的累计收益
- **健康区间**：越高越好
- **生活类比**：账户总收益

#### Drawdown（最大回撤）
- **含义**：资金曲线最惨的跌幅
- **健康区间**：越小越好（-5% 以内较稳，-20% 较危险）
- **生活类比**：爬山爬到 100 米，掉到 70 米，那 30 米就是回撤

#### ⭐ Self-Correlation（自相关）
- **含义**：新 alpha 与你已交 alpha 的相似度
- **健康区间**：< 0.6
- **平台硬门槛**：< 0.7
- **生活类比**：你的两份简历"长得有多像" —— 越像说明越重复

## 第 5 步：理解结果，决定是否提交

### 5.1 三色判定（平台自动显示）
- **🟢 绿**：全部指标过线 → 可以提交
- **🟡 黄**：部分指标临界 → 看具体情况
- **🔴 红**：必败指标（如 Sharpe < 1.25）→ 不能提交

### 5.2 你的 anl4 alpha 模拟结果怎么看
假设结果显示：
| 指标 | 数值 | 判定 |
|---|---|---|
| Sharpe | 1.85 | 🟢 |
| Fitness | 1.42 | 🟢 |
| Turnover | 18.5% | 🟢 |
| Returns | 12.3% | 🟢 |
| Self-Correlation | 0.45 | 🟢 |

→ **可以提交** ✅

### 5.3 如果某个指标不达标怎么办
- Sharpe 低 → 改表达式逻辑 / 换数据集
- Fitness 低 → 降换手率 / 提 Sharpe
- Turnover 高 → 加 `trade_when` 控制调仓频率
- Self-Correlation 高 → 换数据集 / 换算子 / 换中性化

## 附录：常见问题 FAQ

**Q1：我没有 Learn 标签怎么办？**
A：在你 Profile 里看是不是 Consultant 等级。Gold 阶段应该都有 Learn 权限。

**Q2：模拟器一直 Running 不结束？**
A：超 10 分钟没结果 = 表达式太复杂或字段不存在。检查表达式，看运行日志。

**Q3：为什么我的 alpha 和 example 一样但 Sharpe 不同？**
A：可能你选了不同的 Region / Universe / Delay / Neutralization。**这 4 个配置决定指标的"环境"，不可比**。

**Q4：什么时候该换数据集？**
A：跑 20 条同数据集都失败时，就该换数据集了。

**Q5：Tool 跑出来的指标和平台跑出来的不一样？**
A：以平台为准。工具读的可能有 1-2 天延迟。
