# 世界观设计指南

> 本文件负责"设计世界概念"。在动笔写条目之前，先确定这个世界的核心设定。世界书条目化的工作请转 `world-book-guide.md`。

## 本文内容

>本文指引进行世界观架构建设。指引进行判断用户的世界观属于哪种规模，提取核心差异点，用最小设定集搭建完整世界框架。不写条目、不写 YAML、不碰 world book 配置，只输出世界观概念设计文档。

---

## 工作流程

### 阶段一：规模判定

收到用户的世界观描述后，首先判定类型。三种类型的划分标准见下方"详细规范 - 世界观类型判定"。判定依据是问题"AI 对这个世界的底层有多少已知信息？"

### 阶段二：维度取舍

根据类型确定需要设计的维度。规则：

- **A 类：** 只设计虚构部分。真实部分不写。
- **B 类：** 从六个维度中选择 1-3 个核心维度展开。其余维度一句话带过或略过。
- **C 类：** 所有相关维度分层展开。但不相关的不硬凑。没有势力冲突的世界不写势力。

维度取舍表见下方"详细规范 - 设计维度"。

### 阶段三：总纲输出

将所有设计维度浓缩为一个 100-200 字的总纲。总纲是后续所有条目编写的唯一事实来源，所有具体条目从总纲派生。输出格式见下方"详细规范 - 总纲格式"。

---

## 详细规范

### 世界观类型判定（A/B/C）

#### A 类：真实背景

用户的世界观锚定在现实世界（现代中国/日本/美国等）。AI 数据库已掌握该世界的地理、历史、社会、文化等绝大部分信息。

**判定标准：** 问了"这个世界在哪里"之后，用户给出的地点在真实地图上找得到。

**设计原则：只设计虚构部分。**

你需要设计的不是真实世界已有的东西，而是叠加上去的虚构层。例如"现代东京 + 阴阳师秘密存在"。你只设计阴阳师的传承制度、术式体系、隐世组织。东京的地铁、季节、社会习俗由 AI 自己检索。

**典型输出：**
- 虚构组织的架构和运作方式
- 秘密规则体系的原理和限制
- 虚构群体在社会中的位置和伪装
- 不超过 3 个虚构维度

#### B 类：小世界

用户给出一个成熟类型标签（剑与魔法、赛博朋克、修仙、克苏鲁、末日废土等）。AI 知道这个类型的通用套路，但不了解用户的具体设定。

**判定标准：** 用户说了类型标签，AI 能列出 10 条通用设定，但每一条在用户的世界里都可能是错的。

**设计原则：在通用套路之上，定义 1-3 个核心差异。**

AI 知道"修仙要有灵根/筑基/渡劫"，你需要设计的是"这个世界的修炼体系和所有其他修仙作品的区别在哪里"。

**典型输出：**
- 选中的 1-3 个核心维度的详细设计
- 每个维度必须有区别于同类作品的具体设定
- 不需要完整历史年表或详细地图

#### C 类：大世界

用户要构建一个完全原创的世界。原创种族、原创力量体系、原创地理、原创历史。AI 对这个世界的认知为零，所有信息都需要用户提供。

**判定标准：** 用户描述中出现了无法归类到任何现有模板的概念（原创名词、原创规则、原创势力名称等）。

**设计原则：分层设计，由粗到细。**

- **总纲层（100-200 字）：** 一句话核心概念 + 所有关键维度的一句话摘要。任何条目编写者读完总纲就能理解这个世界的基本逻辑。
- **速览层：** 每个维度的要点列表（3-8 条），供快速查阅。写条目时查速览层就够了，不需要每次读详情。
- **详情层：** 关键维度的展开描述。非关键维度到此停止，不写详情。

**分层判断标准：**
- 总纲层必写。
- 速览层：如果有 3 个以上维度，写。否则总纲本身已足够。
- 详情层：只写"直接影响角色行为决策和故事冲突"的维度。其余停在速览层。

### 设计维度（按需选用）

| 维度 | 核心问题 | 写什么 | 不写什么 |
|------|---------|--------|---------|
| 地理 | 这个世界长什么样？ | 独特区域划分、关键地点及其叙事功能、环境特征影响行为的方式 | AI 已知的真实地理、不影响故事的区域 |
| 历史 | 什么事件塑造了现在？ | 改变世界格局的关键事件（不超过 5 个）、距今时间、影响范围 | 完整编年史、与当前故事无关的远古事件 |
| 势力 | 谁在争夺什么？ | 组织/国家/阵营的名称、定位、核心诉求、相互关系、冲突点 | 没有故事关联的势力、只有名字没有立场的空壳 |
| 规则 | 这个世界怎么运转？ | 力量体系的来源/运作/限制/代价/层级 | 通用套路（如"灵气是天地能量"），除非你有独特的来源设定 |
| 社会结构 | 人怎么被分类？ | 阶级/种族/文化墙的形成原因、流动可能性、身份对行为的影响 | 没有叙事冲突的扁平社会 |
| 技术水平 | 能做什么不能做什么？ | 关键技术的存在/限制/垄断状态/社会影响 | 标准现代科技清单 |

**维度选择规则：**
1. 每个选中的维度必须回答"这个设定怎么影响角色和故事"。
2. 删掉一个维度后，故事是否还能成立？能 → 删。
3. A 类至少选 1 个维度（虚构层），B 类选 1-3 个，C 类选所有相关维度。

### 总纲格式

输出一个 100-200 字的纯信息块。每一行都是可被直接查询的数据陈述。不使用任何文学化表达。

```
【世界名】中文名称 | English Name
【类型】A / B / C
【核心概念】一句话——这个世界最根本的决定性设定
【地理】区域划分 + 关键地点（A 类只写虚构部分）
【历史】关键事件 + 距今时间（A 类只写虚构部分）
【势力】组织名称 → 定位 → 诉求 → 与其他势力的关系
【规则】核心机制 + 来源 + 限制 + 代价
【社会】结构描述 + 形成原因 + 矛盾点
【技术】关键技术 + 限制条件 + 社会影响
```

**零度写作要求：** 不使用比喻句。不使用形容词堆叠。每条信息都是名词+数据+逻辑关系的陈述。判断标准。把总纲导入数据库，每行可作为一个独立记录查询。

**正确示例：**
```
【地理】雨隐村——终年降雨的忍者村落，建筑以金属管道和塔楼连接，人口约 3000。
```

**错误示例：**
```
【地理】雨隐村被永不停歇的雨水环抱，铁灰色的塔楼刺破灰暗的天际线，整座城市在细密的水幕中静默地呼吸着。
```

### 独创指南

1. **从故事反推设定，不脑补完整架空历史。** 用户提供的是"一个世界观"的创作需求，但你需要的是"影响这个故事的设定集"。从故事出发问：这个故事发生，最少需要哪些设定？答案就是你的输出范围。

2. **AI 已知信息不写。** 以下内容直接跳过：
   - 真实世界常识（"日本是岛国""东京是首都"）
   - 类型通用套路（"修仙要吸收灵气""精灵有尖耳朵""赛博朋克有霓虹灯"）
   - 与当前故事无关的历史填充

3. **分层仅限 C 类。** A 类和 B 类不写速览层和详情层。A 类一个总纲 + 虚构维度要点。B 类总纲 + 各维度的简短描述。只有 C 类才需要三层结构。

4. **区分"关键设定"和"背景填充"。** 测试方法是：删掉这个设定，角色在故事中的行为会不会改变？不会 → 它是背景填充，不展开。

5. **原创世界观的每个概念都要有"锚点"。** 引入一个原创名词时，必须有对应的功能描述。不能只抛出一个名字。"该大陆存在一种叫'晶脉'的资源"。然后不解释晶脉是什么、谁在用、为什么重要。

## 自查清单

- [ ] 类型判定是否正确（A/B/C）？判定依据是否来自"AI 已知多少"这个标准？
- [ ] A 类：是否只写了虚构层，没有复述真实世界常识？
- [ ] B 类：是否找到了 1-3 个核心差异点，而非把类型通用套路重写一遍？
- [ ] C 类：是否分层设计（总纲→速览→详情），而非一次展开所有内容？
- [ ] 总纲是否在 100-200 字范围内？
- [ ] 总纲是否零度写作（无比喻、无形容词堆叠、纯信息陈述）？
- [ ] 选取的维度是否都通过了"删掉后故事还能成立吗"的测试？
- [ ] 是否避免了 AI 已知信息的重复？
- [ ] 每个维度是否有具体内容，而非空洞标签（如"科技发达""历史悠久"）？
- [ ] 原创概念是否都有功能锚点（名称 → 是什么 → 谁在用 → 为什么重要）？
- [ ] 文件自身是否不含禁词（参考 `writing-optimization-guide.md` 禁词清单）？

## 常见错误示例

### 错误 1：A 类写了真实世界常识

```
错误：
【地理】东京位于日本关东地区，由 23 个特别区组成，人口约 1400 万。

正确：
【地理】东京——阴阳厅总部设于新宿地下 30 米，"隐世"入口分布于都内 12 座指定神社的鸟居。虚构部分仅限于此，其余东京地理由 AI 自行检索。
```

**原因：** AI 知道东京在哪、有多大。你只写与虚构设定相关的部分。

---

### 错误 2：B 类复述了类型通用套路

```
错误：
【规则】灵气——天地间游离的能量物质，修道者通过吸收灵气淬炼身体，修为越高需要的灵气纯度越高。

正确：
【规则】灵气的来源并非天然——上古"界膜战争"撕裂了此世界的空间壁障，灵气从裂缝中渗入。三大宗门各自垄断一条主裂缝，散修只能吸收被稀释后的残灵。这意味着：散修的上限由地理位置决定，而非天赋。
```

**原因：** AI 知道灵气是什么，不需要解释定义。你只需要写"这个世界的灵气和所有其他修仙作品的区别在哪里"。

---

### 错误 3：C 类脑补了完整的架空历史

```
错误：
【历史】
创世纪元：神族创世
第一纪元（约 3000 年）：龙族统治
第二纪元（约 1500 年）：精灵兴起
第三纪元早期（约 200 年）：人兽战争
第三纪元中期（约 500 年）：魔法革命
第三纪元晚期（约 300 年）：帝国建立
第四纪元：现时代（已持续约 800 年）

正确：
【历史】200 年前的"界膜战争"是当前世界格局的唯一关键节点。战争结果：神族消失、龙族退守北方冻原、人类获得魔法使用权并建立七国同盟。更早的历史时期对当前故事无影响，暂不展开。需要时再追加。
```

**原因：** 列出一万年的历史，但只有最近 200 年与故事相关。前 9800 年只占 token 预算，不提供有效信息。

---

### 错误 4：总纲写成了文学化描写

```
错误：
衔月城悬浮于云海之上，月光流淌在琉璃瓦面，整座城市笼罩在清冷的银辉中。城中百姓世代生活在云端，街巷间仙气弥漫，让人感到超凡脱俗。

正确：
【地理】衔月城——悬浮云海的千年都市，海拔约 3000m，建筑以琉璃瓦和白石为主。城设三座浮空港连接地面。人口约 3 万，以御风术为日常交通。城内无地面道路，所有移动依赖浮空步道和风力通道。
```

**原因：** 总纲是数据库，不是小说开篇。AI 需要的是可查询的结构化信息，不是意境描写。

---

### 错误 5：设计了不影响故事的冗余设定

```
错误：
【社会】衔月城税制：云税（商业税 5%）、风税（交通税每趟 3 铜币）、月税（房产税按面积征收）……共 12 种税目。

正确：
【社会】衔月城——贵族/云商/平民三级分层。贵族垄断御风术教学资格，云商通过与地面贸易积累财富，平民从事基础劳动。阶级流动需要三代以上的财富积累和一个贵族担保人——这条规则是城中大多数冲突的根源。
```

**原因：** 如果税收制度不会在故事中触发任何事件，它就是冗余信息。社会结构写了三级分层+流动障碍+矛盾根源。这才直接影响角色的行为和故事走向。

---

### 错误 6：原创概念没有锚点

```
错误：
【规则】此世界存在"晶脉"资源，分布于大陆各地。

正确：
【规则】晶脉——大陆地下的结晶矿脉，是驱动所有魔法装置的唯一能源。由帝国矿业公会垄断开采，独立法师无法合法获取。非法开采者被称为"盗脉者"，一经发现终身监禁。晶脉的枯竭速度远超再生速度——已有 3 个边境城市因晶脉枯竭被废弃。
```

**原因：** "晶脉"只是一个词。加上"是什么→谁控制→获取规则→当前问题"后，它才是一个可以被故事使用的设定。
