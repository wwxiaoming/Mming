# 00_三条路线总纲

```yaml
position: 0
constant: true
preventRecursion: true
excludeRecursion: true

content: |
  ## 变量系统

  ### 全局变量
  | 变量名 | 类型 | 说明 |
  |--------|------|------|
  | `selected_hero` | string | 开局选择的英雄（15选1） |
  | `route_type` | string | 路线类型：A(收服者)/B(小英雄)/C(魔王) |
  | `plot_progress` | int | 剧情进度值（0-100） |
  | `current_stage` | string | 当前阶段标识 |
  | `hero_status` | object | 15位英雄的收服状态 |

  ### hero_status结构
```
  hero_status = {
    "问天": "未收服",      // 核心8人
    "千瑞": "未收服",
    "子夜": "未收服",
    "空耀": "未收服",
    "阚泽": "未收服",
    "凯风": "未收服",
    "诗华": "未收服",
    "嘉陵": "未收服",
    "文斌": "未收服",      // 独立7人
    "启润": "未收服",
    "岚云": "未收服",
    "准星": "未收服",
    "星野": "未收服",
    "战炎": "未收服",
    "暮黎": "未收服"
  }
  ```

  ## 开局选择机制

  用户开局需选择：
  1. **路线类型**（A/B/C）
  2. **起始英雄**（15位中任选一位）

  选择后系统自动设置变量：
  - `route_type` = 用户选择的路线
  - `selected_hero` = 用户选择的英雄
  - `plot_progress` = 0
  - `current_stage` = "开局"

  ## 路线A：收服者（从零调教）

  ### 核心机制
  根据`selected_hero`动态生成初始状态：
  - 核心8人之一：该英雄为"未收服"，其余14人状态根据剧情设定
  - 独立7人之一：该英雄为"未收服"，核心8人状态根据剧情设定

  ### 阶段划分（通用）
  | 阶段 | 标识 | 触发条件 | 核心内容 |
  |------|------|---------|---------|
  | A1 | 初遇观察 | 开局自动触发 | 天火观察目标英雄，记录弱点 |
  | A2 | 弱点突破 | A1完成 + 特定事件 | 针对弱点制造独处机会 |
  | A3 | 战斗捕获 | A2完成 + 战斗事件 | 利用战斗场景捕获目标 |
  | A4 | 调教收服 | A3完成 + 调教场景 | 针对性调教，建立主从关系 |
  | A5 | 羁绊深化 | 收服后日常触发 | 通过补魔、训练深化羁绊 |

  ### 动态收服逻辑
  收服一位英雄后：
  1. 更新`hero_status[英雄名]` = "已收服"
  2. `plot_progress` += 约7（15人总计100）
  3. 自动触发下一位英雄的收服剧情或继续当前英雄深化

  ## 路线B：小英雄（8+7双轨制）

  ### 核心机制
  根据`selected_hero`动态分配剧情线：

  #### 若选择核心8人之一
  - **主线**：帝王战队日常任务与队友羁绊
  - **支线**：与天火的互动（调教可选，非强制）
  - **队友**：与其他7位核心成员的羁绊建立

  #### 若选择独立7人之一
  - **主线**：专属独立剧情线（见各英雄篇）
  - **支线**：与核心8人的交集（可选）
  - **发展**：恋爱/羁绊/独立成长，不强制调教

  ### 阶段划分（核心8人通用）
  | 阶段 | 标识 | 触发条件 | 核心内容 |
  |------|------|---------|---------|
  | B1 | 战队日常 | 开局自动触发 | 日常训练、任务准备 |
  | B2 | 任务执行 | B1完成 + 任务事件 | 对抗魔神会、执行委托 |
  | B3 | 队友羁绊 | B2完成 + 互动事件 | 与队友建立羁绊关系 |
  | B4 | 天火互动 | 日常触发 | 与天火的调教/羁绊互动（可选） |

  ### 独立7人剧情线
  每位独立英雄有专属剧情指导（见09-15篇）：
  - 文斌：造神计划→暴走觉醒
  - 启润：校园相识→岚云恋爱
  - 岚云：血脉觉醒→启润恋爱
  - 准星：被皇后捕获→挣扎反抗
  - 星野：化妆舞会→逃亡求生
  - 战炎：初步沦陷→兄弟羁绊
  - 暮黎：龙渊试炼→彻底淫堕

  ## 路线C：魔王（继承8位遗产）

  ### 核心机制
  根据`selected_hero`动态生成初始状态：
  - 核心8人之一：该英雄为"已收服"，天火管理其日常
  - 独立7人之一：该英雄为"未收服"，需制定收服计划

  ### 初始状态（固定）
  路线C开局时：
  - 核心8人：`hero_status` = "已收服"
  - 独立7人：`hero_status` = "未收服"

  ### 阶段划分（通用）
  | 阶段 | 标识 | 触发条件 | 核心内容 |
  |------|------|---------|---------|
  | C1 | 继承遗产 | 开局自动触发 | 确认8位已收服英雄状态，接手管理 |
  | C2 | 管理日常 | C1完成后日常触发 | 管理已收服英雄的调教与补魔 |
  | C3 | 收服新目标 | 特定事件触发 | 针对7位未收服英雄制定收服计划 |
  | C4 | 最终收服 | 完成C3 + 最终事件 | 完成全部15位英雄的收服 |

  ### 已收服英雄管理
  8位已收服英雄的日常：
  - 补魔：定期精液补充
  - 调教：日常敏感度维持
  - 任务：参与战队行动
  - 偶尔反抗：触发惩罚剧情

  ## 三条路线差异对照

  | 维度 | 路线A | 路线B | 路线C |
  |------|-------|-------|-------|
  | {{user}}身份 | 天火/指挥官 | 15位英雄之一 | 天火/主教继承人 |
  | 初始收服数 | 0（从selected_hero开始） | 0 | 8（核心8人已收服） |
  | 主要视角 | 征服者 | 英雄本人 | 统治者 |
  | 核心冲突 | 从零收服15人 | 双轨选择（调教/恋爱/羁绊） | 管理8人+收服7人 |
  | 叙事节奏 | 循序渐进 | 沉浸体验/多线并行 | 高潮收束 |
  | 动态生成 | 根据selected_hero生成初始状态 | 根据selected_hero分配剧情线 | 根据selected_hero确定管理/收服对象 |
```
