# 股票分析技能整合文档 · 详细分析

> 本文是 `/workspace/股票分析技能整合文档.md` 的逐技能深度解读，覆盖核心价值、关键能力、典型使用、优势局限与集成评价，并对整体架构给出改进建议与角色适配方案。

---

## 总体评估

整合文档对 TRAE 内 14 个股票相关技能做了体系化整合，构建了 **「L1 资讯 → L2 选股 → L3 分析 → L4 产出」** 四层漏斗式工作流。优点是覆盖度全、衔接清晰、决策树实用；下面按层级逐个技能做深度拆解。

---

## 一、L1 资讯层（信息输入端）

### 1. `last30days` — 舆情与情绪触发器

| 维度 | 分析 |
|------|------|
| **核心价值** | 抓取 Reddit、X、YouTube、TikTok、HN、Polymarket、GitHub 等海外平台近 30 天的真实讨论，量化「热度」与「共识度」 |
| **数据时效** | 30 天滚动窗口，刚好覆盖一个完整的中期题材发酵周期 |
| **关键能力** | 多平台并行抓取 + 互动量（engagement）排序，能区分"小众深度讨论"与"散户情绪噪音" |
| **典型使用** | 工作流 D 启动器、判断海外催化剂是否传导到 A 股（如英伟达 GPU → A 股光模块、PCB 链） |
| **局限性** | 对中文圈覆盖弱，需配合 A 股个股新闻（a-stock-data 巨潮/东财）使用 |
| **集成评价** | ⭐⭐⭐⭐⭐ 唯一的「外部信号源」，不可替代 |

---

## 二、L2 选股层（候选池构建端）

### 2. `a-share-screener` — A 股多策略批量筛选器

| 维度 | 分析 |
|------|------|
| **核心价值** | 内置价值/成长/动量/分红四套预设策略，可自定义指标阈值 |
| **关键能力** | 低 PE 高 ROE、北向资金加仓、突破年线等多维组合筛选 |
| **典型使用** | 工作流 B 的入口；快速把 5000 只票缩到 30-50 只候选池 |
| **优势** | 覆盖广、策略成熟、开箱即用 |
| **局限** | 缺乏"为什么"——只给名单不给逻辑 |
| **集成评价** | ⭐⭐⭐⭐⭐ A 股宽基筛选的事实标准 |

### 3. `eastmoney_select_stock` — 东方财富条件选股 + 板块映射

| 维度 | 分析 |
|------|------|
| **核心价值** | 接入东财官方条件选股 API + 行业/板块/指数成分股查询 |
| **关键能力** | 板块成分股精准定位（自上而下选行业的必备工具） |
| **典型使用** | 工作流 E 的关键节点（行业→成分股）、动量策略的板块归属识别 |
| **优势** | 数据源权威（东财），板块分类与官方一致 |
| **局限** | 接口稳定性需关注，复杂财务筛选不如 a-share-screener 灵活 |
| **集成评价** | ⭐⭐⭐⭐ 行业映射的最优解 |

### 4. `longbridge-value-investing` — 格雷厄姆/巴菲特方法论

| 维度 | 分析 |
|------|------|
| **核心价值** | 唯一同时覆盖 Graham（NCAV 烟蒂）+ Buffett（经济护城河）双方法论的技能 |
| **关键能力** | 跨报表对账、防御型投资者打分、优质复利打分 |
| **数据源** | 优先 Longbridge CLI → MCP 兜底 → WebSearch 补缺 |
| **典型使用** | 工作流 A/B 的估值锚点；判断一只票是"便宜"还是"该便宜" |
| **优势** | 理论框架扎实、有明确的"通过/不通过"标准 |
| **局限** | 对成长股、题材股不友好，需配合 china-stock-analysis |
| **集成评价** | ⭐⭐⭐⭐⭐ 价值投资者的"宪法" |

### 5. `china-stock-analysis` — akshare 价值投资分析

| 维度 | 分析 |
|------|------|
| **核心价值** | 基于 akshare 公开财务数据，提供低频交易者友好的价值分析 |
| **关键能力** | 个股深度分析、行业对比、估值计算（DCF/相对估值） |
| **典型使用** | 价值投资视角的二次过滤；与 longbridge 形成"双保险" |
| **优势** | 数据源透明（akshare 开源）、逻辑易理解 |
| **局限** | 实时性弱、缺少研报/资金面信号 |
| **集成评价** | ⭐⭐⭐⭐ 价值派的"中文版说明书" |

### 6. `serenity-radar` — 跟踪 Serenity 注意力轮动

| 维度 | 分析 |
|------|------|
| **核心价值** | 监控独立研究者 @aleabitoreddit 的全量提及记录，捕捉其注意力迁移路径 |
| **关键能力** | RADAR 模式（实时动量扫描）+ GENERATOR 模式（生成候选） |
| **方法论** | 主题轮动、选择签名、催化剂剧本三重模式挖掘 |
| **典型使用** | 工作流 D 的起点；学习顶级研究者的标的发现逻辑 |
| **优势** | 把"软信息"（注意力）量化成"硬信号"（轮动方向） |
| **局限** | 强依赖单一信源，需保持批判性，不可照搬 |
| **集成评价** | ⭐⭐⭐⭐ 独门兵器，但**仅作候选生成器，不作交易信号** |

### 7. `serenity-stock-choke` — A 股卡脖子标的筛选

| 维度 | 分析 |
|------|------|
| **核心价值** | 应用 Serenity 的"卡脖子"理论，把任一板块拆解到「断货即停产」环节 |
| **关键能力** | 产业链结构化梳理 + 卡点环节小盘标的筛选 |
| **数据源** | neodata-financial-search + westock-data |
| **典型使用** | 工作流 C 的核心；挖掘被市场低估的国产替代标的 |
| **优势** | 框架可复用、跨板块适用（电力/光模块/医疗器械/军工等） |
| **局限** | 不适用纯题材炒作、对小票流动性有要求 |
| **集成评价** | ⭐⭐⭐⭐⭐ 价值发现的"独门暗器" |

## 三、L3 分析层（深度尽调端）

### 8. `a-stock-data` — A 股全栈数据工具包（七层数据源）

| 维度 | 分析 |
|------|------|
| **核心价值** | **整个整合文档中数据最厚的一个技能**，自包含零外部依赖 |
| **七层数据源** | ① 行情（mootdx+腾讯+百度K线）② 研报（东财+同花顺+iwencai）③ 信号（同花顺热点+北向+龙虎榜+解禁+行业）④ 资金面（融资融券+大宗+股东户数+分红+120日分钟级资金流）⑤ 新闻（东财个股+全球）⑥ 基础数据（mootdx财务/F10+东财+新浪三表）⑦ 公告（巨潮） |
| **工程亮点** | 内置东财限流防封、mootdx/腾讯不封 IP 兜底 |
| **典型使用** | 工作流 A/C/D 的数据底座；任何需要"原始数据 → 结构化结论"的场景 |
| **优势** | 覆盖度碾压同类、自带反爬策略、开箱即用 |
| **局限** | 调用代码量大，需根据场景挑层使用 |
| **集成评价** | ⭐⭐⭐⭐⭐ **整合文档中最核心的"数据中台"** |

### 9. `a-share-analysis` — A 股综合分析

| 维度 | 分析 |
|------|------|
| **核心价值** | 在 a-stock-data 之上做"四维综合分析"：基本面 + 技术面 + 政策面 + A 股特性 |
| **关键能力** | 内置 T+1、涨跌幅限制、北向资金、停复牌规则等 A 股特性 |
| **典型使用** | 工作流 A/B 的核心分析节点；产出一只票的"全息画像" |
| **优势** | 把零散数据整合成投资观点、有"中国特色"考量 |
| **局限** | 不替代深度行业研究、产业链分析 |
| **集成评价** | ⭐⭐⭐⭐⭐ A 股个股分析的"瑞士军刀" |

### 10. `serenity-skill` — 产业链瓶颈深度调研

| 维度 | 分析 |
|------|------|
| **核心价值** | 把研究助手变成"供应链瓶颈猎手"，强调 source-backed 研究 |
| **关键能力** | 产业链/价值链映射 + A 股/HK/US 三市场映射 + 论文压力测试 |
| **典型使用** | 工作流 C 的研究框架层（与 serenity-stock-choke 形成"框架→落地"配对） |
| **优势** | 强调证据链、拒绝拍脑袋、可挑战论点 |
| **局限** | 输出偏长、需要用户有耐心消化 |
| **集成评价** | ⭐⭐⭐⭐⭐ 价值链研究的"深度模式" |

### 11. `stock-watcher` — 自选股管理

| 维度 | 分析 |
|------|------|
| **核心价值** | 维护个人观察池，跟踪近期表现（数据源：10jqka） |
| **关键能力** | 增删查 + 表现摘要 + 持续跟踪 |
| **典型使用** | 工作流 A/B 的"持久层"；投资决策的"中央仓库" |
| **优势** | 简单、稳定、长期主义友好 |
| **局限** | 不做基本面分析、需配合其他技能 |
| **集成评价** | ⭐⭐⭐⭐ 投研工作流的"数据库" |

### 12. `data-analysis` — 本地 Excel/CSV 数据分析

| 维度 | 分析 |
|------|------|
| **核心价值** | 处理用户自有的财务数据包（多 sheet、聚合、过滤、SQL 查询、导出） |
| **关键能力** | 透视表、SQL、CSV/JSON/Markdown 导出 |
| **典型使用** | 工作流 B 的财务数据深度分析；自建数据集的二次加工 |
| **优势** | 解决"我有数据但不会分析"的痛点 |
| **局限** | 仅限本地文件、需先有数据 |
| **集成评价** | ⭐⭐⭐⭐ 离线分析的"瑞士军刀" |

## 四、L4 产出层（报告输出端）

### 13. `investment-memo` — 投资备忘录

| 维度 | 分析 |
|------|------|
| **核心价值** | 撰写专业投决会级别的投资备忘录（VC/PE/二级市场通用） |
| **关键能力** | 投资论点 / 风险 / 建议三段式结构 + 严谨表述 |
| **典型使用** | 工作流 A/D 的最终产出；对内汇报、对外路演 |
| **优势** | 结构稳定、便于归档、决策可追溯 |
| **局限** | 篇幅有限、深度报告需 consulting-analysis |
| **集成评价** | ⭐⭐⭐⭐⭐ 个股决策的"标准交付物" |

### 14. `consulting-analysis` — 咨询级研究报告

| 维度 | 分析 |
|------|------|
| **核心价值** | 双阶段工作流：① 生成分析框架与数据需求清单 ② 拿到数据后产出咨询级成稿 |
| **适用范围** | 市场分析、消费者洞察、品牌分析、财务分析、行业研究、竞品情报、投研尽调 |
| **典型使用** | 工作流 C/E 的顶层产出；产业链/行业尽调级报告 |
| **优势** | 框架感强、嵌入图表、叙事有节奏 |
| **局限** | 对话轮次多、需配合其他技能补数据 |
| **集成评价** | ⭐⭐⭐⭐⭐ 行业/赛道研究的"重武器" |

## 五、整体架构评价

### 5.1 优点

1. **分层合理**：L1→L4 天然对应"信号输入→候选生成→深度验证→成品输出"的投研漏斗。
2. **方法论覆盖完整**：既有量化（a-share-screener）又有价值（longbridge/china-stock-analysis）还有产业链（serenity）还有情绪（last30days）。
3. **衔接明确**：文档第七节"技能联动速查表"给出了 8 条最常用衔接路径，可直接当 SOP 用。
4. **风险提示到位**：第八节强调"仅供研究支持，不构成交易建议"，且把 A 股特性（T+1、涨跌幅）内置到 a-share-analysis。

### 5.2 可改进点

1. **缺少「复盘层」**：投资决策做完后，没有专门的「业绩归因/错误分析」技能，建议在工作流末尾增加 `stock-watcher` 表现回看。
2. **跨市场协同弱**：港股、美股仅在 a-stock-data 中零星涉及，建议明确港美股投研路径。
3. **风险/合规层缺失**：没有专门的风控/仓位管理技能，建议补充一个简单的组合管理工具。
4. **时序版本管理**：文档 v1.0 缺少 changelog，新增技能时易遗漏更新。

### 5.3 适配建议（按用户角色）

| 用户类型 | 推荐核心工作流 |
|----------|----------------|
| **A 股短线交易者** | last30days → a-share-screener → a-share-analysis → investment-memo |
| **价值投资者** | a-share-screener → longbridge-value-investing → china-stock-analysis → investment-memo |
| **产业链/赛道猎手** | serenity-skill → serenity-stock-choke → a-stock-data → consulting-analysis |
| **跟单学习者** | serenity-radar → serenity-skill → a-stock-data → investment-memo |
| **行业研究员** | consulting-analysis → last30days → eastmoney_select_stock → a-share-screener → consulting-analysis |

## 六、总结

整合文档构建的 14 个技能 + 5 条核心工作流 + 10 套选股策略矩阵，本质上是把 TRAE 打造成了一个**「多策略、可组合、有方法论支撑」**的投研操作系统。**`a-stock-data` 是数据中台、`a-share-analysis` 是 A 股适配层、`serenity` 系列是独门心法、`consulting-analysis/investment-memo` 是交付件**——这 4 类构成核心，其余 10 个技能围绕它们做补强。按工作流模板调用即可覆盖 90% 的 A 股投研需求。
