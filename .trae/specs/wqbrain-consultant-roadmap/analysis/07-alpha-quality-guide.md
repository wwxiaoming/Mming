# Alpha 质量提升实战指南：从"用工具"到"自己会写、会改"

> **目标读者**：会跑 BRAIN 平台、懂一些 Python，但金融和算子都还在"半盲"状态的实战派
>
> **核心命题**：你的 alpha 现在长这样——
>
> ```python
> group_neutralize(
>     ts_decay_linear(
>         ts_backfill(anl4_adjusted_netincome_ft, 120),
>         240
>     ),
>     densify(subindustry)
> )
> ```
>
> 这是一个典型的"模板风格"alpha：拿一个分析师预期数据 → 补全 120 天 → 240 天线性衰减 → 行业中性化。**能用，但有 4 个明显的天花板**：① 没有动量/反转过滤、② 没有交叉验证信号、③ 中性化层级可能不是最优、④ 衰减函数单一。**这份指南就是教你怎么从"会跑"走向"会改"**。

## 一、好 alpha 的 5 个健康指标（金融小白友好版）

BRAIN 平台用一个公式能不能挣到钱，**不靠直觉，靠 5 个数字**。先把这 5 个数字搞明白，再谈改造。

### 1.1 指标总览表

| 指标 | 一句话解释 | 提交门槛 | 健康区间 | "好 alpha" 目标 |
|---|---|---|---|---|
| **Sharpe（夏普比率）** | 承担 1 单位风险能赚多少 | > 1.25 | 1.5–2.5 | ≥ 2.5 |
| **Fitness（健康度）** | 性价比 = Sharpe × √(收益绝对值/换手率) | > 1.0 | 1.5–2.5 | ≥ 3 |
| **Turnover（换手率）** | 每天交易额占本金的比例 | 1%–70% | 15%–40% | ≤ 30% |
| **Returns（年化收益）** | 这套策略一年能赚几个点 | 越高越好 | 5%–15% | ≥ 10% |
| **Drawdown（最大回撤）** | 资金曲线最惨的一次从高到低 | 越低越好 | < 10% | < 5% |
| **Self-Correlation（自相关）** | 和你账户里已有 alpha 的相似度 | < 0.7 | < 0.5 | < 0.3 |

> 💡 **生活类比**：
> - **Sharpe** ≈ 一份工作的"性价比"——年薪 20 万但天天加班 vs 年薪 15 万从不加班，Sharpe 告诉你哪个更划算。
> - **Fitness** ≈ "性价比 × 持久性"——不光赚得多，还得换手少、摩擦成本低。
> - **Turnover** ≈ 装修房子的"翻新频率"——每天翻新太费钱，一年不动又错过新机会。
> - **Returns** ≈ 这套策略一年能赚百分之多少。
> - **Drawdown** ≈ 你买房后房价最惨的跌幅。
> - **Self-Correlation** ≈ 你的两份"工作简历"长得多像——太像了说明没新意。

### 1.2 每个指标的"避坑要点"

#### 坑 1：Sharpe 高但 Fitness 低
- **症状**：Sharpe = 1.8（漂亮），但 Turnover = 65%（几乎天天换手），结果 Fitness 只有 0.9。
- **原因**：交易成本把利润吃掉了。BRAIN 默认每笔交易 1–2bps，Turnover 60% 意味着一年摩擦成本约 15%，再高的 Sharpe 也被吃光。
- **避坑**：高 Sharpe 不等于高 Fitness。看 Fitness 综合分，**才能识别"漂亮但贵"的 alpha**。

#### 坑 2：Fitness 高但 Drawdown 爆表
- **症状**：Fitness = 2.0，回测一看中间某年回撤 35%。
- **原因**：策略可能在 2008、2020 这种极端市直接"打骨折"。Drawdown 大会让你（或平台）拿不住。
- **避坑**：永远要看 Drawdown 曲线。**回撤 < 10% 才有"可重复 alpha"的价值**。

#### 坑 3：Returns 高但 Self-Correlation 高
- **症状**：Returns 20%，但 Self-Correlation = 0.72（> 0.7 红线）。
- **原因**：你这个 alpha 跟账户里已有的某个太像了，平台认为"撞车"，不收。
- **避坑**：每次提交前先做相关性检查，**Self-Correlation < 0.6 是稳妥的"留 0.1 余量"**。

#### 坑 4：Turnover 极低（< 5%）
- **症状**：换手率 3%，看起来"省摩擦费"，但 Returns 也很低。
- **原因**：策略太"懒"，信号更新慢，错过市场变化。
- **避坑**：Turnover 在 **15%–40%** 是"既不懒也不累"的甜区。

#### 坑 5：Sharpe 异常高（> 5）
- **症状**：一个 D1（隔日交易）策略 Sharpe 跑到 6.0。
- **原因**：**90% 的概率是过拟合**——你用了一个未来数据（look-ahead bias）或者踩到了幸存者偏差。
- **避坑**：D1 策略 Sharpe > 3 就要警惕，**> 5 基本就是 overfit**。立刻去检查字段是否引入了未来信息。

## 二、3 类算子的金融含义（配合你的原 alpha 拆解）

把算子按功能分成 3 类，**对你原 alpha 的每一行做白话翻译**。

### 2.1 时序算子（ts_xxx）：看一只股票"自己"的历史

时序算子把"一只股票自己的过去 N 天"做各种数学运算。**相当于你只看一只股票自己的成长曲线**。

| 算子 | 白话解释 | 生活类比 |
|---|---|---|
| `ts_decay_linear(x, d)` | 把过去 d 天的 x 加权平均，越近的权重越大 | 你的"近 7 天平均工资"——最近的权重最大 |
| `ts_backfill(x, d)` | 用最近的有效值向前回填最多 d 天（处理缺失值） | 公司今年没发年终奖 → 用去年那次顶上去 |
| `ts_delta(x, d)` | 今天的 x 减去 d 天前的 x | 你现在比一周前胖了几斤 |
| `ts_rank(x, d)` | 当前 x 在过去 d 天里的排名（百分位） | 今天的温度在近 7 天里排第几热 |
| `ts_mean(x, d)` | 过去 d 天 x 的简单平均 | 过去一周平均每天喝几杯咖啡 |
| `ts_std_dev(x, d)` | 过去 d 天 x 的波动 | 你过去一周花的钱"忽大忽小"的程度 |

**你原 alpha 的应用**：

```python
ts_backfill(anl4_adjusted_netincome_ft, 120)
```
- **做什么**：分析师一致预期净利润（`anl4_adjusted_netincome_ft`）有时候有缺失值，**用最近一次有效的预测值向前回填，最多回填 120 天**。
- **金融含义**：相当于"分析师最近一次说这只股能赚 X 亿，因为分析师不会天天更新报告，最近一次就当现在用，120 天内都算有效"。

```python
ts_decay_linear(..., 240)
```
- **做什么**：对刚才那个值，再做 240 天的线性衰减加权平均——最近 1 天的权重最高，240 天前的权重趋近于 0。
- **金融含义**：分析师预测**离今天越近，对未来股价的指引作用越大**。240 天前的旧预测跟当前股价的相关性已经很低了，权重压低。

### 2.2 截面算子（rank、quantile）：同一时间"横着比"

截面算子在**同一天对所有股票做横向比较**。**相当于全班同学排名次**。

| 算子 | 白话解释 | 生活类比 |
|---|---|---|
| `rank(x)` | 同一时间，x 在所有股票里的排名（0–1 之间） | 全班 50 人，你考第几名 → 0.7 |
| `quantile(x, 5)` | 把所有股票分 5 档（0–4） | 把全班分成 5 个"梯队" |
| `scale(x)` | 按横截面缩放到总和 = 1 | 给每个班干部分配"话语权比例" |

**应用要点**：
- `rank` 把"苹果和橘子"放在同一坐标系——**一只股价 1000 块的茅台和股价 5 块的中小盘，rank 后是公平的**。
- 你的 alpha **没用** rank，但其实可以加一层 `rank(...)` 包裹，让分析师预期在"全市场分位"上有意义，而不是直接用绝对金额。

### 2.3 向量/中性化算子（group_neutralize、densify、trade_when）：做"过滤"和"分组"

这是**让你的 alpha 不偏向某个行业/市场**的关键工具。

| 算子 | 白话解释 | 生活类比 |
|---|---|---|
| `group_neutralize(x, group)` | 在 group 分组内，把 x 减去组内均值（让每个组的平均信号为 0） | 全班平均分一样，**看的是谁比平均分高多少** |
| `densify(group)` | 把 group 字段"稠密化"（处理有些股票没有分类的情况） | 把"未填职业"的同学都打上"未知"标签，方便分组 |
| `trade_when(cond, x)` | 当 cond 条件成立时才交易 x，否则不动 | 雨天才带伞、晴天就不带 |

**你原 alpha 的应用**：

```python
group_neutralize(..., densify(subindustry))
```
- **做什么**：把刚才那个"分析师预测+时间衰减"的信号，**在 subindustry（细分行业）分组内做中性化**。
- **金融含义**：消除"行业偏见"——如果科技股整体被分析师看多，你不希望 alpha 只是在"买科技股"；中性化后，**只保留"在科技股内部、分析师相对更看好某只"的信息**。

> 💡 **整体逻辑**：你的 alpha 等于在说——"**在每个细分行业内部，过去 240 天里分析师最新预测调整方向最积极的那只股**"。这是一个非常合理的"分析师修订动量"思路，**但还有改造空间**。

## 三、6 类经典 alpha 模式构造模板

下面 6 个模板是从 BRAIN 平台 20 万+ 顾问中沉淀下来的"经典骨架"。**每个都配金融逻辑 + 最小表达式 + 撞车预警**。

### 3.1 动量（Momentum）：强者恒强

- **金融逻辑**：涨得好的股票未来一段时间还会涨（市场有"惯性"）。
- **最小骨架**：
  ```python
  ts_delta(close, 20)         # 过去 20 天涨了多少
  ```
- **撞车点**：**这个是 BRAIN 上撞车率最高的模板**。建议至少加一层 `ts_rank(..., 60)` 或除以 `ts_std_dev(returns, 30)` 把它变成"风险调整后的动量"。

### 3.2 反转（Reversal）：涨多了会跌

- **金融逻辑**：短期过度上涨的股票会回归正常，5 天内大幅上涨的反而是卖出信号。
- **最小骨架**：
  ```python
  -ts_arg_min(returns, 5)     # 5 天内最低点出现在最远端 → 短期超跌
  ```
- **撞车点**：用 `ts_arg_min` 替代 `ts_delta` 是常见的差异化技巧。

### 3.3 质量（Quality）：公司是不是"好生意"

- **金融逻辑**：高 ROE、高毛利率的"好公司"长期跑赢。
- **最小骨架**（需要 fundamental6 数据集）：
  ```python
  rank(roe)                    # 净资产收益率排名
  ```
- **撞车点**：纯 `rank(roe)` 已被挖烂。**撞车少的做法**：`rank(roe) * rank(gross_margin) * rank(asset_turnover)`（多维质量打分）。

### 3.4 价值（Value）：便宜的股票会涨

- **金融逻辑**：低 PE、PB 的"便宜股"长期有回归价值。
- **最小骨架**：
  ```python
  -rank(pe_ratio)              # PE 越低（便宜）排名越高
  ```
- **撞车点**：单 PE 已被用烂。**改法**：用 `1 / rank(pe * pb)` 组合估值。

### 3.5 情绪（Sentiment）：分析师/媒体怎么看

- **金融逻辑**：分析师上调预测、新闻正面报道 → 股票后续上涨。
- **最小骨架**（你的原 alpha 就在这类）：
  ```python
  ts_decay_linear(
      ts_backfill(anl4_adjusted_netincome_ft, 120),
      240
  )
  ```
- **撞车点**：BRAIN 上 analyst4 数据集撞车率中等偏高。**改造点**：改成"预测的**变化量**"（详见第四节 案例 A），或者叠加 `news18` 做交叉验证。

### 3.6 波动率（Volatility）：低波股更稳

- **金融逻辑**：低波动率股票长期有"风险溢价"，回报更稳。
- **最小骨架**：
  ```python
  -ts_std_dev(returns, 60)    # 60 天波动率越低越好
  ```
- **撞车点**：纯波动率因子相对小众，**用 `option8`（期权隐含波动率）替代历史波动率是更高级的玩法**。

## 四、5 个可立即尝试的改造案例

你的原 alpha 是个"分析师预期"的 single-dataset 模板。下面 5 个改造**全都基于你的现有风格**，可以一个一个试。

### 案例 A：把"水平值"改成"60 天变化量" → 抓预测修订

- **改造前**：
  ```python
  ts_decay_linear(
      ts_backfill(anl4_adjusted_netincome_ft, 120),
      240
  )
  ```
- **改造后**：
  ```python
  ts_decay_linear(
      ts_delta(anl4_adjusted_netincome_ft, 60),    # 改：抓"60 天变化"
      240
  )
  ```
- **一句话解释**：原版看的是"分析师预测的当前值"，**改造后看的是"分析师过去 60 天是否在大幅上调预测"**。**预测修订**（earnings revision）比预测水平值对股价的指引更强。

### 案例 B：叠加价格动量过滤 → 基本面+技术面双重确认

- **改造后**：
  ```python
  group_neutralize(
      ts_decay_linear(
          ts_backfill(anl4_adjusted_netincome_ft, 120),
          240
      ) * ts_rank(close, 60),                       # 新增：60 天价格排名
      densify(subindustry)
  )
  ```
- **一句话解释**：原 alpha 只看"分析师看好谁"——**但如果价格趋势也向上，信号就更可信**。两个维度共振，过滤掉"基本面好但技术面崩坏"的伪信号。

### 案例 C：换数据集交叉验证（叠加 pv1 量价信号）

- **改造后**：
  ```python
  group_neutralize(
      ts_decay_linear(
          ts_backfill(anl4_adjusted_netincome_ft, 120),
          240
      ) * rank(volume / ts_mean(volume, 20)),       # 新增：相对成交量
      densify(subindustry)
  )
  ```
- **一句话解释**：在分析师看好的股票里，**进一步挑出"成交量突然放大"的那只**。量价共振是经典 alpha 思想。

### 案例 D：换中性化层级（从 Subindustry 试到 Industry）

- **改造后**（对比三档）：
  ```python
  # 选项 1：你的当前（Subindustry）
  group_neutralize(signal, densify(subindustry))

  # 选项 2：更粗（Industry）
  group_neutralize(signal, densify(industry))

  # 选项 3：最粗（Sector）
  group_neutralize(signal, densify(sector))
  ```
- **一句话解释**：中性化越细，**信号越纯**但**越容易被磨平**。**金融小白建议从 Subindustry 开始，若 Sharpe 下降到 1.3 以下就放宽到 Industry**。

### 案例 E：换衰减函数（ts_decay_exp vs ts_decay_linear）

- **改造后**：
  ```python
  # 选项 1：你的当前（线性衰减）
  ts_decay_linear(ts_backfill(anl4_adjusted_netincome_ft, 120), 240)

  # 选项 2：指数衰减（更激进，对最近数据更敏感）
  ts_decay_exp(ts_backfill(anl4_adjusted_netincome_ft, 120), 240)
  ```
- **一句话解释**：**指数衰减**像"距今天 1 天的权重 50%，2 天 25%……"——对最近数据极度敏感；**线性衰减**则是"每天权重平均减少"。**前者更"短线"、后者更"长线"**，看你的策略偏好。

> 💡 **改造的顺序建议**：A → D → E → B → C。**先单点改，再叠加改**。每改一次就回测一次，避免一次叠加太多变量。

## 五、中性化层级的合理选择

中性化是"剔除你不想要的风险"。**层级越细，剔除得越干净；但信号也可能被磨平**。

| 层级 | 含义 | 适用场景 | 风险 |
|---|---|---|---|
| **None** | 不做中性化 | 量价类（pv1、pv13） | 暴露行业 β，**Sharpe 容易虚高** |
| **Market** | 全市场中性 | 情绪类（news18、socialmedia8）、期权类（option8） | 几乎不损失信号，但**完全没剔除行业风险** |
| **Sector** | 11 大板块中性 | 风格因子（动量、波动率） | 太粗，**行业内信号差异被吃光** |
| **Industry** | 60+ 行业中性 | 基本面（fundamental2/6）、分析师（analyst4） | 平衡，**金融小白首选** |
| **Subindustry** | 150+ 细分行业中性 | 你当前的选择 | **最干净，但容易信号消失** |

**你的当前选择（Subindustry）**：
- ✅ **合理**：分析师数据天然有"行业属性"（科技股整体被看好很正常），用 Subindustry 中性化能剔除这种"行业整体偏好"。
- ⚠️ **风险**：在 150+ 个细分组里做中性化，**每组内只有十几只股票**。样本量小，统计意义弱，信号容易被噪音吃掉。
- 💡 **改进建议**：**先用 Industry 中性化试一遍**，若 Sharpe 下降 < 10% 但 Turnover 改善，**就用 Industry**。若 Industry 中性化后 Sharpe 暴跌到 1.0 以下，再退回 Subindustry。

## 六、多数据集交叉验证的方法

**单数据集 alpha = 单维度信号**。**多数据集 alpha = 多个独立信号互相验证**。后者撞车率低、稳健性高。

### 6.1 三大类数据的互补性

| 数据集类别 | 金融含义 | 看你想知道什么 |
|---|---|---|
| **基本面（fundamental2/6）** | 公司财务报表 | "这家公司好不好" |
| **分析师（analyst4/7）** | 卖方一致预期 | "市场怎么看这家公司" |
| **新闻/情绪（news18、socialmedia8）** | 文本情感分析 | "舆论怎么看这家公司" |
| **量价（pv1）** | 交易行为 | "市场资金怎么看这家公司" |
| **期权（option8/9）** | 衍生品预期 | "聪明钱（机构）怎么看这家公司" |

**核心思想**：**基本面告诉你"该值多少"，分析师/情绪告诉你"市场怎么定价"，量价/期权告诉你"聪明资金在做什么"**。三类数据互补。

### 6.2 一个完整的多数据集 alpha 模板

```python
# 基本面 + 分析师 + 量价 三源验证
signal_fundamental = rank(roe)                              # fundamental6
signal_analyst    = ts_decay_linear(                       # analyst4
    ts_delta(anl4_adjusted_netincome_ft, 60), 240
)
signal_technic    = ts_rank(close / volume, 20)              # pv1

# 三源信号共振（只保留"三源都看好"的）
composite = signal_fundamental * signal_analyst * signal_technic

# 中性化 + 阈值过滤
group_neutralize(
    trade_when(composite > 0, composite),                   # 只在多源共振时交易
    densify(industry)
)
```

**这个 alpha 在说什么**：
- "**在行业中性化的基础上，只买'基本面好 + 分析师上调 + 量价配合'三重确认的股票**"。
- **撞车风险**：因为同时引用 fundamental6 + analyst4 + pv1 三个数据集，**全平台撞车率极低**。
- **避坑**：不要简单相乘（会过度拟合），**用 rank 之后相加**更稳健。

## 七、常见"伪 alpha"的识别

工具跑出来的 alpha，**40% 可能是"伪 alpha"**——回测漂亮，实盘亏钱。下面 4 种最常见。

### 7.1 数据前瞻偏差（Look-ahead Bias）

- **症状**：回测 Sharpe > 4（异常高）。
- **原因**：用了"未来的信息"——例如用了今天的收盘价算今天的信号。
- **识别**：检查你的数据字段是否包含 `future` 后缀，或者你的 `ts_delta` 是否用了负数。
- **避坑**：永远用 `ts_delay(x, 1)` 确认信号比价格"早一步"。

### 7.2 幸存者偏差

- **症状**：回测时股票池"永远满 3000 只"。
- **原因**：用了"今天还活着的股票"算过去 10 年的回测，但 10 年里退市的股票没算进来。
- **识别**：BRAIN 平台自动处理这点（用 `pv1` 数据集），但用 `fundamental6` 时要小心——退市公司的数据可能"消失"。
- **避坑**：永远开启平台默认的 **Pasteurization（巴氏化）**。

### 7.3 过度拟合

- **症状**：IS（样本内）Sharpe = 2.5，OS（样本外）Sharpe = 0.3。
- **原因**：你（或工具）调了 50 个参数，硬把回测曲线"拟合"到历史。
- **识别**：Sub-universe test（更小股票池）是否通过？
- **避坑**：**永远保留 30% 数据做 OOS 测试**，不让"看"过去的数据影响"做"现在的策略。

### 7.4 流动性陷阱

- **症状**：回测 Returns 20%，但实盘一买就亏。
- **原因**：策略在"买不到"的小盘股上显示 20% 收益，但实际你买不进去那么多。
- **识别**：看 Turnover 是否 < 30%，Margin 是否 > 4bps。
- **避坑**：**始终在 TOP1000 上做回测**（比 TOP3000 更具流动性）。
