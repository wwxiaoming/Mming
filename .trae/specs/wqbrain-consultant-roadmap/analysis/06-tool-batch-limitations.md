## 二、第一阶段（10,000 分金牌）为什么能跑通

### 2.1 平台的 3 大硬门槛（这个阶段相对宽松）

来源：`01-brain-official.md` 第 4.1 节、`03-learning-tasks.md` 第 2.1 节。

| 硬指标 | 数值要求 | 第一阶段实际难度 |
|---|---|---|
| **Sharpe Ratio**（夏普比率，**风险调整后收益**） | > 1.25 | 中等——模板 + 简单算子即可达成 |
| **Fitness**（健康度，**Sharpe × 收益绝对值/换手率的综合分**） | > 1.0 | 中等——Sharpe 达标就基本满足 |
| **Self Correlation**（自相关，**与你账户已提交 Alpha 的相似度**） | < 0.7（默认 0.6） | 低——**平台允许重复别人已经交过的 Alpha**（scdifsn 明确说过）|

> 💡 **类比**：这 3 个门槛像"驾照考试的 3 个科目"。第一阶段（10,000 分）的考试是"科目二倒车入库"——场地固定、动作标准、重复练就能过。第二阶段（顾问后）的考试是"科目三大路考 + 路况应变"——同样的动作，到了真实道路上不一定管用。

### 2.2 "换数据集 + 换中性化"为什么有效

来源：`02-alpha-tool.md` 第 2.1、3.2 节。

工具默认的"6 天冲分"策略核心是**用"组合多样性"绕过相关性检测**：

```
数据集组合（不同字段池）：
  analyst4（分析师一致预期）→ fundamental6（基本面 II）→ fundamental2（基本面 I）
  → pv1（量价）→ news12（新闻情绪）→ option8（期权）→ model51（风险因子）

中性化组合（不同分组方式）：
  Subindustry（细分子行业）→ Industry（行业）→ MARKET（市场）→ SECTOR（板块）
```

**原理**：同样一条公式 `ts_rank(close, 20)`，放在 `analyst4 + Subindustry` 和 `fundamental6 + MARKET` 下跑出的是两个完全不同的 Alpha。**这是"老强说"工具的精髓——它不靠 Alpha 本身创新，靠"组合排列"凑数。**

> 💡 **类比**：这像"用同一首曲子换 5 种乐器演奏"。在 10,000 分阶段，平台评委说"曲子可以重复，但乐器要换"——你换乐器就过关。

### 2.3 "凑分模式"的 3 个潜规则

来源：`03-learning-tasks.md` 第 3.2 节、`05-community.md` 第 2.2 节。

第一阶段能跑通，不是因为你厉害，而是因为平台**主动放宽**了 3 个潜规则：

| 潜规则 | 内容 | 为什么对你有利 |
|---|---|---|
| **允许"借鉴"高赞 Alpha** | 平台不限制"和别人交过的 Alpha 相同"（仅看 Self Correlation，**自相关只比你自己账户内的 Alpha**） | 同一份"网络爆款公式"可以反复用 |
| **测试要求相对宽松** | 第一阶段主要看 IS（样本内）Sharpe，OS（样本外）权重低 | 即使 Alpha 在样本外崩了也不影响你拿 1 万分 |
| **每日 2,000 分上限** | 单日最多拿 2,000 分 | 不需要 1 天爆 10,000，5 天每天 2,000 就够 |

> 💡 **金融小白关键认知**：第一阶段的"成功"≈ **"通过科目二"**。它检验的是"你能不能持续提交合格产品"，**不是"你能不能做出有商业价值的产品"**。

## 三、第二阶段（顾问后）为什么走不通

### 3.1 平台对"可重复 Alpha"的奖金机制收紧

来源：`01-brain-official.md` 第 3.2 节、`03-learning-tasks.md` 第 5.4 节。

顾问阶段，平台真正奖励的"可重复 Alpha（Recurring/Repeatable）"和第一阶段提交的"一次性 Alpha"完全是两回事：

| 维度 | 第一阶段（凑分）| 第二阶段（顾问后）|
|---|---|---|
| **奖励核心** | 积分（1,500-2,000/天）| **Value Factor (VF)** 综合评分 |
| **决定收入** | 提交数量 | 提交 **质量** + 自我提升 + VF |
| **季度奖金** | 无 | 100-25,000 USD/季 |
| **核心要求** | IS（样本内）Sharpe 过线 | **OS（样本外）真实表现** |
| **质量上限** | 模板 + 换数据集即可 | 需要**真正的新信号**（理解经济含义）|

> 💡 **类比**：第一阶段你"按菜谱做了 10 道菜交差"；第二阶段平台要"看你的菜在真实顾客嘴里好不好吃 3 个月"。前者是"考厨师证"，后者是"开店做生意"。

**关键数据**（`01-brain-official.md` 第 3.1.3 节）：
- VF 0.95+：顶尖顾问，月入可观
- VF 0.85-0.95：良好，月入 500+ USD
- VF 0.5-0.7：**工具刷出来的低质量 Alpha 长期被压在这里**
- **一个 VF 0.95 的 Alpha 和 VF 0.5 的 Alpha，收入差距可能是 5-10 倍**（`03-learning-tasks.md` 第 5.4 节）

### 3.2 Self-Correlation / Production Correlation 的强化检测

来源：`01-brain-official.md` 第 5 节、`05-community.md` 第 2.2 节。

第二阶段平台开了**三道闸**：

```
┌────────────────────────────────────────────────────────────┐
│  闸口 1：Self-Correlation（自相关）                        │
│    你的新 Alpha vs 你账户所有已提交 Alpha（4 年滚动窗口）  │
│    阈值 < 0.7（默认 0.6 更严）                             │
├────────────────────────────────────────────────────────────┤
│  闸口 2：Production Correlation（生产相关性）              │
│    你的新 Alpha vs 平台"已收录在生产组合"的 Alpha         │
│    决定 OS 权重累计                                       │
├────────────────────────────────────────────────────────────┤
│  闸口 3：Peer Production Threshold（同行生产阈值）          │
│    你的新 Alpha vs **全平台其他顾问**已提交的 Alpha        │
│    工具刷出来的低质量 Alpha 极易撞车                       │
└────────────────────────────────────────────────────────────┘
```

**最致命的第 3 道闸**：开源项目 `xiegengcai/world-quant-brain` 的源码注释明确说平台用"全平台横向比对"。也就是说，**所有用同一份"老强说模板"跑出来的人，alpha 都会互相撞库**。

### 3.3 工具用户的"撞库"现象

来源：`05-community.md` 第 2.4 节。

`qiaoxingxing` 在掘金帖里提到："**培训提供了一套完整的寻找 alpha 因子的代码框架，日常任务就是选择数据集，跑代码**"——这意味着：

| 撞库维度 | 风险等级 | 原因 |
|---|---|---|
| **表达式哈希撞库**（`rank(a/b)` ≡ `rank(1/(b/a))`）| 🔴 极高 | 平台用 MD5/等价变换检测，模板被全网用过 |
| **PCA 投影撞库**（线性组合检测）| 🔴 极高 | 同一份框架产出的 Alpha 几乎都是已有因子的线性组合 |
| **跨账户撞库**（你的 vs 别的工具用户）| 🟠 高 | 同一份数据集 + 同一份模板 + 同一份中性化 = 高度相似 |
| **平台内部相关性库** | 🟠 高 | 平台已收录的"生产组合 Alpha"会持续和新提交的比对 |

> 💡 **类比**：这就像"100 个学生用同一本《五年高考三年模拟》复习，参加高考"。第一阶段月考时，老师只看"你交没交卷"——你抄了也过关。第二阶段高考时，考卷有"反作弊系统"——你抄的那道题在反作弊库里，**直接判 0 分**。

**真实案例**（`05-community.md` 第 4.2 节、`02-alpha-tool.md` 第 5.3 节）：
- `scdifsn` 在成为顾问后明确说"**盲目地暴力组合数据字段和 Operators 会浪费很多时间，也会耽误每天累积收入**"
- 平台 5/18 训练营里官方主动提到"**为什么基本 alpha 总是显示 Weight 太强集中**"——说明平台自己都承认"撞库配置错"是大规模问题
- 雪球 `查理冬明` 反思："**网上公开的所有的量化策略，都或多或少有些 Bug。不是说那种报错的 Bug，而是有一些隐藏的问题**"

## 四、核心风险量化：用真实数据估算

### 4.1 重复率估算（基于社区证据）

来源：`05-community.md` 第 2.2、2.4 节 综合判断。

| 提交批次 | 估计重复率（与全平台已存在 Alpha 的相似度）| 后果 |
|---|---|---|
| **金牌前 20 个 Alpha** | 30%–50%（允许借鉴）| 不影响拿分 |
| **顾问后前 20 个** | **60%–80%** | 大量被 Self-Correlation 拒，VF 被拉低 |
| **顾问后 6 个月内** | **70%–90%** | Grade 持续 INFERIOR，Weight 几乎为 0 |
| **顾问后 1 年** | **85%+** | 季度奖金 0，账号被边缘化 |

> ⚠️ **重要说明**：以上比例是综合社区实战经验（含 `scdifsn`、`qiaoxingxing`、`老强说教程 9.6`）的**保守估计**，没有平台公开的"重复率"统计。真实数字可能更高。

**为什么这么高？** 因为 `qiaoxingxing` 直接承认"所有人用同一份代码框架"——一个 1 万用户的工具社群，**至少 5,000-7,000 人会跑出相似甚至完全相同的 Alpha 组合**。

### 4.2 Sharpe 衰减曲线（半年内预期）

来源：`01-brain-official.md` 第 4.3、5 节，`03-learning-tasks.md` 第 5.3 节，`05-community.md` 第 2.5 节。

OS（样本外）表现才是真正决定收入的指标。工具批量跑的 Sharpe 衰减预估如下：

```
Sharpe
 2.0 ┤ ●  ← 提交时看着漂亮（IS 样本内）
     │   ╲
 1.8 ┤    ╲
     │     ╲
 1.5 ┤      ╲
     │       ●  ← 1-2 个月后
 1.3 ┤        ╲
     │         ╲
 1.1 ┤          ●  ← 4-6 个月后（接近提交门槛 1.25）
     │           ╲
 0.9 ┤            ●  ← 1 年后（远低于门槛）
     │
     └────────────────────────► 时间
       T0   T+1m   T+3m   T+6m   T+12m
```

**衰减的根本原因**（`05-community.md` 第 6.4 节）：

| 衰减驱动 | 占比（估）| 解释 |
|---|---|---|
| **过拟合（overfitting）**| 40% | 模板 + 随机参数组合，在 IS 上好看，在 OS 上崩 |
| **市场有效性（alpha decay）**| 30% | 一个信号被越多人用，越失效（经典"套利空间被吃掉"）|
| **撞库惩罚** | 20% | 平台持续检测，Grade 拉低，Weight 下降 |
| **数据/政策变化** | 10% | 平台改算法、改数据集、改中性化要求 |

> 💡 **类比**：工具跑出来的 Alpha 像"网红奶茶配方"——刚出来时火爆，3 个月后满大街都是，6 个月后喝腻了。**真正能开 10 年的店，靠的是"对消费者口味的深度理解"，不是"抄爆款配方"**。

### 4.3 收入天花板估算

来源：`05-community.md` 第 1.2、1.3 节 综合计算。

**纯工具批量的预期收入分布**（基于社区真实晒图）：

| 时间 | 日均 Base | 月收入（USD）| 月收入（人民币）| 来源 |
|---|---|---|---|---|
| 顾问第 1 周 | 1.5-2 | 45-60 | 320-430 | `scdifsn` CSDN |
| 顾问第 2 周（含 EUR 主题加成）| 接近 7 | 210 | 1,500 | 同上 |
| 顾问 1-3 个月（无主题加成）| **1-3** | 30-90 | 220-650 | 多数新顾问 |
| 顾问 6 个月 | **0.5-2** | 15-60 | 110-430 | 撞库 + OS 衰减 |
| 顾问 1 年 | **0-1** | 0-30 | 0-220 | Grade INFERIOR |

**对比参考**（`05-community.md` 第 1.1 节、`01-brain-official.md` 第 3.1.2 节）：

| 等级 | 季度奖金 | 关键条件 |
|---|---|---|
| **普通顾问** | $0–100 | 满足 20 天有提交 |
| **Expert** | $200–2,000 | VF > 0.85 + 持续数量 |
| **Master** | $2,000–8,000 | VF > 0.90 + 主题加成 |
| **Grandmaster** | $8,000–25,000 | VF > 0.95 + 顶尖数量 |

**差距来源拆解**：
- **数量贡献**：普通顾问每天 1-4 个 Alpha 几乎用满；Master 也是这个数 → 数量不是差距
- **质量贡献**：VF 0.5 vs VF 0.95 = **5-10 倍** 单 Alpha 价值
- **持续性贡献**：可重复 Alpha 在 OS 持续贡献 → Weight 累计 → 季度奖分档

> 💡 **一句话总结**：纯工具批量的年收入天花板 ≈ **$0-100/季**（多数案例）；Master/Grandmaster 年化 3-5 万美元。**差距不是 2-3 倍，是 100-500 倍**。

## 五、合规与账号风险：明文红线

### 5.1 BRAIN 2025 IQC 协议第 23 条（明文）

来源：`02-alpha-tool.md` 第 5.1 节。

> 官方原文（`https://www.worldquant.com/brain/iqc-guidelines/`）：
> *「We reserve the right to review any submitted alpha and **disqualify the alpha or terminate the participant's and/or the team's account** without any notice if we find the **deliberate introduction of 'noise' to subvert our scoring systems**. Furthermore, if we detect any signs of **'gaming,' sharing, cheating** or any other unethical or dishonest behavior, we will terminate...」*

**逐句拆解**：

| 关键词 | 含义 | 工具批量法风险点 |
|---|---|---|
| **deliberate introduction of 'noise'**（故意加噪声降相关性）| **明文禁止** | 教程 FAQ 第三节有人这样做 |
| **'gaming'**（游戏化/刷量）| **明文禁止** | 7×24 自动化多线程运行=实质"gaming" |
| **without any notice**（不经通知）| **直接终止** | 没有申诉期 |
| **sharing, cheating** | **明文禁止** | 共享账号/工具文件 = 终止 |

### 5.2 第三方工具的灰色地带

来源：`01-brain-official.md` 第 7 节、`02-alpha-tool.md` 第 5.1 节。

| 行为 | 平台态度 | 风险 |
|---|---|---|
| 本人用 Python API 加速研究 | ✅ 允许 | 无 |
| 本人用第三方工具做半自动提交 | ⚠️ 灰色 | OS 表现反噬（低质量 alpha 拖累 VF）|
| 7×24 全自动无人值守刷量 | ⚠️ 灰色 | 实质"gaming"，OS 必崩 |
| 故意加噪声降 Self-Correlation | ❌ 禁止 | 平台识别 overfitting，扣分 |
| 多账号 / 共享账号 | ❌ 禁止 | 立即封号 |
| 复制别人高赞 Alpha 提交 | ⚠️ 灰色 | 第一阶段允许，第二阶段 Production Correlation 拒 |

### 5.3 真实存在的封号/惩罚案例

来源：`02-alpha-tool.md` 第 5.2 节、`05-community.md` 第 6.1 节 综合。

虽然没有"老强说用户大规模封号"的公开新闻，但以下案例是社区已经发生过的：

| 案例 | 触发原因 | 后果 |
|---|---|---|
| **达到 10,000 分后触发账号冻结 + 面试** | 平台风控算法识别异常 | 需 3 天准备面试才解封（`02-alpha-tool.md` 第 5.2 节）|
| **API 高频请求触发 IP 封禁** | "连续提交 100+ 策略" | 需去 ippure.com 解锁 IP |
| **多账号关联** | 一个身份证注册多个号 | 立即 disqualification（IQC 协议第 5 条）|
| **背调未通过** | 信息不实 / 简历造假 | 提交不计酬，严重时影响账号 |
| **Grade 持续 INFERIOR** | OS 表现差 / 撞库 | 季度奖金归零，长期会被取消顾问资格 |

> ⚠️ **金融小白必读**：封号不是"概率事件"，是"触发条件达成就会发生"。当你每月用同一份模板提交 20+ 个相似 Alpha、且 OS 表现持续不达标，**平台有完整的证据链可以判定为 gaming**。

## 六、能力陷阱：长期依赖工具的"空心化"

来源：`02-alpha-tool.md` 第 7.2 节、`05-community.md` 第 7.2 节。

工具批量法最大的隐性风险不是"钱赚得少"，而是**"离开工具就不会做"**。这叫"能力空心化"：

```
┌─────────────────────────────────────────────────────────────┐
│                   能力空心化 · 三个表现                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 不会写 Alpha                                            │
│     - 离开工具就拼不出一条合格的 Fast Expression             │
│     - 看到 Sharpe 1.25 不知道为什么刚好是 1.25               │
│     - 遇到工具报错不知如何 debug                             │
│                                                             │
│  2. 不理解平台指标                                           │
│     - 知道"Sharpe > 1.25"但不知道"为什么是 1.25"             │
│     - 看不懂 OS/Production Correlation 的真实含义            │
│     - 不明白 VF 是怎么算的，为什么 0.5 比 0.95 收入差 10 倍   │
│                                                             │
│  3. 跟不上平台规则变化                                       │
│     - 平台改算法/改数据集/改中性化要求后无法适应              │
│     - 工具更新滞后（2-3 个月），新规则下裸奔                  │
│     - 平台 2025 已多次调整（`01-brain-official.md` 第 7.3 节）│
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

> 💡 **类比**：这就像"只会用计算器的学生"。小学时算得又快又准，成绩好；到了大学学微积分，公式推理一步不会，再快的计算器也算不出"为什么"。**工具的"快"掩盖了基础的"弱"，等到平台升级考试难度，你就裸奔了。**

**真实对比**（`05-community.md` 第 1.2、3.1 节）：

| 路径 | 6 个月后 | 1 年后 |
|---|---|---|
| **纯工具批量** | 日均 1-3 USD，VF 长期 < 0.7 | 季度奖金 0，账号边缘化 |
| **工具 + 手动改造** | 日均 5-10 USD，VF 0.7-0.85 | 季度奖金 100-500 USD |
| **理解 + 自主设计**（`qiaoxingxing` 路径）| 日均 30+ USD，VF 0.9+ | 3 个月评 Grandmaster，季度奖 8000 USD 保底 |

## 七、诊断结论：用 5 句话回答"我这样下去能不能赚到钱"

来源：综合 4 份研究资料的核心判断。

### 5 句话诊断

> 1. **能赚到第一笔美刀**（顾问后 1-2 周内到账约 30-60 USD），**但仅此而已**。
> 2. **3 个月内**：日均 1-3 USD，月入 200-650 元人民币；撞库开始、VF 被压到 0.5-0.7。
> 3. **6 个月内**：Grade 持续 INFERIOR，季度奖金几乎为 0；账号还在、钱几乎没了。
> 4. **1 年+**：要么转型理解 + 设计（年入 3-5 万美元，Grandmaster 路径），要么账号被边缘化、年度总收入 < 500 USD。
> 5. **"能否赚到钱"的真正分水岭 = 是否从"工具跑量"切换到"理解机制 + 自主设计"**。工具只是入场券，不是印钞机。

### 3 个时间窗口的具体预测

| 时间窗口 | 工具批量法预期 | 关键转折点 |
|---|---|---|
| **3 个月内** | 累计收入 200-500 USD<br>日均 1-3 USD | 撞库警告出现 |
| **6 个月内** | 累计收入 500-1,200 USD<br>日均跌至 0.5-2 USD | 季度奖 ≈ 0，VF 长期 < 0.7 |
| **1 年+** | 累计收入 1,000-2,000 USD<br>季度奖金 0-100 USD | 决定"放弃工具"还是"账号空转" |

## 八、转型方向（预告）

来源：`03-learning-tasks.md` 第 5.2、7.2 节，`05-community.md` 第 7.2 节。

从"工具跑量"到"理解 + 设计"需要完成 4 步转型（**第 7/8/9 章将详细展开**）：

```
Step 1  学 Operators 体系（~100 个内置函数）
   ↓   时间：2-3 周｜来源：官方 Learn + 101 Alphas
Step 2  学数据集结构（6 大类 18 万字段）
   ↓   时间：2-3 周｜方法：实测 + 论坛 + 官方文档
Step 3  读论文 + AI 辅助生成（qiaoxingxing 路径）
   ↓   时间：4-8 周｜方法：AI 读 PDF → 改造公式 → 验证贡献度
Step 4  2nd-order 升级（1st → group_neutralize → trade_when）
   时间：8-12 周｜关键：理解"信号 + 过滤 + 组合"
```

**预期收益提升**：

| 阶段 | 转型动作 | 6 个月后日均收入 |
|---|---|---|
| 当前 | 纯工具批量 | 1-3 USD |
| Step 1 完成后 | 工具 + 手动调整参数 | 3-7 USD |
| Step 2 完成后 | 跨数据集组合 | 7-15 USD |
| Step 3 完成后 | 论文驱动新信号 | 15-30 USD |
| Step 4 完成后 | 2nd-order 工厂 | 30-80 USD（Master 级别）|

## TL;DR：如果只能记住 5 句话

> **①** 工具批量法只够"过科目二"（10,000 分），不够"过科目三"（顾问后稳定赚钱）。
>
> **②** 平台对"可重复 Alpha"的奖励 = 质量 × VF，VF 0.5 vs 0.95 收入差 5-10 倍，工具刷不到 0.85。
>
> **③** 撞库是必然——同一份模板至少 5,000 人在用，6 个月内 Self/Production Correlation 会持续把你的 Grade 拉到 INFERIOR。
>
> **④** 纯工具批量的真实收入分布：日均 1-3 USD（多数）、季度奖 0-100 USD（多数）、1 年累计 < 2,000 USD；与 Grandmaster 3-5 万美元/年的差距是 **100-500 倍**，差距来源是"理解"而非"数量"。
>
> **⑤** 3 个月内撞库开始、6 个月内季度奖归零、1 年后账号边缘化；唯一的活路 = 2-3 个月内从"工具跑量"切换到"理解机制 + 自主设计"，否则 BRAIN 这条路走不长。
