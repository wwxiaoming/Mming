### 0.4 你需要先懂的 20 个核心概念

这 20 个词，**你不需要会写代码、也不需要会金融**，但必须能用自己的话讲出来。它们会反复出现在后续章节。

| # | 术语 | 一句话解释 | 生活类比 |
|---|---|---|---|
| 1 | **股票（Stock）** | 代表上市公司的部分所有权 | 投资奶茶店 1% 份额 |
| 2 | **做多（Long）/ 做空（Short）** | 赌涨/赌跌 | 囤房等升值 / 借相机卖掉等降价再买 |
| 3 | **头寸（Position）** | 你现在持有什么 | 钱包里有几张购物卡 |
| 4 | **Alpha / Beta** | 你"真正本事"赚的钱 / 跟着大盘走的钱 | 全班平均分 vs 你多考的那几分 |
| 5 | **因子（Factor）** | 解释股票收益的"调料"（动量/估值/波动率…） | 做菜时的盐、糖、醋 |
| 6 | **夏普比率（Sharpe）**⭐⭐ | 承担 1 份风险能赚多少；> 1.25 算过线 | 工作"性价比"——年薪 20 万加班 vs 15 万不加班 |
| 7 | **Fitness（健康度）** | Sharpe × √(收益绝对值/换手率)；> 1.0 算过线 | 性价比 × 持久性 |
| 8 | **换手率（Turnover）** | 每天交易额占本金比例；< 30% 较优 | 装修翻新频率——太频繁费钱，太懒错过机会 |
| 9 | **回撤（Drawdown）** | 资金曲线最惨的跌幅 | 爬山爬到 100 米，掉到 70 米，那 30 米就是回撤 |
| 10 | **边际收益（Margin）** | 每多交易 1% 换手能多赚多少 bps | 跑 1 公里多赚几块钱 |
| 11 | **信息系数 IC** | 因子预测值与实际收益的相关系数 | 天气预报说"80% 下雨"实际下雨 80% |
| 12 | **信息比率 IR** | IC 的稳定性 = IC 均值 / IC 标准差 | 不光要准，还要稳 |
| 13 | **回测（Backtest）** | 用历史数据模拟策略过去的表现 | 拿着今天的剧本看过去的比赛 |
| 14 | **过拟合（Overfitting）**⭐⭐ | 策略"死记硬背"历史，泛化失效 | 学生背了所有模拟题答案，换新题就崩 |
| 15 | **IS / OS（样本内/样本外）** | 训练集 / 测试集 | 考前做 1000 题（IS）vs 真正高考（OS） |
| 16 | **自相关（Self Correlation）** | 新 alpha 与你已交 alpha 的相似度；< 0.7 | 你的两份简历"长得有多像" |
| 17 | **中性化（Neutralization）** | 消除策略对市场/行业的偏向 | 班级考试排除干扰科目，看你真本事 |
| 18 | **延迟（Delay）** | 0=当天买、1=隔天买 | 看天气预报立刻出门 vs 看到明天也下雨再带伞 |
| 19 | **Value Factor（VF）** | 平台对你 alpha 质量的综合评分（0-1） | 平台给你打的"信用分" |
| 20 | **Challenge Score** | 挑战积分，10,000 分升 Gold | 平台经验值 |

> 💡 **小白提示**：⭐⭐ 是最关键的 4 个，先把它们吃透；其他的遇到再查附录 A 的速查表。

## 1. BRAIN 平台速通：等级、奖金、规则

> **这一章解决：作为金融小白，我到底在玩什么？钱从哪来？怎么算我能赚多少？**

### 1.1 等级体系：Bronze → Silver → Gold → Consultant → Expert → Master → Grandmaster

BRAIN 把用户分成 **6 级**，像游戏里的段位（来源 `research/01-brain-official.md` 第 2 节）。

```
注册账户 (无等级)
   ↓  累计 2,000 Challenge 积分
  Bronze（铜牌）
   ↓  累计 5,000 积分
  Silver（白银）
   ↓  累计 10,000 积分
  Gold（金牌）── 自动收到顾问邀请
   ↓  背调通过
  Consultant（正式研究顾问）── 收益可提现
   ↓  持续高质量提交
  Expert → Master → Grandmaster（季度奖金分档）
```

**关键规则**（来源 `01-brain-official.md` 第 2.2 节）：

| 规则 | 数值 | 你需要做什么 |
|---|---|---|
| 目标分 | 10,000 = Gold | 已经达成 ✅ |
| 每日上限 | 2,000 分/天 | 不要一天想爆 10,000 |
| 单 Alpha 得分 | 1,500-2,000（优秀）/ 1,000-1,500（普通） | 1 个 Excellent 就能拿满当日 |
| 积分更新 | T+1，北京时间 14:00-15:00 刷新 | 晚上提交不算当天 |
| 衰减机制 | 积分小幅衰减 | 每天稳定 1-2 个优于一次性堆量 |

**国内申请者特别路径**（来源 `03-learning-tasks.md` 第 1.4 节）：

- 国际：10,000 分 → 顾问邀请
- 国内：10,000 分 + 累计 ≥ 20 个 Alpha + 通过研究能力测试
- 实际周期：5-7 天达 10,000 分，1-3 个月走完背调

### 1.2 顾问的奖金结构：4 大模块

按 `research/01-brain-official.md` 第 3 节整理，顾问的"工资"由 4 部分组成：

```
你的顾问收入
├── 1. Base Payment（基础日津贴）        每日结算  1-120 USD/天
├── 2. Quarterly Payment（季度奖金）    每季结算  100-25,000 USD/季
├── 3. Competition（比赛奖金）          不定期    数额不等
└── 4. Referral（推荐奖金）             一次性    200 USD / 成功推荐 1 人
```

#### 1.2.1 Base Payment：日津贴 1-120 USD

| 决定要素 | 通俗解释 |
|---|---|
| (1) 提交数量（全球相对排名）| 不是看你一个人交多少，是看你在全平台顾问里排第几 |
| (2) 提交质量（全球相对排名）| 同上 |
| (3) 与自己历史相比的成长 | 进步越大分越高（平台鼓励"越做越好"）|
| (4) **Value Factors（VF）** | 平台对 alpha 质量的综合评分（0-1）|

> 💡 **类比**：Base Payment 像"底薪 + 绩效"——底薪按你的相对排名给，绩效按你的成长速度给。

实战案例（来源 `research/05-community.md` 第 1.2 节）：
- `scdifsn`（CSDN 博主）第 1 周：日均 1.5-2 USD
- `scdifsn` 第 2 周：单日最高 7 USD（含 EUR D1 主题 1.5x 加成）
- `qiaoxingxing`（掘金博主）3 个月评 Grandmaster：日均 ≈ 100 元人民币

#### 1.2.2 Quarterly Payment：季度奖 100-25,000 USD

| 等级 | 季度奖范围 | 触发条件 |
|---|---|---|
| 普通顾问 | 0-100 USD | 上季度 ≥ 20 天有提交 |
| Expert | 200-2,000 USD | VF > 0.85 + 持续数量 |
| Master | 2,000-8,000 USD | VF > 0.90 + 主题加成 |
| Grandmaster | 8,000-25,000 USD | VF > 0.95 + 顶尖数量 |

**关键规则**：
- 季度奖 = **平台分配的"权重账户（Weight）" × OS（样本外）表现**
- "上季度 < 20 天提交"= 当季季度奖归零
- **真正决定你能不能持续赚大钱的是"可重复 Alpha"**，不是"一次性 Alpha"

#### 1.2.3 比赛与推荐

- **比赛**：Super Alpha Competition、ACE、Global Alphathon、IQC 等；Global Alphathon 2025 奖金池 $100,000（冠军 $20,000）
- **推荐**：每成功推荐 1 人拿 200 USD（条件：被推荐人 10 天提交 + 保持顾问身份 1 个月+）

### 1.3 6 个核心评价指标的健康区间

| 指标 | 提交门槛 | 健康区间 | "好 alpha" 目标 | 通俗解释 |
|---|---|---|---|---|
| **Sharpe** | > 1.25 | 1.5-2.5 | ≥ 2.5 | 性价比 |
| **Fitness** | > 1.0 | 1.5-2.5 | ≥ 3 | 性价比 × 持久性 |
| **Turnover** | 1%-70% | 15%-40% | ≤ 30% | 换股频率 |
| **Returns** | 越高越好 | 5%-15% | ≥ 10% | 年化收益 |
| **Drawdown** | 越低越好 | < 10% | < 5% | 最惨跌幅 |
| **Self-Correlation** | < 0.7 | < 0.5 | < 0.3 | 与自己历史 alpha 的相似度 |

**例外豁免**：若新 Alpha 的 Sharpe 比历史相似 alpha **高出 10% 以上**（即 > 1.375），可豁免 Self-Correlation 限制（来源 `01-brain-official.md` 第 4.1 节）。

> 💡 **小白提示**：**Sharpe > 3 就要警惕过拟合，D1 模式下 Sharpe > 5 几乎一定是 overfit**。详细识别见第 5.7 节。

### 1.4 平台对"重复 alpha"的检测机制：3 道闸口

平台通过 **3 道闸口** 防重复（来源 `01-brain-official.md` 第 5 节）：

```
┌────────────────────────────────────────────────────┐
│  闸口 1：Self-Correlation（自相关）                │
│    你的新 alpha vs 你账户所有已提交 alpha           │
│    阈值 < 0.7（默认 0.6 更保守）                   │
├────────────────────────────────────────────────────┤
│  闸口 2：Production Correlation（生产相关性）      │
│    你的新 alpha vs 平台"已收录在生产组合"的 alpha   │
│    决定 OS 权重累计                                │
├────────────────────────────────────────────────────┤
│  闸口 3：Peer Production Threshold（同行生产阈值）  │
│    你的新 alpha vs 全平台所有顾问已提交 alpha        │
│    工具刷出来的 alpha 极易撞车                      │
└────────────────────────────────────────────────────┘
```

**暗线防护**（来源 `05-community.md` 第 2.2 节）：
- 表达式哈希比对（识别 `rank(a/b) ≡ rank(1/(b/a))` 这种等价变换）
- PCA 投影分析（判断是不是已有因子的线性组合）

## 3. 直击痛点："工具批量法"为什么长期赚不到钱

> **这一章解决：纯工具跑下去的真实收入是多少？为什么？**
>

### 3.1 现状还原：你的完整工作流

按 `analysis/06-tool-batch-limitations.md` 第 1 节，你的当前流水线长这样：

```
Step 1  选数据集与中性化
   │   - Day1-2: analyst4 + Subindustry
   │   - Day3-4: fundamental6 + industry / analyst4 + MARKET
   │   - Day5-6: fundamental2/6 + INDUSTRY / MARKET
   ▼
Step 2  工具批量生成 + 自动回测（7×24 多线程）

   ▼

   │   - Sharpe > 1.25  ✔

   ▼

> 💡 **类比**：这套流程像"用自动售货机做菜"——你只负责按按钮（点确认），工具负责"投币、取货、加热"（生成、回测、筛选）。好处是速度快、不用学厨艺；坏处是你永远不知道"菜"是怎么做出来的，机器换个配方你就吃不上饭。

### 3.2 第一阶段（10,000 分）为什么能跑通

来源 `06-tool-batch-limitations.md` 第 2 节 + `01-brain-official.md` 第 4.1 节 + `03-learning-tasks.md` 第 3.2 节。

**核心原因**：第一阶段平台**主动放宽**了 3 个潜规则。

| 潜规则 | 内容 | 为什么对你有利 |
|---|---|---|
| 1️⃣ 允许"借鉴"高赞 Alpha | 平台不限制"和别人交过的 Alpha 相同"（仅看 Self Correlation）| 同一份"网络爆款公式"可以反复用 |
| 2️⃣ 测试要求宽松 | 第一阶段主要看 IS（样本内）Sharpe，OS 权重低 | 即使 Alpha 在样本外崩了也不影响拿分 |
| 3️⃣ 每日 2,000 分上限 | 单日最多拿 2,000 分 | 不需要一天爆 10,000，5 天每天 2,000 就够 |

> 💡 **类比**：第一阶段的"成功" ≈ **"通过科目二"**（场地固定、动作标准、重复练就能过）。它检验的是"你能不能持续提交合格产品"，**不是"你能不能做出有商业价值的产品"**。

**"换数据集 + 换中性化"为什么有效**：

```
数据集组合：analyst4 → fundamental6 → fundamental2 → pv1 → news12 → option8 → model51
中性化组合：Subindustry → Industry → MARKET → SECTOR
```

原理：同样一条公式 `ts_rank(close, 20)`，放在 `analyst4 + Subindustry` 和 `fundamental6 + MARKET` 下跑出的是两个完全不同的 Alpha。**这是"老强说"工具的精髓——它不靠 Alpha 本身创新，靠"组合排列"凑数**。

### 3.3 第二阶段（顾问后）为什么走不通

按 `06-tool-batch-limitations.md` 第 3 节整理。

#### 3.3.1 平台对"可重复 Alpha"的奖金机制收紧

| 维度 | 第一阶段（凑分）| 第二阶段（顾问后）|
|---|---|---|
| 奖励核心 | 积分（1,500-2,000/天）| **Value Factor (VF)** 综合评分 |
| 决定收入 | 提交数量 | 提交 **质量** + 自我提升 + VF |
| 季度奖金 | 无 | 100-25,000 USD/季 |
| 核心要求 | IS（样本内）Sharpe 过线 | **OS（样本外）真实表现** |
| 质量上限 | 模板 + 换数据集即可 | 需要**真正的新信号**（理解经济含义）|

> 💡 **类比**：第一阶段你"按菜谱做了 10 道菜交差"；第二阶段平台要"看你的菜在真实顾客嘴里好不好吃 3 个月"。前者是"考厨师证"，后者是"开店做生意"。

**VF 的真实差距**（来源 `03-learning-tasks.md` 第 5.4 节）：

| VF 范围 | 收入水平 | 通俗解释 |
|---|---|---|
| 0.95+ | 非常高 | 顶尖顾问，靠这个吃饭 |
| 0.85-0.95 | 较高 | 良好水平，月入 $500+ |
| 0.7-0.85 | 良好 | 需要重点改进 |
| 0.5-0.7 | 较低 | **工具刷出来的低质量 Alpha 长期被压在这里** |

**一个 VF 0.95 的 Alpha 和 VF 0.5 的 Alpha，收入差距可能是 5-10 倍**。

#### 3.3.2 平台开了 3 道更严的闸

第二阶段平台开了**三道闸**（前文已述，此处强调第三道）：

> **最致命的第 3 道闸**：开源项目 `xiegengcai/world-quant-brain` 的源码注释明确说平台用"全平台横向比对"。**所有用同一份"老强说模板"跑出来的人，alpha 都会互相撞库**。

#### 3.3.3 工具用户的"撞库"现象

`qiaoxingxing` 在掘金帖里直接承认："**培训提供了一套完整的寻找 alpha 因子的代码框架，日常任务就是选择数据集，跑代码**"——一个 1 万用户的工具社群，**至少 5,000-7,000 人会跑出相似甚至完全相同的 Alpha 组合**。

| 撞库维度 | 风险等级 | 原因 |
|---|---|---|
| **表达式哈希撞库** | 🔴 极高 | 平台用 MD5/等价变换检测，模板被全网用过 |
| **PCA 投影撞库** | 🔴 极高 | 同一份框架产出的 Alpha 几乎都是已有因子的线性组合 |
| **跨账户撞库** | 🟠 高 | 同一份数据集 + 同一份模板 + 同一份中性化 = 高度相似 |
| **平台内部相关性库** | 🟠 高 | 平台已收录的"生产组合 Alpha"会持续和新提交的比对 |

> 💡 **类比**：这就像"100 个学生用同一本《五年高考三年模拟》复习，参加高考"。第一阶段月考时，老师只看"你交没交卷"——你抄了也过关。第二阶段高考时，考卷有"反作弊系统"——你抄的那道题在反作弊库里，**直接判 0 分**。

## 4. 你的"代表作 alpha"拆解

> **这一章解决：我的 alpha 现在长什么样？它在做什么？有什么坑？**

按 `analysis/07-alpha-quality-guide.md` 第 2 节重新组织，我们把你那条金牌阶段的"代表作 alpha"逐行拆开。

### 4.1 完整表达式

```python
group_neutralize(
    ts_decay_linear(
        ts_backfill(anl4_adjusted_netincome_ft, 120),
        240
    ),
    densify(subindustry)
)
```

### 4.2 逐行白话翻译

按 `07-alpha-quality-guide.md` 第 2.1-2.3 节拆解：

#### 第 1 层（最内）：`ts_backfill(anl4_adjusted_netincome_ft, 120)`

- **做什么**：分析师一致预期净利润（`anl4_adjusted_netincome_ft`）有时候有缺失值，**用最近一次有效的预测值向前回填，最多回填 120 天**。
- **金融含义**：相当于"分析师最近一次说这只股能赚 X 亿，因为分析师不会天天更新报告，最近一次就当现在用，120 天内都算有效"。
- **生活类比**：你最近一次量体重是 65kg，公司让你"用最近一次值"作为今天的体重数据，但如果距离上次称重超过 120 天，那这个数据就太旧了，作废。

#### 第 2 层：`ts_decay_linear(..., 240)`

- **做什么**：对刚才那个值，再做 **240 天的线性衰减加权平均**——最近 1 天的权重最高，240 天前的权重趋近于 0。
- **金融含义**：分析师预测**离今天越近，对未来股价的指引作用越大**。240 天前的旧预测跟当前股价的相关性已经很低了，权重压低。
- **生活类比**：你的"近 7 天平均工资"——最近的权重最大。

#### 第 3 层：`densify(subindustry)`

- **做什么**：把 `subindustry` 字段"稠密化"——处理有些股票没有细分子行业分类的情况，把未分类的打上"未知"标签，方便后续分组。
- **金融含义**：让 group_neutralize 能正确对所有股票做行业归类。

#### 第 4 层（最外）：`group_neutralize(..., densify(subindustry))`

- **做什么**：把"分析师预测+时间衰减"的信号，**在 subindustry（细分行业）分组内做中性化**。
- **金融含义**：消除"行业偏见"——如果科技股整体被分析师看多，你不希望 alpha 只是在"买科技股"；中性化后，**只保留"在科技股内部、分析师相对更看好某只"的信息**。
- **生活类比**：班级考试，语文数学都考 90 分，是数学特别强还是语文拖后腿？**中性化 = 排除这些干扰，看你的真本事**。

### 4.3 整体策略逻辑

把你的整条 alpha 翻译成一句"经济故事"：

> **"在每个细分行业内部，过去 240 天里分析师最新预测调整方向最积极的那只股"**。

这是一个非常合理的"分析师修订动量"思路。**但还有 4 个明显的天花板**（来源 `07-alpha-quality-guide.md` 第 0 节）：

| # | 短板 | 解释 |
|---|---|---|
| 1 | 没有动量/反转过滤 | 只看"分析师预期"，没看"价格趋势是否配合" |
| 2 | 没有交叉验证信号 | 单数据集（analyst4），如果叠加 pv1/news18 验证更稳 |
| 3 | 中性化层级可能不是最优 | Subindustry 太细（150+ 组），Industry（60+）可能更平衡 |
| 4 | 衰减函数单一 | 线性衰减 + 240 天固定，可以试 ts_decay_exp 指数衰减 |

### 4.4 为什么这条能跑出高 Sharpe

按 `07-alpha-quality-guide.md` 第 1 节整理"好 alpha 的健康指标"：

- ✅ **Sharpe > 1.25**：分析师预期信号是经典 alpha 源，基础扎实
- ✅ **Fitness > 1.0**：240 天衰减让信号稳定，不会每天剧烈变化
- ✅ **Sub-universe 测试**：行业中性化后子池测试容易通过
- ⚠️ **Turnover**：240 天衰减天然低换手（< 30% 概率高）
- ⚠️ **Self-Correlation**：模板特征明显，撞库风险中高

### 4.5 潜在风险与坑

按 `07-alpha-quality-guide.md` 第 7 节整理"伪 alpha 识别"：

| 风险 | 在你 alpha 中的具体表现 | 检查方法 |

| **过度拟合** | 模板 + 240 天固定参数 | 跑 OOS 测试，看 Sharpe 衰减 |
| **流动性陷阱** | TOP3000 vs TOP1000 | 建议用 TOP1000 |

> ⚠️ **特别提醒**：**D1 模式下 Sharpe > 3 就要警惕，> 5 基本就是 overfit**。立刻去检查字段是否引入了未来信息。

## 5. Alpha 质量提升：从"会用工具"到"会改 alpha"

> **这一章解决：怎么把工具跑出来的 alpha 改成"原创"的？**
>
> **重要：你的那条 alpha 会在多个改造示例中反复出现。**

### 5.1 好 alpha 的 5+1 个健康指标

按 `07-alpha-quality-guide.md` 第 1 节 + `01-brain-official.md` 第 4.2 节整理（6 个指标）：

| 指标 | 一句话解释 | 提交门槛 | 健康区间 | "好 alpha" 目标 |
|---|---|---|---|---|
| **Sharpe** | 风险调整后收益 | > 1.25 | 1.5-2.5 | ≥ 2.5 |
| **Fitness** | 性价比综合分 | > 1.0 | 1.5-2.5 | ≥ 3 |
| **Turnover** | 换股频率 | 1%-70% | 15%-40% | ≤ 30% |
| **Returns** | 年化收益 | 越高越好 | 5%-15% | ≥ 10% |
| **Drawdown** | 最惨跌幅 | 越低越好 | < 10% | < 5% |
| **Margin** | 每 1 美元交易能赚多少 bps | > 4 bps | > 4 bps | > 8 bps |

#### 5 个常见"伪 alpha"症状及避坑

| 症状 | 避坑 |
|---|---|
| Sharpe 高但 Fitness 低 | 看 Fitness 综合分，不被 Sharpe 单独漂亮迷惑 |
| Fitness 高但 Drawdown 爆表 | 永远要看 Drawdown 曲线 |
| Returns 高但 Self-Corr > 0.7 | 提交前自检 Self-Corr < 0.6 |
| Turnover 极低（< 5%）| 换手率"既不懒也不累"的甜区是 15%-40% |
| Sharpe 异常高（> 5）| **90% 是过拟合**——立刻查 look-ahead bias |

### 5.2 3 类算子的金融含义

按 `07-alpha-quality-guide.md` 第 2 节整理。

#### 5.2.1 时序算子（ts_xxx）：看一只股票"自己"的历史

| 算子 | 白话解释 | 生活类比 |
|---|---|---|
| `ts_decay_linear(x, d)` | 把过去 d 天的 x 加权平均，越近权重越大 | 你的"近 7 天平均工资" |
| `ts_backfill(x, d)` | 用最近的有效值向前回填最多 d 天 | 今年没发年终奖 → 用去年那次顶上去 |
| `ts_delta(x, d)` | 今天的 x 减去 d 天前的 x | 你现在比一周前胖了几斤 |
| `ts_rank(x, d)` | 当前 x 在过去 d 天里的排名（百分位） | 今天温度在近 7 天里排第几热 |
| `ts_mean(x, d)` | 过去 d 天 x 的简单平均 | 过去一周平均每天喝几杯咖啡 |
| `ts_std_dev(x, d)` | 过去 d 天 x 的波动 | 过去一周花钱"忽大忽小"的程度 |

**你原 alpha 的应用**：

```python
ts_backfill(anl4_adjusted_netincome_ft, 120)  # 缺失值用最近一次回填
ts_decay_linear(..., 240)                       # 240 天线性衰减
```

#### 5.2.2 截面算子（rank、quantile）：同一时间"横着比"

| 算子 | 白话解释 | 生活类比 |
|---|---|---|
| `rank(x)` | 同一时间，x 在所有股票里的排名（0-1 之间） | 全班 50 人，你考第几名 → 0.7 |
| `quantile(x, 5)` | 把所有股票分 5 档（0-4） | 把全班分成 5 个"梯队" |
| `scale(x)` | 按横截面缩放到总和 = 1 | 给每个班干部分配"话语权比例" |

**应用要点**：你的 alpha **没用** rank，但其实可以加一层 `rank(...)` 包裹，让分析师预期在"全市场分位"上有意义。

#### 5.2.3 向量/中性化算子（group_neutralize、densify、trade_when）

| 算子 | 白话解释 | 生活类比 |
|---|---|---|
| `group_neutralize(x, group)` | 在 group 分组内，把 x 减去组内均值 | 全班平均分一样，**看的是谁比平均分高多少** |
| `densify(group)` | 把 group 字段"稠密化" | 把"未填职业"的同学都打上"未知"标签 |
| `trade_when(cond, x)` | 当 cond 条件成立时才交易 x | 雨天才带伞、晴天就不带 |

**你原 alpha 的应用**：

```python
group_neutralize(..., densify(subindustry))  # 细分行业内中性化
```

### 5.3 6 类经典 alpha 模式构造模板

按 `07-alpha-quality-guide.md` 第 3 节整理。

| 模式 | 金融逻辑 | 最小骨架 | 撞车点 |
|---|---|---|---|
| **动量（Momentum）** | 强者恒强 | `ts_delta(close, 20)` | 🔴 极高（必须加风险调整） |
| **反转（Reversal）** | 涨多了会跌 | `-ts_arg_min(returns, 5)` | 🟡 中 |
| **质量（Quality）** | 好公司长期跑赢 | `rank(roe)` | 🔴 极高（必须多维合成） |
| **价值（Value）** | 便宜的股票会涨 | `-rank(pe_ratio)` | 🟡 中（叠加 PB） |
| **情绪（Sentiment）** | 分析师/媒体怎么看 | `ts_decay_linear(...)` | 🟠 较高（叠加新闻） |
| **波动率（Volatility）** | 低波股更稳 | `-ts_std_dev(returns, 60)` | 🟢 低（用 option8 更优） |

> 💡 **小白解释**：
> - **动量** ≈ "涨得好的股票还会涨"（惯性思维）
> - **反转** ≈ "涨太多了会跌回来"（均值回归）
> - **质量** ≈ "好生意长期跑赢差生意"
> - **价值** ≈ "便宜货迟早被市场发现"
> - **情绪** ≈ "舆论风向标"
> - **波动率** ≈ "稳稳的才是好生意"

**你的原 alpha 属于"情绪类"**（`analyst4` = 分析师情绪）。

### 5.4 5 个可立即尝试的改造案例

> **改造顺序建议**：A → D → E → B → C。先单点改，再叠加改。每改一次就回测一次。

#### 案例 A：把"水平值"改成"60 天变化量" → 抓预测修订

```python
# 改造前（你的原版）
ts_decay_linear(
    ts_backfill(anl4_adjusted_netincome_ft, 120),
    240
)

# 改造后
ts_decay_linear(
    ts_delta(anl4_adjusted_netincome_ft, 60),    # 改：抓"60 天变化"
    240
)
```

- **一句话解释**：原版看的是"分析师预测的当前值"，**改造后看的是"分析师过去 60 天是否在大幅上调预测"**。
- **金融含义**：**预测修订**（earnings revision）比预测水平值对股价的指引更强——分析师"调高预测"通常意味着他们看到了公司基本面的实质改善。

#### 案例 B：叠加价格动量过滤 → 基本面+技术面双重确认

```python
# 改造后
group_neutralize(
    ts_decay_linear(
        ts_backfill(anl4_adjusted_netincome_ft, 120),
        240
    ) * ts_rank(close, 60),                       # 新增：60 天价格排名
    densify(subindustry)
)
```

- **一句话解释**：原 alpha 只看"分析师看好谁"——**如果价格趋势也向上，信号就更可信**。两个维度共振，过滤掉"基本面好但技术面崩坏"的伪信号。
- **金融含义**：分析师看多 + 资金流入 = 双确认。

#### 案例 C：换数据集交叉验证（叠加 pv1 量价信号）

```python
# 改造后
group_neutralize(
    ts_decay_linear(
        ts_backfill(anl4_adjusted_netincome_ft, 120),
        240
    ) * rank(volume / ts_mean(volume, 20)),       # 新增：相对成交量
    densify(subindustry)
)
```

- **一句话解释**：在分析师看好的股票里，**进一步挑出"成交量突然放大"的那只**。量价共振是经典 alpha 思想。
- **金融含义**：分析师看多 + 资金涌入 = 三方验证。

#### 案例 D：换中性化层级（从 Subindustry 试到 Industry）

```python
# 选项 1：你的当前（Subindustry）
group_neutralize(signal, densify(subindustry))

# 选项 2：更粗（Industry）
group_neutralize(signal, densify(industry))

# 选项 3：最粗（Sector）
group_neutralize(signal, densify(sector))
```

- **一句话解释**：中性化越细，**信号越纯**但**越容易被磨平**。
- **建议**：**先用 Industry 中性化试一遍**，若 Sharpe 下降 < 10% 但 Turnover 改善，**就用 Industry**。若 Industry 中性化后 Sharpe 暴跌到 1.0 以下，再退回 Subindustry。

#### 案例 E：换衰减函数（ts_decay_exp vs ts_decay_linear）

```python
# 选项 1：你的当前（线性衰减）
ts_decay_linear(ts_backfill(anl4_adjusted_netincome_ft, 120), 240)

# 选项 2：指数衰减（更激进，对最近数据更敏感）
ts_decay_exp(ts_backfill(anl4_adjusted_netincome_ft, 120), 240)
```

- **一句话解释**：**指数衰减**像"距今天 1 天的权重 50%，2 天 25%……"——对最近数据极度敏感；**线性衰减**则是"每天权重平均减少"。**前者更"短线"、后者更"长线"**。

### 5.5 中性化层级的合理选择

按 `07-alpha-quality-guide.md` 第 5 节整理：

| 层级 | 含义 | 适用场景 | 风险 |
|---|---|---|---|
| **None** | 不做中性化 | 量价类（pv1、pv13） | 暴露行业 β，**Sharpe 容易虚高** |
| **Market** | 全市场中性 | 情绪类（news18、socialmedia8）、期权类（option8） | 几乎不损失信号，但**完全没剔除行业风险** |
| **Sector** | 11 大板块中性 | 风格因子（动量、波动率） | 太粗，**行业内信号差异被吃光** |
| **Industry** | 60+ 行业中性 | 基本面（fundamental2/6）、分析师（analyst4） | 平衡，**金融小白首选** |
| **Subindustry** | 150+ 细分行业中性 | 你当前的选择 | **最干净，但容易信号消失** |

**你的当前选择（Subindustry）**：
- ✅ **合理**：分析师数据天然有"行业属性"
- ⚠️ **风险**：在 150+ 个细分组里做中性化，**每组内只有十几只股票**。样本量小，统计意义弱
- 💡 **改进建议**：**先用 Industry 中性化试一遍**，若 Sharpe 下降 < 10% 但 Turnover 改善，**就用 Industry**

### 5.6 多数据集交叉验证

按 `07-alpha-quality-guide.md` 第 6 节整理：

**核心思想**：**基本面告诉你"该值多少"，分析师/情绪告诉你"市场怎么定价"，量价/期权告诉你"聪明资金在做什么"**。三类数据互补。

| 数据集类别 | 金融含义 | 看你想知道什么 |
|---|---|---|
| **基本面（fundamental2/6）** | 公司财务报表 | "这家公司好不好" |
| **分析师（analyst4/7）** | 卖方一致预期 | "市场怎么看这家公司" |
| **新闻/情绪（news18、socialmedia8）** | 文本情感分析 | "舆论怎么看这家公司" |
| **量价（pv1）** | 交易行为 | "市场资金怎么看这家公司" |
| **期权（option8/9）** | 衍生品预期 | "聪明钱（机构）怎么看这家公司" |

**一个完整的多数据集 alpha 模板**：

```python
# 基本面 + 分析师 + 量价 三源验证
signal_fundamental = rank(roe)                              # fundamental6
signal_analyst    = ts_decay_linear(                       # analyst4
    ts_delta(anl4_adjusted_netincome_ft, 60), 240
)
signal_technic    = ts_rank(close / volume, 20)              # pv1

# 三源信号共振（只保留"三源都看好"的）
composite = signal_fundamental * signal_analyst * signal_technic

# 中性化 + 阈值过滤
group_neutralize(
    trade_when(composite > 0, composite),                   # 只在多源共振时交易
    densify(industry)
)
```

**这个 alpha 在说什么**：
- "**在行业中性化的基础上，只买'基本面好 + 分析师上调 + 量价配合'三重确认的股票**"。
- **撞车风险**：因为同时引用 fundamental6 + analyst4 + pv1 三个数据集，**全平台撞车率极低**。

### 5.7 常见"伪 alpha"的识别

按 `07-alpha-quality-guide.md` 第 7 节整理。**工具跑出来的 alpha，40% 可能是"伪 alpha"**——回测漂亮，实盘亏钱。

| 类型 | 症状 | 识别 | 避坑 |
|---|---|---|---|
| **数据前瞻偏差（Look-ahead Bias）** | 回测 Sharpe > 4 | 查字段是否有 `future` 后缀 | 用 `ts_delay(x, 1)` 确认信号比价格"早一步" |

| **流动性陷阱** | 回测 Returns 20%，实盘一买就亏 | 看 Turnover < 30%，Margin > 4bps | **始终在 TOP1000 上做回测** |

## 6. 降低重复率：让 alpha 与众不同的 10 个技巧

> **这一章解决：怎么避免"和别人撞库"？**

### 6.1 什么是"重复 alpha"

按 `analysis/08-deduplication-strategy.md` 第 1 节整理。重复 alpha = 跟**全平台已有 alpha** 相似度过高的 alpha。

**BRAIN 的三道重复检测闸口**（前文 1.4 节已述，此处强调影响）：

- 0.6-0.7 区间：可接受，但**不要连续 3 个都在 0.6+**（会被 VF 惩罚）
- > 0.7：硬性拒收

**对顾问评分的具体影响**：
- **直接后果**：alpha 不能提交（绿色标记缺失），白浪费 1 个名额
- **间接后果**：长期提交低差异化 alpha 会拉低 VF 评分
- **季度奖金影响**：重复率高的顾问，OS 权重累计慢，季度奖金分档低

### 6.2 为什么工具用户最容易撞库

按 `08-deduplication-strategy.md` 第 2 节整理。

**核心原因**：
1. **模板代码框架是公开的**——`qiaoxingxing` 直接承认"所有人用同一份代码框架"
2. **换数据集/中性化的"伪差异化"陷阱**——把 `anl4` 换成 `anl4_eps`、从 `subindustry` 换成 `industry`、加 `delay` 都不算真正的差异化
3. **公开数据集 + 公开算子的组合空间有限**——BRAIN 公开数据集约 16 个，公开算子约 100 个

### 6.3 降重的 3 大核心思路

> **核心原则**：**至少在"数据集 + 中性化 + 算子组合"三个维度中改两个**——只改一个维度的"微调"几乎一定撞库。

| 思路 | 改造方向 | 难度 | 撞库率下降 |
|---|---|---|---|
| **A. 换数据集** | 从 anl4 换到 news18、option8、socialmedia8 等冷门数据集 | ⭐⭐ | 高 |
| **B. 换中性化层级** | 从 subindustry 换到 industry / sector / market | ⭐ | 中 |
| **C. 换算子组合** | 用不常见的 ts_decay_exp / trade_when / quantile | ⭐⭐⭐ | 高 |

**最推荐组合**：A + C（换数据集 + 换算子）。这是社区公认最有效的差异化路径。

### 6.4 10 条可立即执行的降重技巧

> 以下以你的 anl4 alpha 起点，每条都给**改造前/后 + 预期效果**。

#### 技巧 1：把"水平值"换成"变化量"

```python
# 改造前（撞库高发区）
ts_rank(anl4_eps_mean, 20)

# 改造后（修订变化）
ts_rank(anl4_eps_mean / ts_delay(anl4_eps_mean, 60) - 1, 20)
```
- **预期效果**：与模板 alpha 的相关性通常从 0.7+ 降到 0.3-0.5。

#### 技巧 2：把单一数据集换成多数据集交叉

```python
# 改造后：anl4 + pv1 交叉
ts_rank(anl4_eps_mean * close, 20)
```
- **预期效果**：与单一 anl4 alpha 的相关性可降到 0.2-0.4。

#### 技巧 3：把简单时序窗口换成多窗口组合

```python
# 改造后：ts_decay_linear + ts_rank 组合
ts_decay_linear(ts_rank(anl4_eps_mean, 60), 30)
```
- **预期效果**：撞库率显著下降，且对近期信号更敏感。

#### 技巧 4：用 trade_when 加上"开仓/平仓"条件

```python
# 改造后：涨势中持有，跌破均线平仓
trade_when(close > ts_mean(close, 60), ts_rank(anl4_eps_mean, 20), 0)
```
- **预期效果**：PnL 序列被条件过滤后与模板高度不同。

#### 技巧 5：把截面 rank 换成 quantile 分位数

```python
# 改造后
-quantile(anl4_eps_mean, 5)
```
- **预期效果**：与同字段 `rank` 类 alpha 的相关性可降到 0.3 以下。

#### 技巧 6：用 densify 处理的字段做更细的中性化

```python
# 改造后
group_neutralize(ts_rank(densify(anl4_eps_mean, subindustry), 20), subindustry)
```
- **预期效果**：与同字段的 group_neutralize alpha 相关性下降 20-30%。

#### 技巧 7：尝试用 option8 / option9 这类冷门期权数据

```python
# 改用 option8（隐含波动率）
-rank(opt8_iv_30d)  # 隐含波动率越低，未来收益越好（风险溢价）
```
- **预期效果**：与 anl4 用户的相关性 < 0.2（完全不同的数据域）。

#### 技巧 8：尝试 news18 / socialmedia8 这类情绪数据的非常规用法

```python
# 冷门用法 1：情绪的"动量"
ts_rank(news18_sentiment, 5) - ts_rank(news18_sentiment, 60)

# 冷门用法 2：情绪与基本面的"剪刀差"
ts_rank(news18_sentiment, 20) - ts_rank(anl4_eps_mean, 20)
```
- **预期效果**：与 anl4 模板 alpha 的相关性 < 0.25。

#### 技巧 9：用 ts_decay_exp 替代 ts_decay_linear

```python
# 改造后
ts_decay_exp_window(ts_rank(anl4_eps_mean, 60), 20, 0.95)
```
- **预期效果**：与线性衰减类 alpha 的相关性降到 0.4 左右。

#### 技巧 10：参考论文改造

| 论文 | 核心 alpha 模板 | 落地改造 |
|---|---|---|
| **Jegadeesh & Titman (1993) 动量** | `ts_rank(returns, 60) - ts_rank(returns, 250)` | 用 anl4 字段做"盈利预期动量" |
| **Fama-French (2015) 五因子** | `rank(value) + rank(momentum) + rank(size)` | 用 anl4 + fundamental6 多因子合成 |
| **Stambaugh & Yuan (2017) 误定价** | 12 个 anomaly 组合 | 用 anl4 + pv1 实现 mispricing 因子 |
| **Novy-Marx (2013) 毛利因子** | `rank(gross_profit/assets)` | 直接用 fundamental6 的 `gp/a` 字段 |
| **Asness, Frazzini (2013) 质量-动量** | `rank(quality) * rank(momentum)` | anl4 盈利稳定性 × pv1 价格动量 |

- **预期效果**：论文派 alpha 的"经济直觉"清晰，OS 表现稳健，长期价值高。

### 6.7 撞库检测与应对

按 `08-deduplication-strategy.md` 第 7 节整理。

#### 提交前自检

| 检查项 | 方法 | 阈值 |
|---|---|---|
| Self-Correlation | 用开源工具 `xiegengcai/world-quant-brain` 的 `self_correlation.py` | < 0.6 |
| Production Correlation | 平台 simulation 结果自动给出 | < 0.7 |
| 表达式哈希 | 自己用 MD5 算表达式哈希 | 哈希不同 ≠ 一定不同，但哈希相同 = 一定相同 |
| 视觉检查 | 你的 alpha 与最近 10 个通过 alpha 的 PnL 曲线是否"长得像" | 主观但有效 |

#### 被判重复后怎么改

**步骤**：
1. 查看平台反馈的"高相关 alpha ID"
2. 拉取那个 alpha 的 PnL 序列
3. 用 `pearson(your_pnl, their_pnl)` 算精确相关
4. 在**三个维度中至少改两个**（数据集/中性化/算子组合）
5. 重新提交

**核心原则**：**小修小补无效，必须做"结构性改造"**。

#### 看起来差异化，实际不算差异化

| 假差异化 | 为什么没用 |
|---|---|
| 单纯换 `delay`（1→0 或 0→1） | PnL 序列只是时移，相关性变化 < 0.05 |
| 单纯换窗口（20→21） | 皮尔逊对微窗口变化不敏感 |
| 单纯换中性化（subindustry→industry） | 中性化只调整"截面权重"，PnL 走势高度相似 |
| 给 alpha 加微小噪声 | **平台明文禁止**——IQC 协议第 23 条 |
| 改字段名（`anl4_eps_mean` → `anl4_eps`） | 字段不同但 PnL 几乎相同 |

#### 看起来撞车，实际不算撞车

| 情况 | 解释 |
|---|---|
| **不同 Region**（USA / EUR / ASI）的同样表达式 | Region 数据完全不同，平台**分开计算相关性** |
| **不同 Universe**（TOP3000 / TOP1000） | Universe 是子集筛选，PnL 序列会不同 |
| **不同 Delay**（D0 vs D1） | D0 算"日内 alpha"，平台有专门的相关性库 |
| **不同 Dataset**（anl4 vs anl7） | 平台按 dataset 隔离判重 |

### 6.8 本章 TL;DR

> **本章记住 3 句话**：
> 1. 降重 3 维度（数据集 + 中性化 + 算子组合）至少改 2 个，10 条技巧按需取用。
> 2. **小修小补无效**（换 delay/窗口/字段名都无效），必须"结构性改造"。
> 3. 用 2 年缩样本 + Self-Corr 自检，把撞库风险消灭在提交前。

**可立即执行的动作**：⭐ 从 10 条技巧里选 1 条（推荐"技巧 1：水平值换变化量"），用你的那条 anl4 alpha 做改造，记录改造前后的 Self-Correlation 数字。⭐

│   ├── 分析师+量价：anl4_signal * rank(volume)
│   ├── 三源共振：fundamental × analyst × pv1

## 附录 C：经典 alpha 模式速查

> 6 类经典 alpha 模式的最小表达式骨架。

### C.1 动量（Momentum）：强者恒强

```python
# 风险调整动量（推荐）
ts_rank(close, 20) / ts_std_dev(returns, 30)

# 二阶动量
ts_rank(ts_rank(close, 20), 5)
```

### C.2 反转（Reversal）：涨多了会跌

```python
# 短期反转
-ts_arg_min(returns, 5)

# 反转+波动率
-ts_arg_min(returns, 5) / ts_std_dev(returns, 30)
```

### C.3 质量（Quality）：好公司长期跑赢

```python
# 多维质量打分
rank(roe) * rank(gross_margin) * rank(asset_turnover)
```

### C.4 价值（Value）：便宜的股票会涨

```python
# PE + PB 组合
1 / (rank(pe_ratio) * rank(pb_ratio))
```

### C.5 情绪（Sentiment）：分析师/媒体怎么看

```python
# 你的原 alpha 属于这类
ts_decay_linear(
    ts_backfill(anl4_adjusted_netincome_ft, 120),
    240
)

# 改造版：抓预测修订
ts_decay_linear(
    ts_delta(anl4_adjusted_netincome_ft, 60),
    240
)
```

### C.6 波动率（Volatility）：低波股更稳

```python
# 历史波动率
-ts_std_dev(returns, 60)

# 期权隐含波动率（进阶）
-rank(opt8_iv_30d)
```

### C.7 三源共振模板（推荐组合）

```python
signal_fundamental = rank(roe)
signal_analyst    = ts_decay_linear(ts_delta(anl4_eps_mean, 60), 240)
signal_technic    = ts_rank(close / volume, 20)
composite = signal_fundamental * signal_analyst * signal_technic
group_neutralize(
    trade_when(composite > 0, composite),
    densify(industry)
)
```
