# 条目内容优化模板（content-template）

> 基于 5 个参考世界书的内容结构，归纳 7 类条目的优化目标模板。
> **YAML 格式外壳保持不变**（key / secondary_keys / enabled / position / order / content 块结构不动），仅优化 `content: |` 块内的文字与子章节。

---

## 模板 A：英雄设定（hero）

适用于：`英雄设定/01_问天.md` ~ `15_文斌.md`（15 个文件）

```yaml
content: |
  ## 基本信息
  - 姓名 / 年龄 / 身高 / 体重 / 阴茎长度
  - 属性评级（力量/魔法/综合）
  - 身份、代号、所属战队
  - 核心能力概述
  - 异香（蓝莓/柠檬/草莓等）
  - 武器/装备

  ## 核心敏感度
  - 列表化呈现敏感部位与等级（SSS/SS/S）
  - 至少含 4 个分区

  ## 核心资质
  - 4-6 条关键资质（被动/天赋/神纹/机械改造等）
  - 每条含机制说明与对调教/战斗的影响

  ## 技能
  - 3-5 个主动技能，含技能名、效果、消耗

  ## XP 触发场景
  - 至少 3 条具体 XP 场景
  - 每条含：触发条件 → 感官过程 → 身体反应 → 心理变化

  ## 外貌
  - 整体描述（≥ 100 字）
  - 面部、身材、标志性物品
  - 战斗/调教状态下的特殊形态

  ## 堕落神装
  - 装备外观、隐藏机关、战败/羞耻形态

  ## 性格
  - 核心特质（4+ 条）
  - 性相关特质（2+ 条）
  - 矛盾对比（表 vs 里）

  ## 说话方式
  - 至少 5 种场景下的台词（日常/被挑逗/羞耻/自述/与天火互动）
  - 含具体对白引用

  ## 说话风格语料
  - 至少 6 条不同情绪的台词
  - 覆盖：温和/被挑逗/羞耻/自述/兄弟互动/暴走/屈服

  ## 路线分支
  ### 路线A 收服者
  - 触发条件、初始状态、3-5 个剧情节点
  ### 路线B 小英雄
  - 触发条件、初始状态、3-5 个剧情节点
  ### 路线C 已收服
  - 触发条件、初始状态、3-5 个剧情节点

  ## 关联条目
  - 列出 3+ 个相关文件路径（世界观/剧情/NPC/装置等）
```

## 模板 B：NPC 设定（npc）

适用于：`NPC设定/01_魔神会干部.md` ~ `06_龙神.md`（6 个文件）

```yaml
content: |
  ## 身份与定位
  - 姓名 / 代号 / 势力归属 / 阶级（SS/S/A）
  - 核心职责（情报/改造/调教/实验等）

  ## 外貌
  - 整体描述、面部特征、身材、标志物

  ## 性格
  - 核心特质（4+ 条）
  - 行为模式（与天火/英雄的交互方式）

  ## 能力与装备
  - 核心异能概述
  - 装备/道具列表
  - 战斗记录或战绩

  ## 调教手段（适用 NPC）
  - 主要手段（感官剥夺/心理暗示/奖励机制等）
  - 对不同英雄的针对性策略

  ## 失败模式/弱点
  - 心理/能力上的弱点
  - 与天火对抗时的破绽

  ## 关联人物
  - 列出与其相关的其他英雄/NPC 路径

  ## 关联条目
  - 3+ 个相关文件路径
```

```yaml
content: |
  ## 规则概述
  - 适用场景 / 参与角色 / 触发条件

  - 双重人格/认知扭曲的触发

  ## 与其他规则的交互
  - 与补魔/战斗/任务系统的联动

  ## 关联条目
  - 3+ 个相关文件路径
```

## 模板 D：专属装置（device）

适用于：`专属装置/01_榨精自行车.md` ~ `15_誓约锁链·弱点反噬调教台.md`（15 个文件）

```yaml
content: |
  ## 装置概述
  - 名称 / 类型 / 改造原型 / 适用英雄
  - 设计目标（榨精/束缚/调教/感官剥夺等）

  ## 固定方式
  - 手部 / 腰部 / 腿部 / 脚部 / 颈部束缚细节
  - 束缚的强制姿态（俯身/仰躺/跪姿/M字腿等）

  ## 核心组件
  - 至少 3-5 个核心模块
  - 每模块含：结构 + 触发逻辑 + 与敏感度的联动

  ## 核心机制
  - 状态模式（至少 3-4 种）
  - 每模式含：触发条件 + 执行流程 + 终止条件
  - 联动机制（与英雄行为的反馈闭环）

  ## 调教效果
  - 即时效果 / 长期效果 / 心理效果

  ## 关联条目
  - 3+ 个相关文件路径（适用英雄/世界观/其他装置）
```


## 优化执行原则

1. **不动 YAML 外壳**：仅修改 `content: |` 块内的内容
2. **保留已有信息**：原有内容若符合模板某维度，则在原位置精炼扩展
3. **补全缺失维度**：按模板逐项检查，缺则补
4. **关联钩子必须**：每个文件文末必须含 `## 关联条目` 段
5. **说话风格语料至少 5 条**：英雄类条目硬性要求
6. **路线分支 3 选齐**：剧情类条目硬性要求
