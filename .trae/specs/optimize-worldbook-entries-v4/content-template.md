# 世界书条目内容模板 v4

## 模板 B：NPC 设定

```yaml
key: "<key名>"
secondary_keys: ["<键1>", "<键2>", ...]
enabled: true
position: 1
order: <数字>
content: |
  ## 概述
  - 一句话定义

  ## 详细设定
  - 身份与定位
  - 外貌
  - 性格
  - 能力与装备
  - 调教手段（如适用）
  - 失败模式

  ## 在三条路线中的状态
  - 路线A / 路线B / 路线C 下的不同呈现

  ## 关键事件 / 关键人物
  - 历史上/剧情中的标志性事件或关键人物

  ## 关联条目
  - 3+ 个相关文件路径（相对路径）
```

## 模板 E：地区 / 阵营 / 组织 设定

```yaml
key: "<key名>"
secondary_keys: ["<键1>", "<键2>", ...]
enabled: true
position: 1
order: <数字>
content: |
  ## 概述
  - 一句话定义

  ## 详细设定
  - 地理位置 / 组织结构
  - 气候 / 规模
  - 势力控制 / 派系
  - 标志性建筑 / 关键机构
  - 关键人物

  ## 在三条路线中的状态
  - 路线A / 路线B / 路线C 下的不同呈现

  ## 关键事件 / 关键人物
  - 历史上/剧情中的标志性事件或关键人物

  ## 关联条目
  - 3+ 个相关文件路径（相对路径）
```

## 通用规则

1. **每个文件必须以 ` ```yaml ` 开头并以 ` ``` ` 结尾**
2. **YAML 块内的 `content: |` 之后所有行必须缩进 2 个空格**
3. **必须含 `## 关联条目` 段**（含 3+ 个真实存在的相对路径）
4. **必须含 `## 在三条路线中的状态` 段**
5. **关联条目路径必须真实**（指向 `../英雄设定/01_问天.md` 等已存在的文件）
