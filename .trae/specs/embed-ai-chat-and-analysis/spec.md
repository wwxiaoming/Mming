# 网站内嵌 AI 聊天 + AI 分析助手 Spec

## Why

参考截图中的量化交易网站形态，希望在任意业务网站（行情/策略/分析类）中嵌入一个**侧边抽屉式 AI 助手**，既能像普通聊天机器人一样自由对话，又能结合当前页面上下文（例如当前股票、当前策略、当前图表数据）进行"分析型"回答。**API 不内置托管**，由用户在"设置"中自行填入第三方 LLM 的 Base URL、API Key、模型名，确保零成本、可换源、可私有化。

## What Changes

- 新增一个全局可复用的"AI 助手抽屉"组件，挂载在网站右上角
- 新增一个"设置"页/弹层，提供 LLM Provider 配置（Base URL、API Key、模型名、temperature、最大上下文条数）
- 新增"自由聊天"模式：用户输入任何问题，助手基于历史消息 + 页面上下文回答
- 新增"AI 分析"模式：用户点击预置快捷指令（如"分析一只股票的近期走势"、"帮我优化这个策略"），助手自动注入对应上下文
- 新增"附加上下文"机制：用户可主动把当前页数据（K 线、策略代码、自选股）作为一次性上下文塞入本轮对话
- 新增"重置/新建对话"按钮、"模型切换"下拉、"字数计数"显示
- 聊天历史仅保存在浏览器本地（localStorage），不发送到自家后端，API 请求直接从浏览器发往用户配置的 Base URL（支持 CORS 透传或可选自建代理）
- 新增"免责声明"水印（"以上分析仅供参考，不构成投资建议"之类）

## Impact

- Affected specs：无（新增功能）
- Affected code：
  - 新增：`src/components/ai-assistant/`（抽屉、消息列表、输入框、设置面板）
  - 新增：`src/lib/llm/`（Provider 抽象、OpenAI 兼容协议、流式请求、上下文裁剪）
  - 新增：`src/lib/ai-assistant/`（对话状态机、上下文注入、历史持久化）
  - 新增：`src/pages/settings/` 或 `src/components/settings/ai-settings-panel`
  - 修改：`src/App.tsx` 或 `src/layouts/`（挂载抽屉入口、设置入口）
  - 修改：业务页面（如行情页、策略编辑器）按需调用 `ai-assistant.open(payload)` 注入上下文

## ADDED Requirements

### Requirement: 全局 AI 助手抽屉

系统 SHALL 提供一个常驻在屏幕右侧、可折叠/展开的 AI 助手抽屉，类似截图中的"AI 量化助手"。

#### Scenario: 打开与关闭
- **WHEN** 用户点击导航栏或右下角的"AI 助手"入口
- **THEN** 抽屉从右侧滑出，宽度约 360~420px，覆盖部分右侧内容但不阻挡左侧主操作区
- **AND** 再次点击入口或抽屉内的"关闭"按钮可收起抽屉
- **AND** 抽屉状态在刷新页面后保持（写入 localStorage）

#### Scenario: 空状态欢迎语
- **WHEN** 用户首次打开抽屉且无历史对话
- **THEN** 显示一段欢迎语（如"您好！我是您的 AI 量化助手 👋，请选择您想做的事情："）
- **AND** 列出 4~6 个快捷指令气泡（"我是新手，先帮我跑一个策略看看效果"、"我有一段策略代码，想回测一下"、"我有一个策略想法，想验证可行性"、"我有一个实战策略，帮我优化一下"、"帮我分析一只股票的近期走势"等）
- **AND** 点击任一快捷指令等同于在输入框中输入对应文本并发送

### Requirement: 自由聊天

系统 SHALL 允许用户在抽屉底部的输入框中输入任意文本并按 Enter 发送，与已配置的 LLM 进行多轮对话。

#### Scenario: 发送与流式接收
- **WHEN** 用户在输入框中输入文本并按 Enter（Shift+Enter 视为换行）
- **THEN** 用户消息立刻出现在消息列表底部（右侧气泡）
- **AND** 助手回复以"打字机"流式方式从底部追加（左侧气泡），逐字渲染
- **AND** 发送期间输入框禁用，底部出现"停止生成"按钮，可中断当前流

#### Scenario: 历史上下文
- **WHEN** 用户已发送至少一轮对话
- **THEN** 后续每轮请求 SHALL 自动携带最近 N 轮历史（默认 N=10，可配置）
- **AND** 超出 N 轮的旧消息 SHALL 被裁剪，但系统提示词与本轮"附加上下文"必须保留
- **AND** 抽屉底部显示"AI 会记住本次对话的上下文，点 + 开始新对话"的提示

#### Scenario: 字数与限制
- **WHEN** 用户在输入框输入
- **THEN** 实时显示 `当前字符数 / 上限`（默认 500）
- **AND** 超出上限时禁止发送并提示

### Requirement: 设置中可自由配置 LLM API

系统 SHALL 在"设置"中提供完整的 LLM 配置项，支持任意 OpenAI 兼容协议的服务（DeepSeek、OpenAI、Moonshot、Together、Groq、自建 vLLM/Ollama 等等）。

#### Scenario: 配置项
- **WHEN** 用户进入"设置 → AI 助手"
- **THEN** 提供以下字段：
  - `Base URL`：API 根地址，例如 `https://api.deepseek.com/v1`、`https://api.openai.com/v1` 或本地 `http://localhost:11434/v1`
  - `API Key`：对应服务的密钥（密码框，可"显示/隐藏"）
  - `模型名`：下拉或自由输入，例如 `deepseek-chat`、`gpt-4o-mini`、`llama3.1:8b`
  - `Temperature`：滑块 0~2，默认 0.7
  - `最大上下文轮数 N`：数字输入，默认 10
  - `请求超时(秒)`：默认 60
  - `自定义系统提示词`：多行文本，用于替换默认 prompt
  - `请求模式`："浏览器直连"或"经本站代理"（用于解决 CORS）

#### Scenario: 测试连接
- **WHEN** 用户填好配置后点击"测试连接"
- **THEN** 系统 SHALL 用当前配置向 `Base URL + /models` 发送 GET 请求
- **AND** 成功后弹出"连接成功，检测到 N 个模型"提示
- **AND** 失败时显示具体错误（401/403/CORS/超时）并给出修复建议

#### Scenario: 持久化与隔离
- **WHEN** 用户点击"保存"
- **THEN** 配置 SHALL 写入 localStorage（key: `ai-assistant.config`）
- **AND** API Key SHALL 至少做一次 base64 编码 + 提示"密钥仅保存在本机浏览器"
- **AND** 抽屉顶部的"模型"下拉 SHALL 立即同步为新模型，无需刷新

#### Scenario: 缺失配置拦截
- **WHEN** 用户未配置 API 就尝试发送消息
- **THEN** 抽屉 SHALL 自动弹出设置面板高亮必填项
- **AND** 顶部出现横幅提示"请先在设置中配置 API"

### Requirement: 附加上下文（K 线/自选股/策略代码等）

系统 SHALL 提供一种"把当前页数据作为一次性上下文注入本轮对话"的机制，类似截图中的"附加 K 线"按钮。

#### Scenario: 手动附加
- **WHEN** 用户在抽屉中点击"+"或"附加上下文"按钮
- **THEN** 弹出一个上下文选择面板，列出当前页面可用的"上下文包"（K 线摘要、当前股票基本信息、当前策略代码、自选股列表等）
- **AND** 用户勾选后点击"插入"，所选上下文 SHALL 以"系统提示"形式附在**本轮**用户消息之后，**不会**进入历史

#### Scenario: 业务页面主动注入
- **WHEN** 业务页面（例如行情页）调用 `ai-assistant.open({ context: {...} })`
- **THEN** 抽屉 SHALL 自动展开并把传入的 context 作为本轮预置上下文
- **AND** 输入框中可预填一段引导语，例如"请帮我分析这只股票的近期走势"

#### Scenario: 自动注入策略代码
- **WHEN** 用户在策略编辑页点击"回测个股 600519"按钮
- **THEN** 系统 SHALL 把当前编辑框中的策略描述、买入/卖出条件连同股票代码 600519 一起注入本轮上下文
- **AND** 助手 SHALL 优先基于该策略回答，而不是泛泛而谈

### Requirement: AI 分析快捷指令

系统 SHALL 预置若干"分析型"快捷指令，点击即触发对应 prompt 模板。

#### Scenario: 预置指令集（可扩展）
- **WHEN** 用户点击以下任意预置气泡
- **THEN** 系统 SHALL 自动拼装 prompt 并发送：
  - "我是新手，先帮我跑一个策略看看效果" → 注入"用户为新手、希望从最简单的均线策略开始"的系统提示
  - "我有一段策略代码，想回测一下" → 引导用户粘贴代码
  - "我有一个策略想法，想验证可行性" → 引导用户用自然语言描述
  - "我有一个实战策略，帮我优化一下" → 引导用户贴出当前回测结果
  - "帮我分析一只股票的近期走势" → 引导用户输入股票代码或选择自选股

#### Scenario: 自定义指令
- **WHEN** 高级用户进入"设置 → AI 助手 → 快捷指令管理"
- **THEN** 可新增/编辑/删除/排序自己的快捷指令
- **AND** 每条指令包含"标题"和"prompt 模板"，模板支持 `{{context.stock}}` 等占位符

### Requirement: 免责声明

系统 SHALL 在每次 AI 回复末尾固定追加免责声明，**且必须**对最终输出做一次"免责声明必须存在"的客户端校验。

#### Scenario: 追加与校验
- **WHEN** AI 流式回复结束
- **THEN** 消息气泡末尾 SHALL 追加一行小字 "⚠️ 以上分析仅供参考，不构成投资建议"（或用户自定义的免责文本）
- **AND** 若用户自定义系统提示词试图移除该声明，客户端 SHALL 在保存时弹出二次确认
