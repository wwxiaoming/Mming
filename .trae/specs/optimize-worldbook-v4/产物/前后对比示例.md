# 前后对比示例

## 示例 1: 英雄设定/01_问天.md

### 优化前

```yaml
# 英雄：问天

```yaml
## 基本信息
position: 1
order: 21
keys: 问天, 秩序之子, 祖龙, 脚奴

## 基础资料
年龄: 14岁
...
content: |
  ## 外貌
  ...
```

**问题清单**:
- ❌ 缺 `constant` 字段(无法判断蓝灯/绿灯)
- ❌ 缺 `preventRecursion` / `excludeRecursion`(可能引发 token 爆炸)
- ❌ 缺 `scanDepth`(绿灯会全量扫描,造成浪费)
- ❌ `position: 1` 而非 `4` — 角色详情应在 D0 位置注入,而非 after_char
- ❌ keys 用中文逗号 + 空格(`问天, 秩序之子`)— 触发失效
- ❌ keys 仅 4 个,缺六元素掌控者等关键能力名

### 优化后

```yaml
# 英雄：问天

```yaml
position: 4
depth: 0
role: 0
constant: false
preventRecursion: true
excludeRecursion: true
scanDepth: 2
keys: "问天,祖龙,秩序之子,脚奴,六元素掌控者"

## 基本信息
order: 21

## 基础资料
年龄: 14岁
...
content: |
  ## 外貌
  ...
```

**改进清单**:
- ✅ `constant: false` 绿灯(多角色卡,按需加载)
- ✅ `position: 4, depth: 0, role: 0` 最高优先级注入点
- ✅ `preventRecursion + excludeRecursion` 双递归防 token 爆炸
- ✅ `scanDepth: 2` 仅扫最近 2 条消息
- ✅ keys 5 个,英文逗号无空格,覆盖全名+称号+能力名
- ✅ `order: 21` 保留参与排序

## 示例 2: 世界观/00_世界观总纲.md

### 优化前

```xml
# 世界观总纲

```xml
<yaml>
key: "世界观总纲"
enabled: true
position: 0
constant: true
content: |
  ## 世界背景
  ...
</yaml>
```
```

**问题清单**:
- ❌ 缺 `preventRecursion` / `excludeRecursion`
- ❌ `key: "..."` 单数,而非 `keys: [...]` 复数(SillyTavern 规范是 `key` 数组)
- ❌ `enabled` 字段多余(应该用 `disable` 字段)

### 优化后

```xml
# 世界观总纲

```xml
<yaml>
position: 0
constant: true
preventRecursion: true
excludeRecursion: true

content: |
  ## 世界背景
  ...
</yaml>
```
```

**改进清单**:
- ✅ 完整的 4 字段配置(position/constant/双递归)
- ✅ 删除多余 `key` 和 `enabled` 字段
- ✅ 保留 `content: |` 结构,内容未动
- ✅ 仍为 `position: 0` 蓝灯(全局常驻)
