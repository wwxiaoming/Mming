# 道具设定 03/04/05/06 整合优化 Spec

## Why
当前 `/workspace/帝王战队资料/世界书条目v4/道具设定/` 下的 03/04/05/06 四个文件（魔神会黑科技、通用调教道具、魔物相关道具、战斗装备与武器）内容彼此重复、维度互相交叉，且未与 Tavo_异能英雄调教模拟器这种"以具体道具条目为单位"的成熟结构对齐。需要将 4 个文件合并为 1 个综合"道具图鉴"文件，按 4 大类分组列出具体道具条目，参照异能英雄调教模拟器的"道具名:描述"精炼格式，使 30+ 件道具形成清晰的"图鉴式"索引。

**约束（基于用户反馈）**：

- 道具内容优先从 `帝王战队.txt` / `帝王战队番外.txt` 原文段落中找具体出现的物件
- 缺乏小说出处时可参考 Tavo_异能英雄调教模拟器 30 件精炼条目作模板

## ADDED Requirements

### Requirement: 单文件 4 大类整合
The new `07_道具图鉴.md` SHALL be a single YAML-wrapped Markdown file with one top-level `key: "道具图鉴"`，其 content 块 SHALL 按 4 个二级大类分组：
1. **魔神会黑科技类**（参考 03）
2. **通用调教道具类**（参考 04）
3. **魔物相关道具类**（参考 05）
4. **战斗装备与武器类**（参考 06）

#### Scenario: 文件存在且 YAML 合法
- **WHEN** 读取 `07_道具图鉴.md`
- **THEN** 文件以 ```yaml fence 开头，YAML 字段 key/secondary_keys/enabled/position/order 全部存在且可解析

### Requirement: 每类下列出 ≥ 5 件具体道具
The new file SHALL 在每个大类下用 H3 子标题列出至少 5 件具体道具，每件以"**道具名**: 描述"格式一行写完，参照 Tavo_异能英雄调教模拟器 order 134-163 的精炼风格。

#### Scenario: 四大类道具条目齐全
- **WHEN** 检查 4 大类下条目数
- **THEN** 魔神会黑科技 ≥ 5 条、通用调教道具 ≥ 5 条、魔物相关道具 ≥ 5 条、战斗装备与武器 ≥ 5 条

### Requirement: 不写路线分支、不写专属装置
The new file SHALL **不**包含 "在三条路线中的状态" 章节（路线分支由英雄/剧情设定承载），且 **不**收录 `/workspace/帝王战队资料/世界书条目v4/专属装置/` 目录下已独立的 15 件专属装置（如榨精自行车、千手魔罂粟、六轮犬穴榨精笼等）。

#### Scenario: 不出现路线分支与专属装置名
- **WHEN** Grep "路线A|路线B|路线C" 与专属装置名（榨精自行车|千手魔罂粟|六轮犬穴|七罪分身|银狐种马|风压涡流|时间油质|圣光自缚|龙裔|暗影格斗|九重阴茎|誓约锁链|高刺激肉棒|魔能系带|时空虫巢穴）
- **THEN** "路线A/B/C" 命中数 = 0（仅在 `## 关联条目` 中可引用路径）

### Requirement: 事实可追溯到小说原文
道具描述 SHALL 优先取自 `帝王战队.txt` / `帝王战队番外.txt` 实际出现过的物件，如：
- **魔神会黑科技**：罂粟种子培养器 / 神纹烙印装置 / 淫纹（魔王主教烙印）/ 智能紧身衣改造系统 / 复制量产设备

- **魔物相关道具**：淫魔触手 / 触手按摩足垫 / 罂粟种子（半活体）
- **战斗装备与武器**：堕落神装紧身衣 / 勇者臂环 / 神魔合金战甲 / 灵能编织纤维
