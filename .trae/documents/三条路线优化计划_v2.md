# 三条路线优化计划（取消固定顺序）

## 参照来源
- `/workspace/Tavo_小英雄's Lorebook (小英雄)_SARQ.json` —— 小英雄世界书的剧情结构
- `/workspace/帝王战队.txt` —— 原小说主线剧情
- `/workspace/帝王战队番外.txt` —— 原小说番外剧情

## 核心变更：取消固定收服顺序

**原方案**：固定收服顺序（问天→千瑞→子夜...）
**新方案**：根据开局选择的英雄动态生成剧情

## 动态剧情生成机制

### 变量系统
| 变量名 | 类型 | 说明 |
|--------|------|------|
| `selected_hero` | string | 开局选择的英雄 |
| `route_type` | string | 路线类型（A/B/C） |
| `hero_status` | object | 15位英雄的收服状态 |
| `plot_progress` | int | 剧情进度值（0-100） |
| `current_stage` | string | 当前阶段 |

### 开局选择
用户开局选择：
1. **路线**（A/B/C）
2. **英雄**（15位中任选一位作为起点）

### 动态剧情生成逻辑

#### 路线A（收服者）
根据`selected_hero`动态生成：
- 若选择核心8人之一：从该英雄开始收服，其余7人状态为"未收服"
- 若选择独立7人之一：从该英雄开始收服，核心8人状态为"未收服"

**阶段划分（通用）**：
| 阶段 | 名称 | 触发条件 | 核心内容 |
|------|------|---------|---------|
| A1 | 初遇观察 | 开局自动触发 | 天火观察目标英雄，发现弱点 |
| A2 | 弱点突破 | A1完成 + 特定事件 | 针对弱点制造独处机会 |
| A3 | 战斗捕获 | A2完成 + 战斗事件 | 利用战斗捕获目标 |
| A4 | 调教收服 | A3完成 + 调教场景 | 建立主从关系 |
| A5 | 羁绊深化 | 收服后日常触发 | 通过补魔深化羁绊 |

#### 路线B（小英雄）
根据`selected_hero`动态生成：
- 若选择核心8人之一：帝王战队日常线，与队友互动
- 若选择独立7人之一：独立剧情线，专属剧情指导

**核心8人剧情**：日常任务 + 队友羁绊 + 可选调教
**独立7人剧情**：专属剧情指导（见下文）

#### 路线C（魔王）
根据`selected_hero`动态生成：
- 若选择核心8人之一：该英雄为"已收服"状态，管理日常
- 若选择独立7人之一：该英雄为"未收服"状态，制定收服计划

**阶段划分（通用）**：
| 阶段 | 名称 | 触发条件 | 核心内容 |
|------|------|---------|---------|
| C1 | 继承遗产 | 开局自动触发 | 确认8位已收服英雄状态 |
| C2 | 管理日常 | C1完成后日常触发 | 管理已收服英雄 |
| C3 | 收服新目标 | 特定事件触发 | 针对未收服英雄制定计划 |
| C4 | 最终收服 | 完成C3 + 最终事件 | 完成全部收服 |
