# IS Summary 全指标合格线与改进方向表

## 一、指标总览（7 大核心指标）

| 指标 | 定义 | 合格线 | 不合格时建议 |
|---|---|---|---|
| **Sharpe（夏普比率）** | 风险调整后的平均收益 | **> 1.25（提交）/ > 1.5（IQC）** | 加 `rank` / 减换手 / 调整 `decay` |
| **Fitness（健康度）** | `Sharpe * sqrt(abs(Returns) / max(Turnover, 0.125))` | **> 1** | 先压 turnover |
| **Turnover（换手率）** | 交易额 / 持仓额 | **1% - 70%** | 用 `decay_linear` / 减少高频字段 |
| **Returns（年化收益）** | 年化平均盈亏百分比 | **越高越好** | 看 Drawdown 是否过大 |
| **Drawdown（最大回撤）** | 历史最大回撤幅度 | **越小越好** | 检查是否单股集中 |
| **Margin（边际收益）** | 每单位交易金额产生的平均盈亏 | **越高越好** | 减 `truncation` |
| **Self Correlation（自相关性）** | 与历史 alpha 的相关性 | **< 0.7** | 改一组字段 / 改中性化维度 |

## 二、逐项中文解读

### 1. Sharpe（夏普比率）

- **定义**：单位风险（年化波动率）所对应的超额收益，是衡量 alpha"性价比"的最核心指标。
- **合格线**：> 1.25 满足提交硬性条件，> 1.5 更易进 IQC 与高分池。
- **改进方向**：
  - 给原始信号加 `rank` 中性化以平滑极端值；
  - 适当降低换手从而压低波动率分母；
  - 调整 `decay`（如改 `decay_linear` 或拉长衰减窗口）让信号更稳定。

### 2. Fitness（健康度）

- **定义**：`Sharpe * sqrt(abs(Returns) / max(Turnover, 0.125))`，把收益、换手、Sharpe 揉成一个综合健康分。
- **合格线**：> 1，越高代表"以更少换手换到更多收益"。
- **改进方向**：
  - **优先压 turnover**：换手对 Fitness 的影响是平方根级的，下降效果立竿见影；
  - 在不显著降低 Sharpe 的前提下提升绝对收益 `Returns`；
  - 用更长 `decay` 或加 `trade_when` 条件让开仓更稀疏。

### 3. Turnover（换手率）

- **定义**：每期调仓中，买入+卖出金额占组合总市值的比例。
- **合格线**：1% - 70%；IQC 偏好的"甜区"约 10% - 90%（0.1 - 0.9）。
- **改进方向**：
  - 引入 `decay_linear` 让信号随时间线性衰减，平滑换仓节奏；
  - 减少日频、分钟级高频字段（如高频价格反转因子）；
  - 对信号做 `quantile` 截断或加入 `trade_when` 持仓条件。

### 4. Returns（年化收益）

- **定义**：回测区间内的年化平均盈亏百分比。
- **合格线**：越高越好，但需结合 Sharpe、Drawdown 一起看。
- **改进方向**：
  - 若收益过低，先检查 `Drawdown` 是否异常大（亏损期吃掉收益）；
  - 调整中性化维度（行业 / 国家 / 市值）以释放被压制的 alpha；
  - 替换或追加强解释力字段。

### 5. Drawdown（最大回撤）

- **定义**：回测期内累计收益曲线从峰值到谷底的最大跌幅。
- **合格线**：越小越好，过大说明存在极端风险暴露。
- **改进方向**：
  - **检查是否单股集中**：若最大回撤主要由 1 - 2 只股票贡献，需加强分散化（`rank` + 中性化 + 限额）；
  - 加 `truncation` 限制单只股票最大权重；
  - 在熊市/极端行情段加风控条件（`trade_when` 配合市场状态过滤）。

### 6. Margin（边际收益）

- **定义**：每单位交易金额产生的平均盈亏，反映"换手值不值"。
- **合格线**：越高越好；过低说明换来换去没赚到钱。
- **改进方向**：
  - 减少 `truncation`（截断过强会让信号在"该重仓时"不敢重仓）；
  - 提高信号集中度：用更窄的 `quantile` 区间聚焦最强信号；
  - 拉长持仓周期（`decay` 增大）让每笔交易有更长时间兑现收益。

### 7. Self Correlation（自相关性）

- **定义**：新 alpha 与平台已有 alpha 序列的相关系数，反映"撞车"程度。
- **合格线**：< 0.7；Sharpe > 1.375 时可豁免。
- **改进方向**：
  - **改一组字段**：换掉贡献最大的几个字段或替换数据集；
  - **改中性化维度**：从行业中性改为市值/动量/波动率中性；
  - 调整时间窗口或信号组合方式（线性组合 vs 乘积 vs 条件触发）。

## 三、改进优先级建议

| 优先级 | 操作 | 适用场景 |
|---|---|---|
| **P0** | 压 Turnover | Fitness 不达标的首选 |
| **P0** | 改字段 / 改中性化 | Self Correlation 超标 |
| **P1** | 调整 `decay` 形态 | Sharpe 在 1.25 附近徘徊 |
| **P1** | 检查单股集中 | Drawdown 异常大 |
| **P2** | 减少 `truncation` | Margin 偏低 |
| **P2** | 替换/追加字段 | Returns 持续低于同类 alpha |

## 四、典型组合搭配

- **Sharpe 低 + Turnover 高** → 加 `decay_linear` + 减少高频字段。
- **Fitness < 1** → 优先压换手，再考虑提收益。
- **Drawdown 突刺** → 查单股集中 + 加 `truncation`。
- **Self Correlation 高** → 改一组字段 + 改中性化维度，二选一即可见效。
