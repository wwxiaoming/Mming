# submission-strategist（提交策略建议）

> 根据学员当日账户画像（用量 / 余额 / 进度 / 已交 Regular 数）算出"今日提交计划"：能启动几个三阶段任务、阈值要不要提、距 Super Alpha 解锁还差几个。

---

## 1. 能力定位

**一句话**：把学员的"用量 / 余额 / 进度 / 已交 Regular 数"四要素算清楚，给出当日启动任务数 + Sharpe 阈值建议 + Super 解锁倒计时，**对接到 [`audit-report.md`](../../templates/audit-report.md) 模板的"基本信息 + 调整建议 + Checklist"三节**。

---

## 2. 触发场景

学员在老强工具"Alphas"或"统计"页看完成绩后，问"今天还要交几个 / 怎么排"。

- 学员原话示例：
  - "我今天 714/1800 用量、已经交了 50 个 Regular，还能不能再开几个三阶段？"
  - "今天剩余 100 次仿真，要不要启动新任务？"
  - "我已交 95 个 Regular、还差 5 个能解锁 Super，今天应该怎么排？"
  - "今天 Sharpe 阈值要不要动态调整？"

---

## 3. 输入格式

学员要贴出账户画像四要素（缺一不可）：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 当日用量 | 必填 | `已用 / 总额` 形式，例如 `714 / 1800` |
| 当日余额 | 选填 | 与用量互补，可由用量自动算出 |
| 累计已交 Regular 数 | 必填 | 整数（0-100+） |
| 累计已交 Super 数 | 选填 | 整数（解锁后才有） |

可附：当前时间（用于判断"还来不来得及跑完三阶段"）、策略偏好（保守/激进）。

**输入示例**：

```yaml
当日用量: 714 / 1800
累计已交 Regular: 50
累计已交 Super: 0
当前时间: 2026-06-02 14:30
策略偏好: 平衡
```

---

## 4. 输出格式

**复用 [`audit-report.md`](../../templates/audit-report.md) 模板**，但聚焦"提交策略"维度，包含：

1. **基本信息**（用量、已交 Regular、已交 Super）
2. **当日提交计划**（建议启动任务数 + 优先级 + 时间窗）
3. **阈值调整建议**（Sharpe 阈值随剩余量动态调整）
4. **Super Alpha 解锁倒计时**（差几个 + 预计日期）
5. **操作 Checklist**

> 模板填写规范见 [audit-report.md](../../templates/audit-report.md) 第三节、第四节。

---

## 5. 决策规则 / 评分细则

### 5.1 用量与剩余仿真

- [ ] **当日剩余仿真** = 总额 - 已用。
- [ ] **每个高质量 alpha 需 200-400 次仿真**（一阶段挖掘 + 二阶段筛选 + 三阶段精筛合计）。
- [ ] **建议启动三阶段任务数** = `floor(剩余 / 300)`，下取整，保证每个任务有充足仿真预算。

| 剩余仿真 | 建议启动三阶段任务数 | 备注 |
| --- | --- | --- |
| **< 100** | 0 | 只跑收尾，避免新任务半途没跑完 |
| **100 - 300** | 1 | 保守，只开 1 个三阶段 |
| **300 - 600** | 1-2 | 中等节奏 |
| **600 - 1000** | 2-3 | 标准节奏 |
| **1000 - 1500** | 3-5 | 充足，可激进 |
| **> 1500** | 5+ | 顶格，任务排队执行 |

### 5.2 Sharpe 阈值随剩余量动态调整

剩余仿真越少，越要把阈值提严以保证每个新启动任务的"产出质量"，避免浪费额度在低 Sharpe 废品上。

| 剩余仿真 | Sharpe 阈值建议 | 理由 |
| --- | --- | --- |
| **< 200** | **≥ 1.5** | 余量紧张，只放过强 IQC 级种子 |
| **200 - 500** | **≥ 1.25** | 标准提交门槛 |
| **> 500** | **0.75 - 1.0** | 余量充足，可用"种子门槛"批量挖 |

> 阈值依据：[wq-submission-standards.md](../references/wq-submission-standards.md) 第一节"三硬性条件"与第四节"IQC 加分指标"。

### 5.3 Super Alpha 解锁进度

按 [wq-submission-standards.md](../references/wq-submission-standards.md) 第三节"晋级路径"：

- [ ] Regular 累计 < 100 → 还差 `100 - 已交 Regular` 个；
- [ ] Regular 累计 ≥ 100 → **可解锁 Super**，每天 1 个高阶提交额度；
- [ ] 解锁后，Super 累计无明确上限，但日配额 = 1。

| Regular 累计 | 状态 | 距 Super 解锁 |
| --- | --- | --- |
| **< 50** | 起步 | 还差 50+ 个 |
| **50 - 80** | 进阶中 | 还差 20-50 个 |
| **80 - 99** | 临门一脚 | 还差 1-20 个 |
| **≥ 100** | 已解锁 | 0，可申请 Super |

### 5.4 时间窗判断（影响任务数上限）

- 北京时间 **18:00 之后** 启动的三阶段任务大概率跑不完当天，建议把任务数减半或只跑二阶段。
- 北京时间 **22:00 之后** 停止启动新任务，只等收尾。

### 5.5 改进动作优先级

| 优先级 | 触发条件 | 典型动作 |
| --- | --- | --- |
| **P0** | 剩余 < 100 + 还在启动新任务 | 立即停止新任务，专注收尾 |
| **P0** | 距 Super 解锁 ≤ 5 个 | 提 Sharpe 阈值到 1.25，专注"必交"而非"广撒" |
| **P1** | 距 Super 解锁 5-20 个 | 用 1.25 阈值，3-5 个三阶段任务 |
| **P1** | 距 Super 解锁 > 50 个 | 用 0.75-1.0 阈值，5+ 个三阶段任务批量挖 |
| **P2** | 已解锁 Super | 每天 1 个 Super + 1-4 个 Regular 搭配 |

## 6. 工作示例

### 示例 A：起步期（已交 50 个 Regular）

**输入**：

```yaml
当日用量: 714 / 1800
累计已交 Regular: 50
累计已交 Super: 0
当前时间: 2026-06-02 14:30
策略偏好: 平衡
```

**期望输出（摘要）**：

> **基本信息**：
>
> - 当日用量 714/1800，剩余 1086 次仿真
> - 累计已交 50 个 Regular，0 个 Super
>
> **当日提交计划**：
>
> - 建议启动 **3-5 个三阶段任务**，每个预算 200-300 次仿真
> - Sharpe 阈值建议 **0.75-1.0**（剩余充足，用"种子门槛"批量挖）
> - 时间窗 14:30 起，最晚 18:00 前排完队
>
> **Super 解锁倒计时**：
>
> - 距 Super 解锁还差 **50 个** Regular
> - 按当前节奏（每天 1-4 个 Regular），预计 **12-50 天** 解锁
> - 建议在临近 100 时（≥ 80）切换节奏，提阈值、专注"必交"
>
> **Checklist**：
>
> - [ ] 启动 3-5 个三阶段任务
> - [ ] Sharpe 阈值 0.75-1.0
> - [ ] 18:00 前停止新任务
> - [ ] 当日 Regular 目标 1-4 个
> - [ ] 复制今日最优表达式作为"高产模板"

### 示例 B：临门一脚（已交 95 个 Regular）

**输入**：

```yaml
当日用量: 1700 / 1800
累计已交 Regular: 95
累计已交 Super: 0
当前时间: 2026-06-02 20:00
策略偏好: 保守
```

**期望输出（摘要）**：

> **基本信息**：
>
> - 当日用量 1700/1800，剩余 100 次仿真
> - 累计已交 95 个 Regular，0 个 Super
> - 当前时间 20:00，三阶段大概率跑不完
>
> **当日提交计划**：
>
> - 建议启动 **0 个新任务**，只跑收尾
> - Sharpe 阈值建议 **≥ 1.5**（剩余紧张，只放强 IQC 级）
> - 如果有正在跑的二阶段种子：把剩余额度全部灌进去冲三阶段
>
> **Super 解锁倒计时**：
>
> - 距 Super 解锁还差 **5 个** Regular
> - 明天起切换节奏：阈值提到 1.25，每天只交 1-2 个但**保证成功**
>
> **Checklist**：
>
> - [ ] 停止启动新任务
> - [ ] 剩余 100 次仿真全部给到在跑任务
> - [ ] 明天起 Sharpe 阈值 1.25
> - [ ] 距 Super 还差 5 个，预计 3-5 天解锁
> - [ ] 解锁后立即申请 Super 通道
